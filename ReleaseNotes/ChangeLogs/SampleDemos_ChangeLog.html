Changes in CTR-SDK 11.4.0 (2015/09/07 - for System Updater 0.24.66)

Additions and Changes

Additions and Changes37-D01:(BOSS) Added supplement stating that deleting tasks each time with the BOSS sample is not recommended [SampleDemos]

Added supplementary comment regarding sample_nadl_simple in the BOSS sample demo stating that deleting and re-registering tasks each time is not recommended because there is a possibility that duplicate data is distributed.
Because registering and deleting each time in the sample makes it easy to understand, we will leave it the way it is.

Additions and Changes37-D02:(DevEnv) Added definition for alignment restriction [SDK|SampleDemos]

Added definitions for alignment and size restrictions for the following libraries.

  • aacdec
  • aacenc
  • act
  • camera
  • cec
  • dlp
  • fs
  • gx
  • hid
  • http
  • ir
  • os
  • rdt
  • ro
  • snd
  • socket
  • uds

Additions and Changes37-D03:(EC)★ Added eShop REST API feature that provides functions for listing and purchasing service items through the independent server [SDK|SystemUpdater|SampleDemos]

Added eShop REST API feature that provides functions for listing and purchasing service items through the independent server.
In addition, the following was also added.

  • Manual containing required information for developing the independent server using the eShop REST API ($CTR_SDK/documents/TechnicalNotes/EShopRestApiManual)
  • eShop REST API ec/ServiceItemRestApi Demo

Bug Fixes

Bug Fixes37-D04:(EC) Fixed upper character limit to Details Information and Legal/Disclaimer Information displayed when purchasing a service item [SDK|SystemUpdater|SampleDemos]

Fixed upper limit to character count of Details Information and Legal/Disclaimer Information displayed when purchasing a service item.
For Details Information, 64 characters was increased to 128 characters, and there is no longer a limit for line breaks.
For Legal/Disclaimer Information, 512 characters was increased to 1024 characters, and there is no longer a limit for line breaks.
The EC library and ServiceItem sample was fixed along with this.

Bug Fixes37-D05:(HID) Fixed error handling in hid/ExtraPad/AppRecommendedFlow sample [SampleDemos]

Fixed inappropriate error handling during file operation in the hid/ExtraPad/AppRecommendedFlow sample.

Bug Fixes37-D06:(HID) Fixed bug where Circle Pad Pro operates unintentionally during search [SDK|SampleDemos]

There was a bug where the Circle Pad Pro operated unintentionally during search.
In order to fix this, the working memory size passed to the nn::hid::CTR::ExtraPad::Initialize function was increased from 4096 to 12288 bytes.
Also, along with the working memory size increase, it was verified that the nn::hid::CTR::ExtraPad::Initialize function can receive this size.

Bug Fixes37-D07:(NFP) Fixed processing when the NFP sample demo POWER button is pressed [SampleDemos]

Fixed incorrect processing when the POWER button is pressed in the NFP sample demo.
The NFP library must be closed when the POWER button is pressed and causes a transition from the application to the POWER Menu.
