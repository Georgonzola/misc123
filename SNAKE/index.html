SNAKE

1. Overview

The following table provides an overview of SNAKE compared to CTR.

Item Changes From CTR in SNAKE
Clock Three times faster.
L2 Cache 2 MB (new).
VRAM 6 MB (unchanged).
Main Memory 124 MB (an increase from CTR).
Circle Pad Pro Built into the system (ExtraPad-compatible).
Camera More sensitive than CTR (the resolution is the same as the CTR).
NFC Built into SNAKE.

SNAKE has two operating modes. They are as follows.

Name CPU Clock L2 Cache
Standard mode Same as CTR Disabled
Extended mode Three times that of CTR Enabled

The following table summarizes the features available for each desc that can be used when creating applications for SNAKE.

Name NFC Memory Available Operating Modes desc Filename
Standard application Available 64 MB Standard mode Application.desc
Extended application Available 124 MB Extended mode ExtApplication.desc

The operating modes shown in this table are for after the nn::applet::CTR::Enable function is called. All applications run in extended mode after they are started.

The system operates in extended mode while system applets, such as the HOME Menu, POWER Menu, development menu, and Internet Browser, are running.

2. GPU Hangs When Multiple Textures Are Used

On CTR systems, there is a problem with the GPU hanging when multiple textures are used. This problem has been fixed on SNAKE. When an application runs on SNAKE, the use of multiple textures does not cause the GPU to hang.

Note, however, that this does not apply when the application runs on a CTR system. For more information about this problem, see 3DS Programming Manual: Basic Graphics.

3. SNAKE-Specific Titles

SNAKE-specific titles only start from the HOME Menu and will not run with CTR. These titles cannot be imported to CTR. For information about how to create SNAKE-specific titles, see ctr_makebanner.

Back to top

4. NFP Compatible Titles

When running a title supporting the NFP library on SNAKE, make sure that you run in a mode that uses the system internal NFC. This is the same for forced CTR compatible mode due to debugging. Use CTR when using the NFC reader/writer.
