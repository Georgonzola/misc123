1. Introduction

This documentation describes how to use the eShop REST API from an independent server and a client.

The eShop REST API provides features to service providers running independent servers for listing and purchasing service items. It also provides features for getting license information on service items that users own.

Note:

For the 3DS, there are two ways to implement service item handling features. One method uses the eShop REST API, while the other method uses the ECSV library. See Implementation Methods in 7.1.1. 3DS E-Commerce Item Types of the 3DS Overview for details on the differences between the two methods.

1.1. Using the eShop REST API in Applications

Registration in a network usage application website called OMAS is required in order to use the eShop REST API in applications. Refer to the 3DS Overview for details on OMAS and the exact application procedure.
