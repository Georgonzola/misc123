5. Cautions When Using the eShop REST API

5.1. Using the eShop REST API

Access to Nintendo eShop

The device with a registered Nintendo Network account must have accessed the Nintendo eShop once before using the eShop REST API. You cannot use eShop REST API functionality even if you start an application that supports eShop REST API before having accessed the Nintendo eShop.

Accessing the Nintendo eShop may be done by starting the EC applet within the application, or by starting the Nintendo eShop application.

Note:

The EcDevTool can be used to access the Nintendo eShop when testing in the development environment.

Setting ZIP Code and Province

If the country is set to USA, then the purchase cannot be made unless a ZIP Code is entered. Similarly, if the country is set to Canada, the province must be set. If these settings are not performed, a 030-3551 error is returned when making a purchase.

As the ZIP Code and province cannot be set in the purchase confirmation menu applet, users must run the EC applet and use the EcDevTool to set them in items such as the purchase samples in downloadable content.

Note:

In a production environment, the ZIP code and province can also be set from the Nintendo eShop.

5.2. Setting the System Date and Time

When using the eShop REST API, communications between the application and the independent server are encrypted by using a certificate.

Note that the communication may be determined to be illegal and cause an error if the system date and time are not correctly set.

5.3. Implementing System Update Check

When using the eShop REST API, use the nn::ec::CTR::NeedsSystemUpdate function to verify system update.

As this does not check the system update in the purchase confirmation menu applet, even if the display within the applet is changed due to a system update, if the update is not detected on the application side, it advances to the purchase while still using the old display. Because of that, use the nn::ec::CTR::PurchaseConfirmationApplet::RequestShowDialog function to check the system update before calling the purchase confirmation menu applet.

Although it is not necessary to check the system update each time the purchase confirmation menu applet runs, run the check once for each login to the independent server.

5.4. Behavior When Changing the Language Information in System Settings

The lang parameter can be set on several of the eShop REST API requests. This parameter can also be omitted. If omitted, the Nintendo Network account eShop language is used. However, this value could be different from the value set in the system.

The account eShop language setting is updated to the language set in the system when Nintendo eShop or an EC applet, connects to it. If it is necessary to switch languages and check the notation, a display within the application should be performed after updating the eShop language of the account and therefore after the language has been updated.
