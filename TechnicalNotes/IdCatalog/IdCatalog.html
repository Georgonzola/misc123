ID List

(2011/12/12)

Introduction

About This Document

This document presents a list with descriptions of the various identifiers used in the CTR-SDK.

The meanings and usage of terms used in this document and the CTR-SDK may differ from their general meanings and usage.

Explanations of the Items in Each Description

The Descriptions section of this document provides various items of information. Below is an explanation of the meaning of each item.

Specifier
Explains who is responsible for specifying which values are used in the application.
ID Latitude
Explains to what extent the application has any latitude in using ID values.
If there is no latitude, then the value must be used as specified. When there are a number of IDs, then the application can select which ID to use from among multiple IDs, based on the specification you want to implement for the application.
Format
Describes which format the ID value takes.
Usage
Describes what the ID value is being used for.
How to Get
Describes how to get the ID.
Normal Value
When there is some latitude in using ID values, this describes which value should be used in normal situations.
Where Used
Describes where this ID is used for application development.
Supplemental Information
Additional descriptions that do not fit into any of the above categories.

Unique ID (UniqueId)

Specifier
Nintendo.
ID Latitude
None.
Format
A bit array of 20 bits.
Purpose
The ID that identifies the application.
How to Get
Issued by Nintendo.
Normal Value
Use the value specified by Nintendo.
Where Used
Used in the following places.
  • The setting in the RSF.
  • As the expanded save data ID.
  • In creation of system-specific ID.
  • In creation of local communication ID.
  • As the StreetPass unique ID.
  • In creation of infrared communication ID.
Comments
The CTR system identifies an application by its unique ID. In addition, many other IDs including communication-related IDs are created from this unique ID.

Expanded Save Data ID (ExtSaveDataId)

Specifier
The application selects from the unique IDs or expanded save data numbers specified by Nintendo.
ID Latitude
Choose either the assigned unique ID or an expanded save data number.
Format
A bit array of 64 bits.
Purpose
The ID for identifying expanded save data.
How to Get
Use the value of the unique ID.
Other expanded save data numbers are issued by Nintendo.
Normal Value
Use the unique ID of the target application.
Where Used
Used in the following places.
  • The setting in the RSF.
  • The MountExtSaveData function.
Comments
Normally, the unique ID assigned to the target application is used as is for the expanded save data ID. But if multiple applications will share the expanded save data, choose to use the unique ID of one of these applications for the expanded save data ID.
In creating the expanded save data ID, you cannot base it on any arbitrary unique ID value. You must use the unique ID of one of the applications that will use the expanded save data.
Contact Nintendo when you want to get expanded save data anew (except for expanded save data for the local host). Examples include when you want to share expanded save data with other applications, or when you want to use multiple sources of expanded save data in one application.

Local Communication ID (LocalCommunicationId)

Specifier
The application selects from the unique IDs specified by Nintendo.
ID Latitude
Can select from among assigned unique IDs.
Format
A bit array of 32 bits.
Purpose
Used during UDS communication to prevent mixing of communication among different applications.
Communications can only take place between applications that use the same local communication ID.
How to Get
Call the CreateLocalCommunicationId function using the unique ID for the argument to get this value.
Normal Value
Use the value created from the unique ID of the target application.
Where Used
The uds library API.
Comments
Normally, the local communication ID is created from the unique ID assigned to the target application. But if applications are communicating with each other, the local communication ID is created from the unique ID of one of the applications and shared by all.
In creating the local communication ID, you cannot base it on any arbitrary unique ID value. You must use the unique ID of one of the applications that is participating in the communications.
In UDS communications, a separate mechanism prevents communication between applications, even when the same local communication ID is being used.

Nintendo Network ID (NNID)

Specifier
User
ID Latitude
None.
Format
A string of 6 to 16 characters, including alphanumeric characters and the period (.), hyphen (-). and underscore (_).
Purpose
Used to identify users with services that through Nintendo Network account authentication.
How to Get
Use the nn::act::GetAccountId function to get this ID. A null string is returned, however, when no Nintendo Network account is associated with the system.
Normal Value
Varies with each Nintendo Network account.
Used By
Used optionally by the application.
Comments
Although users cannot change their Nintendo Network IDs at will, Nintendo may change IDs for operational reasons. Use the Nintendo Network account principal ID when managing users for applications or independent servers. After an account ID has been modified or deleted, the original ID can never be used again.

Once a Nintendo Network ID has been deleted, a different value is configured to the ID.
It is possible to maintain an association with the same Nintendo Network ID even after initializing the system.
When using System Transfer, the same values are set on the destination system.

Network Account Principal ID (Network_Account_PrincipalId)

Specifier
Specific to a Nintendo Network account.
ID Latitude
None.
Format
A bit array of 32 bits.
Purpose
Used to identify users with services that through Nintendo Network account authentication.
How to Get
Use the nn::act::GetPrincipalId function to get this ID. NN_ACT_INVALID_PRINCIPAL_ID is returned, however, when no Nintendo Network account is associated with the system.
Normal Value
Different for each system.
Used By
Features that rely on Nintendo Network account authentication, and the ECSV library.
Comments
Once a Nintendo Network ID has been deleted, a different value is configured to the ID.
When the same Nintendo Network ID is associated to the system after initializing it, the same value is configured to the principal ID.
When using System Transfer, the same values are set on the destination system.

Universally Unique Identifier (UUID)

Specifier
Indexed by the system upon generation.
ID Latitude
None.
Format
A bit array of 128 bits.
Purpose
Can be used to apply a unique identifier for data and objects generated by the application.
How to Get
The nn::act::CTR::GenerateUuid function can generate this ID as needed.
Normal Value
Different each time its generated.
Used By
Used optionally by the application.
Comments
The CTR system generates UUIDs that comply with version 1 specifications based on RFC 4122.
UUIDs are handled overall as one value; do not use them as values as with node IDs or generation date and times.

System Account UUID

Specifier
Specific to the system account.
ID Latitude
None.
Format
A bit array of 128 bits.
Purpose
The application has the option of using this ID to identify the system account.
System accounts associated with the same Nintendo Network ID have different UUIDs.
In addition, because the ID space differs among applications, this ID cannot distinguish accounts across titles.
Use the Nintendo Network account principal ID to identify users across platforms and devices.
How to Get
Use the nn::act::CTR::GetUuid function to get this ID.
Normal Value
Varies with each system account.
Used By
Used optionally by the application.
Comments
The CTR system generates and assigns a UUID for every system account generated. The UUID remains unmodified as long as the system account is not deleted.
System accounts are deleted and regenerated at the following times. When that happens, a new UUID is assigned.
  • The Nintendo Network ID is deleted.
  • Format the system.

During a System Transfer, the system account is moved to the destination system. As such, the destination system can get the same value.
