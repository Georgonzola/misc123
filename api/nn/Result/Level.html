nn::Result::Level Enumerated Type

Syntax

#include <nn/Result.h>
enum Level;

Value

Value Description
LEVEL_INFO Success. There is additional information.
LEVEL_SUCCESS Success. No additional information.
LEVEL_FATAL Fatal system level error. Software recovery is not possible. Jump to the support center guide sequence.
LEVEL_RESET Unexpected error. The best course of action is to transition to the reset sequence. Reinitializing the module is not likely to lead to recovery, but it may be possible to recover by resetting.
LEVEL_REINIT Error that requires the module to be reinitialized.
LEVEL_USAGE Programming error. Indicates a programming error that always occurs with the argument values that were specified, regardless of the particular internal state of the library.
LEVEL_PERMANENT Common error. You cannot try again.
LEVEL_TEMPORARY Temporary failure. You can try again immediately with the same argument. Succeeds if the number of attempts is relatively low.
LEVEL_STATUS Expected failure.
LEVEL_END These values are not used.

Description

Enumerated type indicating the severity of an error. (Do not use this type for error handling.)

Avoid using this value directly for error handling. This value can be used during debugging to understand the content of an error.

LEVEL_INFO, LEVEL_SUCCESS

The function's processing succeeded. When there is additional information included, the processing can branch based on each function's reference.

The definition of a success depends on the function. For example, for a function that confirms the existence of a connection, even when the connection has not yet been made, if the confirmation was a success, LEVEL_SUCCESS might be returned. In such cases, when result.IsSuccess() would indicate that connection was successful, you do not end up with the intended process.

LEVEL_FATAL

In general, this level of error is supplemented internally, and the system is automatically suspended, with a message displayed prompting the user to contact support.

However, there are errors where the application can continue to play, for example when the camera fails, or even when the device has had a fatal problem. The error is processed based on the reference of the function in use.

LEVEL_RESET

Indicates an error for which recovery is possible with a system reset. The error is processed based on the reference of the function in use.

LEVEL_REINIT

Indicates an error for which recovery is possible with a module reinitialization. The error is processed based on the reference of the function in use.

LEVEL_USAGE

Indicates a bug in the code from which the call was made. This error can be returned when there is a mistake in the way a function is used or a mistake in an argument. Revise it to the correct way based on the reference.

LEVEL_PERMANENT

Indicates a bug in the usage from which the call was made.

Indicates an error that can be resolved without affecting the operations of other applications by revising the calling code. This error is returned when there is an internal problem that results from mistakes made in the procedures used for a library.

LEVEL_TEMPORARY

With this error, success is possible on a retry. The error is processed based on the reference of the function in use.

LEVEL_STATUS

Indicates an error where processing is necessary based on the value of Result.

The error is processed based on the reference of the function in use. The system may also detect this automatically and process it as a fatal error.
