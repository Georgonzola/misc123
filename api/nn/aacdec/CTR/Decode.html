nn::aacdec::CTR::Decode Function

Syntax

bool Decode(
     const u8 * pFrame,
     s16 * pOutputL,
     s16 * pOutputR,
     s32 nFrameSize = 0
);

Parameters

Name Description
in pFrame A pointer to the bitstream.
out pOutputL A pointer to the (left) output PCM buffer.
out pOutputR A pointer to the (right) output PCM buffer.
in nFrameSize The frame size (not needed for the ADTS format).

Return Values

Returns true if successful; otherwise, returns false.

Description

Decodes the frame.

The buffers pointed to by pFrame, pOutputL, and pOutputR must all be located in device memory. In addition, those addresses must be aligned to DECODE_BUFFER_ALIGNMENT (32 bytes).

The amount of PCM data output in one call to the function is 1024 samples (2048 bytes) per channel. When you decode mono AAC data, silence (0) is output to the buffer specified by pOutputR. When you decode dual mono AAC data, the same PCM data output to pOutputL is also output to pOutputR. If decoding fails, the buffers specified by pOutputL and pOutputR are filled with silence (0).
