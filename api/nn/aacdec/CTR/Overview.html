nn::aacdec::CTR Namespace

Description

Namespace for the AAC decoder (AACDEC) library. (It can also be written without the CTR portion.)

Introduction

The AAC Decoder library provides the application with functionality for decoding AAC data.
AAC is an audio standard widely used for such applications as music streaming and mobile phones.

Using the Library

Three files are required to use the AAC Decoder library.

  • nn/aacdec.h
  • libnn_aacdec.a
  • resources/dspcomponents/dspaudio.cdc

When using this library, be sure to run the initialization function before calling the decode function.
Decoding makes use of the DSP, so be sure to initialize the DSP library and load the DSP component ahead of time.
If you are passing RAW data to the decode function, you must specify the sampling frequency using the initialization function.
When passing AAC data to the decode function, be sure to use one complete frame of data, and to set the start of that frame (if it is ADTS data, the start of the ADTS header) to the front of the buffer.
Running the decode function outputs 1024 samples per channel of PCM data.

Reverse Playback Feature

To output PCM data that can be played in reverse, enable the reverse playback feature when you initialize the library, and pass the frames of AAC data in reverse time order.
For PCM data that can play in reverse, the samples are stored in reverse time order in the buffer.

Implementation Notes

You must call the nn::dsp::CTR::LoadComponent function instead of the nn::dsp::CTR::LoadDefaultComponent function, and load the special DSP component file.
To use non-standard DSP components, the DSP component file must be in main memory (it does not have to be device memory) until the nn::dsp::CTR::UnloadComponent function is called.

The Get functions (with the exception of the function that gets the frame size from the ADTS header) will not return a valid value until after 1 frame of data is finished being decoded.

With Advanced Audio Coding, the AAC decoder stores the state of the previously decoded frame in the library and uses that state to decode the current frame. For this reason, if you begin decoding a new AAC file without clearing the internal state, the decoding result of the previous file will affect the decoding of the current file. If you are going to decode a different file, first call nn::aacdec::CTR::Finalize to clear the internal state and then call nn::aacdec::CTR::Initialize to re-initialize the library.

During sleep and HOME Button transitions, the internal states for aacdec and aacenc get initialized. You will need to use nn::aacdec::CTR::SetStateBuffer or nn::aacenc::CTR::SetStateBuffer to save the internal state.

This library does not support the simultaneous decoding of multiple bitstreams.

Constants

S STATE_BUFFER_ALIGNMENT const s32 The address of the buffer specified in SetStateBuffer must be aligned to the number of bytes specified by this function. It is defined as 32 bytes.
S DECODE_BUFFER_ALIGNMENT const s32 The address of the buffer specified in Decode must be aligned to the number of bytes specified by this function. It is defined as 32 bytes.

Functions

Initialization and Finalization
Initialize Initializes the decoder.
Finalize Finalizes the decoder.
SetStateBuffer Configures a buffer for saving the internal state of the decoder when the system enters sleep mode or transitions to the HOME menu.
Decoding Processing
GetAdtsFrameSize Parses the ADTS header and returns a size.
Decode Decodes the frame.
Wait Waits for the decoding process to finish.
TryWait Waits for the decoding process to finish (also takes a fixed timeout value).
Getting Information
GetChannelConfig Returns the channel configuration information for the decoded frame.
GetSampleRateIndex Returns the sampling frequency index for the decoded frame.
GetSampleRate Returns the sampling frequency for the decoded frame.
GetUsedBytes Returns the number of bytes in the decoded frame.
GetErrorCode Returns an error code.
