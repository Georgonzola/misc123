nn::aacenc::CTR::Encode Function

Syntax

bool Encode(
     const s16 * pOutputL,
     const s16 * pOutputR,
     u8 * pFrame
);

Parameters

Name Description
in pOutputL A pointer to the (left) output PCM buffer.
in pOutputR A pointer to the (right) output PCM buffer.
out pFrame A pointer to the bitstream.

Return Values

Returns true if successful; otherwise, returns false.

Description

Encodes the frame.

The buffers pointed to by pOutputL, pOutputR and pFrame must all be located in device memory. The addresses must also be aligned to ENCODE_BUFFER_ALIGNMENT (32 bytes).

1024 samples (2048 bytes) of PCM data are required per channel for input on one call (one frame). For pFrame, prepare a buffer that will have a maximum size of 1536 bytes as a result of encoding.

Encoding has a codec delay. For more information, see Handling Codec Delays in nn::aacenc::CTR.
