nn::aacenc::CTR Namespace

Description

Namespace for the AAC encoder (AACENC) library. (It can also be written without the CTR portion.)

Introduction

The AAC Encoder library provides the application with functionality for encoding AAC data.
AAC is an audio standard widely used for such applications as music streaming and mobile phones.

Using the Library

Three files are required to use the AAC Encoder library.

  • nn/aacenc.h
  • libnn_aacenc.a
  • resources/dspcomponents/dspaudio.cdc

When using this library, execute the initialization function before executing the encode function.
The encoding process uses DSP. Accordingly, you must initialize the DSP library and load the DSP component beforehand.
1024 samples per channel are required for the PCM data passed to the encode function.

Handling Codec Delays

This library has a codec delay of 1600 samples per channel. The following steps are therefore required to get all the PCM data encoded as AAC data.

  1. If the PCM data input in the final frame has fewer than 1024 samples per channel, the data will be padded with zeroes to reach 1024 samples before calling the encode function.
  2. After encoding of the final frame ends, the process calls the encode function with 1024 zeroed-out samples as input.
  3. It then calls the encode function again with 1024 zeroed-out samples as input.

Implementation Notes

You must call the nn::dsp::CTR::LoadComponent function instead of the nn::dsp::CTR::LoadDefaultComponent function, and load the special DSP component file.
To use non-standard DSP components, the DSP component file must be in main memory (it does not have to be device memory) until the nn::dsp::CTR::UnloadComponent function is called.

This library does not support the simultaneous encoding of multiple sound sources.
