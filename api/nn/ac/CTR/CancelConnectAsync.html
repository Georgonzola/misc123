nn::ac::CTR::CancelConnectAsync Function

Syntax

Result CancelConnectAsync();

Parameters

None.

Return Values

Returns the result.

Value Description
Result::IsSuccess Indicates that the call completed successfully.
ResultNotConnecting A connection is not being processed.
ResultAlreadyConnecting A connection has already been processed and established.
Other Indicates failure for other reasons.

Description

Cancels processing if an automatic connection process is underway.

When a process is canceled, this is indicated in the Result return value.
The GetLastErrorCode function can get error codes that are equivalent to the error codes for failures that occur when processes are canceled.
Use the Result value to determine whether the process was successfully canceled.

When canceling succeeds, or when the connection completes before the cancel request is accepted, the nn::os::Event passed by ConnectAsync is signaled, and you can verify the cancellation with GetConnectResult.
If another thread calls this function while the Connect function is running, you can determine whether the process was canceled based on the Connect return value.

This function works differently in SDK 1.x than in SDK 2.x and later.
For SDK 1.x, if this function is called when processing a connection request to AC via background communications, ResultNotConnecting is returned.
For SDK 2.x and later, the device cancels its connection request and the value Result::IsSuccess is returned.
