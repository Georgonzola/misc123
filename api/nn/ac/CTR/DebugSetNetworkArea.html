nn::ac::CTR::DebugSetNetworkArea Function

Syntax

Result DebugSetNetworkArea(
     Config * config,
     NetworkArea networkArea
);

Parameters

Name Description
in config Instance of connection conditions.
in networkArea Specifies the network connection request scope.

Return Values

Returns the result.

Value Description
Result::IsSuccess Indicates that the call completed successfully.
ResultInvalidData NULL was specified as an argument.
ResultDebugOnly Indicates that debug mode was disabled.
Other Indicates failure for other reasons.

Description

Specifies the scope of the network connection request.

When NETWORK_AREA_WAN is specified, the nn::ac::CTR::Connect function will succeed with a [WAN Connected] status only when the access point you are connected to is connected to the Internet.
When NETWORK_AREA_LAN is specified, the function succeeds with a [LAN Connected] status upon connection to the access point, without authenticating a connection to the Internet.
When NETWORK_AREA_AUTO is specified, the function first verifies the connection to the Internet. The function succeeds with a [WAN Connected] status upon connection to the Internet. However, it succeeds with a [LAN Connected] status upon connection to the access point even if connection to the Internet was not possible.



This function is for debugging. It only works in debug mode.
Consequently, the only connection request that can be created for the retail product is that which specifies the default setting, NETWORK_AREA_WAN.
During the development phase, when an Internet connection is not possible, create connection requests specifying either NETWORK_AREA_LAN or NETWORK_AREA_AUTO.

For access points that cannot connect to the Internet, it may take some time for the nn::ac::CTR::Connect function to complete, as it waits for the system to time out on its connection confirmation for the Internet in the background, even when NETWORK_AREA_LAN has been specified.
