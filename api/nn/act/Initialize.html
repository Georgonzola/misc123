nn::act::Initialize Function

Syntax

Result Initialize(
     uptr bufferAddress = 0,
     size_t bufferSize = 0
);

Parameters

Name Description
in bufferAddress Pointer to the start of the communication buffer. Memory for the buffer must be aligned to BUFFER_ALIGNMENT (4096 bytes).
in bufferSize The size of the communication buffer This value must be a multiple of BUFFER_UNITSIZE (4096 bytes).

Return Values

Returns the result of the operation.


Value Description
ResultSuccess Succeeded.
Other Other results can be returned depending on the internal state of the daemon.

Description

Initializes the account library and makes the account features available for use. When using APIs that trigger communication with some servers, you must set a data buffer for communication as an argument. Because the size of the required buffer varies by API, see the reference for the API you are using, and then specify the largest size from there. When you are not using these APIs, you do not need to set anything for these arguments. For example, if you are just using the feature to get a service token, you do not need to specify a communication buffer. This function can be called multiple times even if the account library is already initialized, but the nn::act::Finalize function must be called an equal number of times to completely finalize the account library. This function can only be specified when the communication buffer has not yet been initialized, however. If this function is called when the communication buffer is already initialized, the buffer specified in the argument is not used.
