nn::act::applet::AcquireIndependentServiceTokenV2 Function

Syntax

Result AcquireIndependentServiceTokenV2(
     ServiceTokenV2 * pServiceToken,
     const char * pClientId,
     u32 reusableRangeInSeconds = 0
);

Parameters

Name Description
out pServiceToken Specifies an out pointer to the buffer that stores the service token.
in pClientId Specifies a pointer to the buffer that stores the client ID for the independent service.
in reusableRangeInSeconds Specifies the amount of seconds in the past a cache can have been obtained for a service token and still be valid.

Return Values

Returns the result of the operation.


Value Description
ResultSuccess Succeeded.
ResultNotNetworkAccount Indicates that the account is not a network account.
ResultInvalidPointer Indicates that an invalid pointer (such as a NULL pointer) was passed as an argument.
ResultInvalidClientId Indicates that the client ID is not registered on the server.
ResultNotConnected Not connected to an access point.
ResultAuthenticationError Indicates that authentication failed.
ResultApplicationUpdateRequired An application update is required.
ResultCanceled Canceled.

Description

Gets a service token (version 2) for an independent service.

To get a service token with this function, you must have registered your service title information on OMAS in advance to receive a client ID, and then set a unique ID in your application's RSF file, as indicated in the OMAS application.

This function's security has been improved over that of the prior token function, nn::act::applet::AcquireIndependentServiceToken. Unless there are compatibility issues for past titles, use this function.
For more information about the use of service tokens, see the Account System Developer's Guide.
No parental control checks are performed by this function. Configure the application as needed to implement functionality corresponding to Parental Controls settings.

Note that the lifetime specified by reusableRangeInSeconds does not define the cache expiration period for the service token obtained with this function. Instead, it specifies how long a previously retrieved cache is to remain valid. Service tokens obtained from the account server must be stored in a cache within the system. If the cache includes a service token that was obtained within reusableRangeInSeconds seconds, the function returns that service token and does not communicate with the account server. To get a service token again because it has expired or was rejected by the independent server for another reason, specify 0 for reusableRangeInSeconds to make sure that you get a new token from the server.

This function starts the applet, and blocks for the duration.

If the function fails, the value set in the buffer is undefined.

For information about using this function, see the section about making API functions that trigger communication in the nn::act namespace.
