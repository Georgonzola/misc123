nn::act::applet Namespace

Description

Library namespace for using the login applet.

You must call the nn::act::Initialize function in advance and initialize the library to use any of the functions within the nn::act::applet namespace. In addition, when using the function for getting a token, the application must explicitly connect to a wireless access point.

For information about using this function, see the section in the nn::act namespace that describes how to make API functions that trigger communication.

If you run the nn::act::applet::AcquireIndependentServiceTokenV2 or nn::act::applet::AcquireOlvServiceToken function, the login applet runs, and processes such as password input and token acquisition that occur in conjunction with account authentication are handled by the function. The login applet continues to run until the series of account authentication and token acquisition processes end.

When the token acquisition function fails, the return value's result object is passed to the nn::act::GetErrorCode function to get the network error code. In addition to displaying the network error code in the error display applet, terminate the application's token acquisition sequence.
A nn::act::ResultCanceled object, however, indicates that the user explicitly canceled the token acquisition sequence. If this object is returned, terminate the application's token acquisition sequence without displaying the error in the error display applet. The application has no need to individually handle errors other than nn::act::ResultCanceled, but the application can optionally handle the nn::act::ResultApplicationUpdateRequired result to proceed to the sequence that performs a check for the download of patches.

Login Applet
AcquireEcServiceToken Gets a service token for an e-commerce service. Currently, this function can be used only to implement features for purchasing service items through an independent server.
No parental control checks are performed by this function. Configure the application as needed to implement functionality corresponding to Parental Controls settings.

For pEcServiceToken, pass a pointer to a buffer at least as big as NN_ACT_INDEPENDENT_SERVICE_TOKEN_SIZE bytes.
