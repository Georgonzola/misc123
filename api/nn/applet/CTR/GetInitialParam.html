nn::applet::CTR::GetInitialParam Function

Syntax

bool GetInitialParam(
     AppletId * pSenderId,
     u8 * pParam,
     size_t paramSize,
     s32 * pReadSize
);

Parameters

Name Description
out pSenderId ID of the applet that sent the initial parameters.
out pParam The buffer that gets the initial parameters.
in paramSize Buffer size (in bytes).
out pReadSize Amount of data actually read in (in bytes).

Return Values

Returns true if the parameter exists, or false if it does not. Even when true is returned, if 0 is stored in pReadSize, there are no valid initial parameters.

Description

Gets the initial parameter. When an application is started, it can be passed up to NN_APPLET_PARAM_BUF_SIZE bytes of data (the initial parameters). This function enables the application to receive those parameters. Normally, however, no initial parameters are set on the HOME Menu. Apart from setting initial parameters when starting applications from the HOME Menu, the initial parameters may also be set by the originating application during jumps between applications or by system applets when they start applications. The content of the parameters depends on the conventions used between applications or system applets. The library is not involved in handling the content.
