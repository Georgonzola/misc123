nn::applet::CTR Namespace

Description

The namespace of the CTRapplet library. (It can also be written without the CTR portion.)

Classes

nn::applet::CTR::ResultNotFound Indicates an attempt to get something that did not exist.
nn::applet::CTR::ResultAlreadyExist Indicates an attempt to create or register something that already exists.
nn::applet::CTR::ResultNoArea Indicates a lack of free space during an attempt to run a process that required a region or resource.
nn::applet::CTR::ResultNotEmpty Indicates that during an attempt to run a process, a required buffer was in use and not empty.
nn::applet::CTR::ResultNoData Indicates that there was no data.
nn::applet::CTR::ResultNoCaller Indicates that the caller or sender does not exist.
nn::applet::CTR::ResultNotAllowed Indicates that processing was attempted from a state in which it was not allowed.
nn::applet::CTR::ResultBusy Indicates an attempt to run a process that was busy because of a mutex or for another reason. In this case, you can retry the process.
nn::applet::CTR::ResultAlreadyInit Indicates that initialization has already been done.
nn::applet::CTR::ResultNotInitialized Not initialized.
nn::applet::CTR::ResultDifferentMode Indicates differing modes.
nn::applet::CTR::ResultDifferentVersion Indicates differing versions.
nn::applet::CTR::ResultTransitionBusy Indicates that another transition is in progress.
nn::applet::CTR::ResultShutdownProcessing Indicates that a shutdown is in process.
nn::applet::CTR::ResultVersionMustLaunchDirectly Indicates that the version requires direct startup without using menus.

Enumerated Types

HomeButtonState Enumerated type indicating the state of the HOME Button. You can get this state with the GetHomeButtonState() function. The state switches to HOME_BUTTON_NONE when you call the ClearHomeButtonState() function.
WakeupState Enumerated type indicating wakeup requests. This is used as the return value from the WaitForStarting() function.
QueryReply Enumerated type indicating a response to a sleep query. It is used as the return value from the sleep query callback and as an argument to the ReplySleepQuery() function. Specify either REPLY_REJECT, REPLY_ACCEPT, or REPLY_LATER as the return value from the sleep query callback. Specify either REPLY_REJECT or REPLY_ACCEPT as an argument to the ReplySleepQuery() function.
ScreenCapturePostPermission The post permission for the screenshots the system gets at HOME Menu transitions.
ResultDescription Module-dependent value for the Description in nn::Result.

typedef Definitions

AppletId Applet ID.
AppletAttr The applet attributes.
AppletHomeButtonCallback The type of the callback function invoked when HOME is pressed.
AppletSleepQueryCallback The type of the callback function invoked by a sleep query.
AppletSleepCanceledCallback Function type representing a sleep cancel callback.
AppletAwakeCallback The type of the callback function invoked upon awaking from sleep states.
AppletPowerButtonCallback The type of the callback function invoked when the POWER Button is pressed.
AppletCloseCallback This function represents the callback invoked when the system enters a state where the application normally closes, such as the POWER Button being detected or a drop in power.

Constants

WAIT_INFINITE const nn::fnd::TimeSpan The function continues running until a result is obtained. (Used by applet library functions that specify a time.)
NO_WAIT const nn::fnd::TimeSpan The function only runs once and does not repeat even if no results are obtained. (Used by applet library functions that specify a time.)

Initialization
Enable Enables applet library features.
Getting Information
GetInitialWakeupState Gets wakeup information when an application is started.
GetInitialParam Gets the initial parameter. When an application is started, it can be passed up to NN_APPLET_PARAM_BUF_SIZE bytes of data (the initial parameters). This function enables the application to receive those parameters. Normally, however, no initial parameters are set on the HOME Menu. Apart from setting initial parameters when starting applications from the HOME Menu, the initial parameters may also be set by the originating application during jumps between applications or by system applets when they start applications. The content of the parameters depends on the conventions used between applications or system applets. The library is not involved in handling the content.
IsActive Determines whether an applet is currently running.

Nintendo eShop
JumpToEShopTitlePage Jumps to the software information page in Nintendo eShop. The titles displayed in Nintendo eShop are specified by their unique IDs. This function can only jump to the pages of titles that are publicly searchable. Publicly searchable titles are those titles that can be searched for by name and accessed on Nintendo eShop. The application ends when this function is called. If there is no information about the title on Nintendo eShop or if the value is invalid, an error displays on Nintendo eShop. You must use the nn::applet::IsEShopAvailable function to determine in advance whether Nintendo eShop has been installed. Always call this API after confirming that Nintendo eShop is installed. When Nintendo eShop is not present on the system, we recommend that you let the user know that a system update via the Internet is required. For example, you could display a message that says something like: "Cannot use Nintendo eShop. An Internet update of your system is needed. Proceed?" In the development environment, you can test the jump to Nintendo eShop feature by importing DummyNintendoEShop. For more information, see the detailed description of the DummyNintendoEShop tool. When using this function to make jumps, make an OMAS application for the destination title. For information about how to apply, see 3DS Guidelines: E-Commerce.
JumpToEShopPatchPage Jumps to the Patch page for the specified title in Nintendo eShop. If an error is returned from the authentication or account server that indicates the need for an application update, such as nn::act::ResultApplicationUpdateRequired, use that error message to lead users to the Patch page for your title on Nintendo eShop. The titles displayed in Nintendo eShop are specified by their unique IDs. The application ends when this function is called. When you have remastered your application instead of using a patch, a Patch page for the title does not exist on Nintendo eShop. In that case, an error message is displayed that indicates that the title does not exist, and the Nintendo eShop start screen appears to the user. When working with a remaster, use the nn::applet::JumpToEShopTitlePage function. You must use the nn::applet::IsEShopAvailable function to determine in advance whether Nintendo eShop has been installed. Always call this API after confirming that Nintendo eShop is installed. When Nintendo eShop is not present on the system, we recommend that you let the user know that a system update via the Internet is required. For example, you could display a message that says something like: "Cannot use Nintendo eShop. An Internet update of your system is needed. Proceed?" In the development environment, you can test the jump to Nintendo eShop feature by importing DummyNintendoEShop. For more information, see the detailed description of the DummyNintendoEShop tool. When using this function to make jumps, make an OMAS application for the destination title. For information about how to apply, see 3DS Guidelines: E-Commerce.
IsEShopAvailable Determines whether Nintendo eShop is installed. In some cases, Nintendo eShop may not be installed on the system. This is the case if the system was shipped shortly after release and has never had a system update. This function can be used to check whether it is installed. Nintendo eShop is not installed on the system in the development environment. You can get this function to return true by importing the dummy Nintendo eShop included with the SDK. The dummy Nintendo eShop can be removed from DevMenu after it is imported.

Call E-manual
JumpToManual Jumps to the e-manual. Jumping to the e-manual is similar to the HOME Menu transition process. Consequently, the application must wait for calls to this function in the same way it waits for calls to nn::applet::CTR::ProcessHomeButton() when the HOME Button is pressed. See the function reference for nn::applet::CTR::ProcessHomeButton(). Depending on the system mode setting, this function might not work correctly. For more information, see the documentation under CTR Tools > Configuration.
