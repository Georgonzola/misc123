nn::boss::GetStorageInfo Function

Syntax

nn::Result GetStorageInfo(
     size_t * pStorageSize,
     u16 * pEntryCount
);

Parameters

Name Description
out pStorageSize Returns the size of the BOSS storage region.
out pEntryCount Returns the maximum file count for BOSS storage.

Return Values

Returns the execution result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates successful retrieval.
ResultStorageNotFound The BOSS storage corresponding to the application is not registered. Determine whether the BOSS storage is registered.
ResultIpcNotSessionInitialized Indicates that the session is not initialized.
The function returns this Result if it is called before Initialize is called.
Always call the Initialize function before you use the BOSS library.
Other than those noted above An unexpected error. (For information about handling this error, see the Programming Manual.)

Description

Gets the size of the registered BOSS storage region and the maximum file count.
This function can also be used to determine whether BOSS storage (the save region on the game device) was registered to the system using RegisterStorage.
This function cannot be used to check for BOSS storage in the extended save data region. BOSS storage is automatically generated by the system as part of the extended save data. (For information about checking for extended save data, see nn::fs::MountExtSaveData.)
