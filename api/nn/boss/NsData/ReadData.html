nn::boss::NsData::ReadData Member Function

Syntax

s32 ReadData(
     u8 * pDataBuf,
     size_t bufLen
);

Parameters

Name Description
out pDataBuf Specifies the NS data buffer.
in bufLen Specifies the NS data buffer length.

Return Values

Returns the number of bytes read. Returns 0 if it reached the end of the data.

Value Description
NN_BOSS_NSDATA_READ_ERROR_GET_HEADER Failed to get the NS data file header (information of the length, nn::boss::NSD_LENGTH). For a detailed error, get GetHeaderInfo.
NN_BOSS_NSDATA_READ_ERROR_READ_DATA Failed to load NS data. Perform error handling as if the NS data is corrupted (damaged, or spoofed). For more on error handling methods, see the programming manual.
NN_BOSS_NSDATA_READ_ERROR_IPC Indicates that the session is not initialized. If this function is called before nn::boss::Initialize is called, this Result is returned. Always call the nn::boss::Initialize function before you use the BOSS library.
NN_BOSS_NSDATA_READ_ERROR_UPDATED The target NS data version was updated since the last time data was obtained.

Description

Reads the NSD data block.

After preparing an adequately sized buffer in advance, call this function multiple times to read an entire NS data file.
Returns 0 upon reaching end of file, indicating that the read is finished.
Applications only can read one NS data file at a time. Even when a large NSD is read, you are not blocked from reading other applications' NS data files, as long as the same NS data files are not being accessed.
Note that there will be times when an error will be returned when the desired NS data cannot be found, because it has been deleted by another application, because there was insufficient memory during an NSA download, or because it was automatically deleted.
The function NN_BOSS_NSDATA_READ_ERROR_UPDATED in those rare instances where the specified NS data version has been updated since the last time the data was obtained.
In such cases, call the nn::boss::NsData::Initialize function and then read the data again.
