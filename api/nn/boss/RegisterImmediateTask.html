RegisterImmediateTask ( Task *, TaskAction *, TaskPolicy *, TaskOption *, u8 ) Registers an immediate-execution task to BOSS (to the save region on the game device). To register a dedicated immediate-execution task, you must first prepare an instance of a subclass of the TaskAction class.
For information about which attributes must be set, see the API Reference topic for each class.
When you have finished registering a task, you can reuse each of these instances as necessary by calling the Initialize() function.
You can issue a task execution instruction by calling the nn::boss::FgOnlyTask::StartImmediate() function for a registered task. 　 Common Task Registration Errors 1. BOSS storage has not been registered when registering an NADL task (ResultStorageNotFound). Use RegisterStorage to register BOSS storage for the NADL task.
RegisterImmediateTask ( Task *, TaskAction *, const char *, TaskPolicy *, TaskOption *, u8 ) Registers an immediate-execution task to BOSS (to the save region on the game device) in concert with the file specified as the target. At the present time, this function can be used only to register DataStore upload tasks. For more information, see nn::boss::RegisterImmediateTask, which has the same specification but without the argument to take the file path.

The file corresponding to the specified file path is copied to BOSS storage during the task registration process. Copying can take a long time (upwards of 10 seconds), depending on file size and SD card type. As a result, it can take a long time for this function to respond.
