nn::boss::RegisterPrivateRootCa Function

Syntax

nn::Result RegisterPrivateRootCa(
     const u8 * pCertData,
     size_t certDataSize
);

Parameters

Name Description
in pCertData Specifies the root CA certificate data. The data format conforms to the ASN.1 standard.
in certDataSize Specifies the size of the root CA certificate data.

Return Values

Returns the execution result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates that registration succeeded.
ResultIpcNotSessionInitialized Indicates that the session is not initialized.
The function returns this Result if it is called before Initialize is called.
Always call the Initialize function before you use the BOSS library.
Other than those noted above An unexpected error. (For information about handling this error, see the Programming Manual.)

Description

Registers a root CA certificate as an application certificate in the BOSS library. Certificates are used when using HTTPS communication as a connection when executing a task.
There is no need to call this function if a root CA certificate is not used as an application certificate. Call nn::boss::TaskAction::SetPrivateRootCa to use a registered independent root CA certificate. An application certificate and the built-in device certificate can be used together.
Only one root CA certificate can be set as an application certificate.
If settings are duplicated, the last set root CA certificate is valid. Note: Currently, registering tasks that connect to servers other than the BOSS data server provided by Nintendo is not permitted.
