nn::boss::RegisterStorage Function

Syntax

nn::Result RegisterStorage(
     bit32 storageId,
     size_t storageSize,
     nn::boss::StorageType storageType = nn::boss::USER_EXT_SAVEDATA_STORAGE
);

Parameters

Name Description
in storageId Specifies the ID of an expanded save data region that creates BOSS storage.
in storageSize Specifies the maximum total size of files that can be held in BOSS storage.
in storageType Specifies the BOSS storage type.
(Omit this parameter because usually the default BOSS storage for the user's expanded save data region is used.)

Return Values

Returns the execution result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates that registration succeeded.
ResultStorageAccessPermission The program does not have access privileges for the expanded save data region. Reconfirm your access rights.
ResultIpcNotSessionInitialized Indicates that the session is not initialized.
The function returns this Result if it is called before Initialize is called.
Always call the Initialize function before you use the BOSS library.
Other than those noted above An unexpected error. (For information about handling this error, see the Programming Manual.)

Description

Registers BOSS storage information to BOSS (to the save region on the game device). BOSS storage must be registered before registering or executing NADL tasks.
NS data cannot be downloaded if the storage is not registered. Before registering BOSS storage, you must first create an extended save data region in memory.
You must also create an expanded save data region, while specifying more than the total number of NS data files to store.
This function returns an error if you specify an expanded save data region ID in the storageID argument to which the application does not have access rights. During downloading of NS data files, when the total size of the NS data files in the BOSS storage exceeds the maximum data size, NS data files with the oldest serial ID (that is, the smallest serial ID) are automatically deleted. There is a limit on the number of BOSS storage information registrations. Information can be automatically unregistered by BOSS. For more information, see the Programming Manual. 1) To get NS data without the application registering a task:
After BOSS storage is registered, if NS data files are included for an application for which BOSS storage is registered within a downloaded NS archive, by a task registered by a different application, those NS data files are automatically downloaded to the registered BOSS storage. 2) To download NS data addressed to the application that registers a task:
If the application registers BOSS storage and then registers and runs a task, it can download NS data addressed to the application and store it in the BOSS storage specified when it registered the task.
As with step 1, if NS data files are included for an application for which BOSS storage is registered within a downloaded NS archive, by a task registered by a different application, those NS data files are automatically downloaded to the registered BOSS storage. 3) To share NS data with another application: You can get NS data downloaded to another application by registering BOSS storage ahead of time in the same extended save data region used for another application.
An application can even get this NS data from other applications when it downloads its own NS data.
For more information about these operations, see the GetNsDataIdList and GetNewDataNsDataIdList functions.
