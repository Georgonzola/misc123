RegisterTask ( Task *, TaskPolicy *, TaskAction *, TaskOption *, u8 ) Registers a task to BOSS (to the save region on the game device). To register tasks, you must prepare instances of the subclass of the TaskPolicy class and the TaskAction class. (NsaDownloadAction class, DataStoreDownloadAction class, and DataStoreUploadAction class).
For information about which attributes must be set, see the API Reference topic for each class.
When you have finished registering a task, you can reuse each of these instances as necessary by calling the Initialize() function.
You can issue a task execution instruction by calling the nn::boss::Task::Start() function for a registered task. 　 Common Task Registration Errors 1. BOSS storage has not been registered when registering an NADL task (ResultStorageNotFound). Use RegisterStorage to register BOSS storage for the NADL task. 2. A task with the same task ID has already been registered (ResultTaskIdAlreadyExist). Either use UnregisterTask to unregister the task, or reuse the task that has already been registered.
RegisterTask ( Task *, TaskPolicy *, TaskAction *, const char *, TaskOption *, u8 ) Registers a task to BOSS (to the save region on the game device) in concert with the file specified as the target. At the present time, this function can be used only to register DataStore upload tasks. For more information, see nn::boss::RegisterTask, which has the same specification but without the argument to take the file path.

The file corresponding to the specified file path is copied to BOSS storage during the task registration process. Copying can take a long time (upwards of 10 seconds), depending on file size and SD card type. As a result, it can take a long time for this function to respond.
