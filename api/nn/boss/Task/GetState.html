nn::boss::Task::GetState Member Function

Syntax

TaskStateCode GetState(
     bool acknowledge = false,
     u32 * pCount = NULL,
     u8 * pStepId = NULL
);

Parameters

Name Description
in acknowledge After a task finishes running, this stores the "task is complete" state so that the task's execution results can be checked. Calling the function with this flag set to true clears the "complete" state. For example, with a task whose execution count is not zero, the subsequent return value becomes TASK_WAITING_TIMER because the task is rescheduled immediately after the prior task execution completion. (Optional.)
out pCount Returns the execution count. (Optional.)
out pStepId Returns the task step ID. (Optional.)

Return Values

Returns a code indicating the task state.

Description

Gets the state of a task.

In most cases, task states transition from state 1 through state 5 below.
(1) TASK_REGISTERED (The task is registered.)
(2) TASK_WAITING_TIMER (Waiting for the time after the Start function is called.
(3) TASK_WAITING (Waiting for execution.)
(4) TASK_RUNNING or TASK_RETRY (The task is running.)
(5) TASK_DONE or TASK_ERROR (The task is complete.)
There are also two special states, TASK_PAUSED (paused) and TASK_STOPPED (stopped by a call to Cancel).

Note: Because state transitions sometimes occur in a short period of time, the function may not always be able to get all states.
