nn::boss::Task::GetStateDetail Member Function

Syntax

nn::Result GetStateDetail(
     TaskStatus * pStatus,
     bool acknowledge = false,
     u8 * pStepId = NULL,
     u8 taskStep = CURRENT_STEP_ID
);

Parameters

Name Description
out pStatus Returns the task status.
in acknowledge After task execution is complete, TaskStateCode, the code indicating the task execution complete state within TaskStatus, is maintained such that the execution result can be confirmed. When called with this flag set to true, the execution completion state is no longer maintained.
out pStepId Returns the task step ID.
in taskStep Specifies a task step ID. (Defaults to the current step if omitted.)

Return Values

Returns the execution result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess The reset state was obtained successfully.
ResultInvalidTaskStatus The task status pointer is NULL.
ResultInvalidTaskId Indicates that the task ID pointer is NULL or a zero-length string.
ResultTaskNotExist The specified task ID was not found. Confirm whether it is registered yet.
ResultIpcNotSessionInitialized Indicates that the session is not initialized. The function returns this Result if it is called before Initialize is called. Always call the Initialize function before you use the BOSS library.
Other than those noted above An unexpected error. (For information about handling this error, see the Programming Manual.)

Description

Gets the task status.

You must prepare a TaskStatus instance beforehand. Note that this instance requires approximately 128 bytes of memory. For more information about obtaining the values of the attributes, see the GetProperty member function of the TaskStatus class.
