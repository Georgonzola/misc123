nn::boss::Task::Start Member Function

Syntax

nn::Result Start(
     void
);

Parameters

None.

Return Values

Returns the result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Successfully issued the instruction to start the task.
ResultInvalidTaskId Indicates that the task ID pointer is NULL or a zero-length string.
ResultTaskNotExist The specified task ID was not found. Confirm whether it is registered yet.
ResultIpcNotSessionInitialized Indicates that the session is not initialized. The function returns this Result if it is called before Initialize is called. Always call the Initialize function before you use the BOSS library.
Other than those noted above An unexpected error. (For information about handling this error, see the Programming Manual.)

Description

Gives an instruction to start the task.

The scheduled tasks automatically run in the background when infrastructure connections are enabled.
Except for the first time a task is run, this function works the same as StartBgImmediate. With this function, the first time that a task is run, it does not start to run until after the task execution interval of time from the call to when the function has passed.
In contrast, with the StartBgImmediate function, as soon as the function executes, it is time for the first task to run. The first task runs immediately upon infrastructure connection after the function executes.

When a task finishes running, the execution count specified in the task's attributes is decremented by 1 regardless of whether task execution resulted in an error.
Unless the remaining execution count is 0, the task runs again after the execution interval specified in the task's attributes has elapsed.

Note: After its execution count reaches 0, a task is sometimes deleted automatically if the internal database is filled to capacity.
In the following circumstances, tasks are not executed even if a command is issued to start a task.
The task's execution count is 0.The execution priority of the task is PRIORITY_STOPPED.Parental Controls are enabled, and the attributes of the task do not override Parental Controls.The user has not accepted the EULA, and the attributes of the task do not override EULA acceptance.A policy list blocks task execution. The server hosting the download service issued a "service suspended" notice. The administrator has temporarily stopped all tasks. The GetCount function can be used to confirm whether the task execution count is the relevant case among those listed above.
You can tell when the administrator has temporarily stopped all tasks because TASK_PAUSED is returned by the GetState function for all running tasks.

The following methods can be used to wait for task completion.
Use the WaitFinish function to wait until execution completes. Use the WaitFinish function to wait until execution completes, specifying a maximum wait time. Use the GetState function to poll for task completion. Wait for a "new arrival" event using an Event registered by the RegisterNewArrivalEvent function.
Upon detection of completion, you can check the task state with GetState, the task execution result code with GetResult, and the HTTP communication result code with GetHttpStatusCode.
You can also get other details by using the GetStateDetail function.
