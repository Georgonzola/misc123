nn::boss::UnregisterTask Function

Syntax

nn::Result UnregisterTask(
     Task * pTask,
     u8 taskStep = DEFAULT_STEP_ID
);

Parameters

Name Description
in pTask Specifies a registered task.
in taskStep Specifies a task step ID. (This parameter may be omitted if there is only one task step to be executed.)
Note: This parameter is not currently supported because it is used with extended features. Do not specify it.

Return Values

Returns the execution result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Deregistration was successful.
ResultInvalidTaskId The pTask parameter is NULL.
Alternatively, the pTask parameter is uninitialized, or the pointer of the task ID specified by pTask is NULL or a zero-length string.
ResultTaskNotExist The specified task ID was not found. Confirm whether it is registered yet.
ResultAppNotExist There is no information about the calling application. Returns this error if the application has not run any tasks on BOSS or registered any BOSS storage.
ResultFileAccess Indicates that deletion of a work file for the target task in BOSS storage has failed. (This may occur when the SD card is removed.) The task itself can be unregistered.
ResultIpcNotSessionInitialized Indicates that the session is not initialized.
The function returns this Result if it is called before Initialize is called.
Always call the Initialize function before you use the BOSS library.
Other than those noted above An unexpected error. (For information about handling this error, see the Programming Manual.)

Description

Unregisters a registered task. Although you can unregister a task that is currently running or is waiting to run, you cannot check that task's execution results.
If you want to check the task's execution results, check after the task has finished running, call nn::boss::Task::Cancel() to prevent the next task from running, and then unregister the task. When unregistering a task, work files for the target task in BOSS storage are also deleted. In cases where an SD card that has BOSS storage is removed, deletion of work files fails and the error ResultFileAccess returns. However, in this case the task itself is unregistered. Error handling is not required because work files may also be deleted due to the deletion of extended save data that uses BOSS storage. (You can expect the user to take care of it when the application is not being used.) (If this error returns, you cannot directly delete any work files that could not be deleted. However, if a task having the same task ID has been registered, work files that could not be deleted are used in the task. After registering a task that has the same task ID, you delete work files by executing UnregisterTask for that task.）
