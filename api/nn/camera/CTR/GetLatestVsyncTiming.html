nn::camera::CTR::GetLatestVsyncTiming Function

Syntax

void GetLatestVsyncTiming(
     nn::fnd::TimeSpan * pTime,
     Port port,
     size_t past = 1
);

Parameters

Name Description
out pTime Specifies the location where the timing is stored.
in port Specifies the port to which the camera you are getting the timing of is connected.
in past Specifies the number of past timing values to get. Specify this parameter as an integer between 1 and 5. If omitted, it defaults to 1. The pTime argument stores the timing values for recent past V-Sync events, starting with the most recent one. The pTime argument must be allocated with enough space to store past V-Sync timings. If the number of V-Sync signals input since the camera was activated (called "frame" here for the sake of convenience) is smaller than the value of past, indeterminate values are stored for the values between pTime[frame] and pTime[past-1].

Return Values

None.

Description

Gets the timing history of camera V-Sync signal inputs.

The camera interrupt handler gets the timing history by obtaining the tick value.
