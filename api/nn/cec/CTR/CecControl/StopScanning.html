nn::cec::CTR::CecControl::StopScanning Member Function

Syntax

static nn::Result StopScanning(
     bool b_Immediate = false,
     bool b_Async = false
);

Parameters

Name Description
in b_Immediate When true, saving of data being transmitted is also canceled and stopped.
in b_Async When true, returns immediately.

Return Values

nn::Result

Value Description
Result::IsSuccess Indicates success.
ResultStateBusy The halt process occurred but did not enter the halted state.
Other Indicates failure.

Description

Manually halts StreetPass (CEC) operations.

If communications are in progress, the communications are interrupted and changed to stopped (IDLE) status.
Note that background communication might be interrupted.
If this function is called when the state that GetCecState can get is DAEMON_STATE_COMMUNICATING or DAEMON_STATE_BUSY, the communication or save process may be interrupted.

To access a MessageBox, communications must be in the stopped state.
Because StopScanning(b_Immediate = true) is called within MessageBox::OpenMessageBox, there is no need to consciously stop communications. However, you can wait on a halted state without interrupting a data save by using StopScanning(b_Immediate = false) in advance.



When b_Async is set to true, it might not enter IDLE status after exiting this function, such as when this function is called multiple times and cancels are issued.
Note that even if MessageBox::WriteMessage or similar functions are called after this function is called, an error might be generated because it is in the BUSY state.
