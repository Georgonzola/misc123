nn::cec::CTR::GetCecInfoBuffer Function

Syntax

nn::Result GetCecInfoBuffer(
     u32 cecTitleId,
     u8 pCecInfoBuffer[],
     size_t size
);

Parameters

Name Description
in cecTitleId Specifies the StreetPass ID.
out pCecInfoBuffer[] Specifies the buffer to receive the CecNotificationData.
in size Specifies the buffer size.

Return Values

nn::Result

Value Description
Result::IsSuccess Indicates success.
ResultNoData No data could be obtained.

Description

Gets information about a Message received in the immediately previous StreetPass communication.

GetCecEvent can be used to determine whether, for example, there is data for a foreground application after receiving notification of the start of communication.

The "most recent StreetPass communication" refers to the most recent StreetPass communication with another system.
Note that this is not the last communication with the specified application.

For example, say StreetPass communication occurs twice, with data exchanges for each application occurring as shown below. The information this function can receive is limited to that for the second communication, for Application A and Application D.
The other application did not receive any data in the last communication and so no information can be obtained for it.

First Communication: Application A (receives data), Application B (receives data), Application C (receives data) Second Communication: Application A (receives data), Application B (sends data), Application D (receives data)

When a specified application receives a message from an immediately prior communication, the value for CecNotificationData.num is 1. When nothing is received, the value is 0.
This function can only get information about a single Message, even when multiple messages are received as a group. In other words, the maximum value for CecNotificationData.num is 1.

It does not include information about received data that was discarded because it was duplicated.

Only information from the most recent communication can be obtained. Information from communications prior to that cannot be obtained.

This function can get information about communication that has just occurred. Subsequent box changes (such as the deletion of inbox messages) are not reflected.

The CecNotificationData.count value is incremented every time StreetPass data is exchanged.
This value is only incremented by 1 each time a StreetPass communication occurs regardless of how many applications exchanged data in that communication.
Also, this value is reset to 0 every time the CTR system is restarted.
You can check for changes in this value to determine whether any StreetPass communications have occurred.
