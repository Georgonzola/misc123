nn::cec::CTR::Message Class

Syntax

class Message

Description

Class that handles CEC Message objects.

Parameters Set in Messages

The parameters set in messages placed in the outbox affect the message functionality, such as the search conditions for communication peers.
You can set multiple messages in the outbox, but the parameters set to the first message are the parameters that are used as conditions. When a message is sent successfully, that message is moved to the end of the queue.

MessageID

Identifies the message.
It is generated and assigned to a message by MessageBox::WriteMessage when it writes the message to a box.
Messages are deemed to be identical if they have the same message ID. A box cannot contain multiple instances of the same message.
This value cannot be changed manually.

GroupID

You can place multiple messages in the outbox.
The outbox normally sends messages one at a time, in the order they were stored.
You can send multiple messages simultaneously by assigning them a nonzero GroupId.
If the first message in the outbox has a nonzero GroupId, any other messages in the outbox with that same group ID are sent together with it.
The total size of the grouped messages must be no more than 100 KB.

You can switch between sending to peers that are friends or non-friends by preparing multiple messages with different MessageTypeFlag values and the same GroupId.

MessageTypeFlag

The flag for switching messages depending on the recipient peer.
Specify MESSAGE_TYPEFLAG_ANYONE to send regardless of whether the peer is a friend.
Specify MESSAGE_TYPEFLAG_FRIEND to send only to peers that are registered as friends.
Specify MESSAGE_TYPEFLAG_NON_FRIEND to send only to peers that are not registered as friends.
When sending to a StreetPass Relay point, the recipient is handled as a peer not registered as a friend. Data with MESSAGE_TYPEFLAG_ANYONE or MESSAGE_TYPEFLAG_NON_FRIEND is sent.

SendMode

The sending mode for messages.
SENDMODE_RECV : Receive only
SENDMODE_SEND : Send only
SENDMODE_SENDRECV : Send and receive
SENDMODE_EXCHANGE : Exchange

Communication is not performed if the send mode of the message the peer is attempting to send does not match this mode.
The combinations are as follows.
Receive only: Can communicate when peer is set to "send only" or "send and receive"
Send only: Can communicate when peer is set to "receive only" or "send and receive"
Send and receive: Can communicate when peer is set to "receive only", "send only", or "send and receive"
Exchange: Can communicate when peer is set to "exchange"

Send count.

Specifies the send count. The send count is decremented by one with each send. Data is no longer sent after the count reaches zero.
If you specify MESSAGE_SENDCOUNT_UNLIMITED to the send count, this value is not decremented and data is sent repeatedly.
If data does not reach the peer because communications have been interrupted or for another reason, the send count is not decremented.
The message remains in the outbox even after it has been sent and its send count is 0.
When the send count for all messages in an outbox reaches 0, sending from that box stops.

Propagation count.

Specifies the propagation count.
If this value is 2 or higher, the received data is copied to the outbox and sent to a different person.

The send count and propagation count values cannot both be set to greater than 1 at the same time.

Member Constants

S MESSAGE_BODY_UNITSIZE const size_t Specifies the number of bytes that the size of the data body specified in the SetMessageBody function must be a multiple of. Defined as 4 bytes.
S MESSAGE_BODY_SIZE_MAX const size_t Constant that represents the maximum size of the data body that can be specified in the SetMessageBody function. Defined as 96 KB.
