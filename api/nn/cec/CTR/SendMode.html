nn::cec::CTR::SendMode Type

Syntax

typedef u8 nn::cec::CTR::SendMode;

Description

The sending mode for messages. This value is specified in the message. (Such as Message::NewMessage.)

SENDMODE_RECV: Receive only.
SENDMODE_SEND: Send only.
SENDMODE_SENDRECV: Send and receive.
SENDMODE_EXCHANGE: Exchange.

Communication is not performed if the send mode of the message the peer is attempting to send does not match this mode.
The combinations are as follows.
Receive only: Can communicate when the peer is set to "send only" or "send and receive."
Send only: Can communicate when the peer is set to "receive only" or "send and receive."
Send and receive: Can communicate when the peer is set to "receive only," "send only," or "send and receive."
Exchange: Can communicate when the peer is set to "exchange."

If the system is set to "receive only," it does not send messages, and instead only waits for messages sent from peers.
If set to "send only," the system does not receive messages even if the peer is set to "send and receive."
For messages that are set to "exchange," if the message is not sent successfully (see note), the received message is not saved.
Also, the MessageID of the message that was sent with the exchange is recorded in the received message, and you can track which message it was exchanged with. This MessageID can be acquired with Message::GetMessageId_Pair.

Note: When the send operation completes, it is judged to have succeeded. There is no check to see whether the partner has fully saved the data. When received data is interrupted mid-save (such as with CecControl::StopScanning) or is duplicated in the peer's inbox, the send operation is judged to have succeeded, even though there has been no determination of the peer's receipt.
