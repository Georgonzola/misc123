nn::cx Namespace

Description

The namespace of the data compression and decompression library.

CTR-SDK includes the ctr_compress tool for Windows that creates the compressed files that the cx library handles. For more details, see the ctr_compress manual.

GetGzipUncompressedSize Gets the size of the deflate compressed data in GZIP format after decompression.
UncompressAny The compression type is determined from the data header, and the appropriate decompression process is executed. Because they do not have headers, LZ backward compressed files (BLZ) cannot be handled by this API. In addition, LH and LRC files that require a work buffer cannot be handled by this API. This function results in the linkage of the decompression code for all compression types. If you are using only certain compression formats, we recommend using the decompression functions specific to those being used.
UncompressRL Decompresses run-length compressed data.
UncompressLZ Decompresses LZ77 compressed data.
UncompressBLZ Decompresses LZ backward compressed (BLZ) data.
UncompressHuffman Decompresses Huffman compressed data.
UncompressLH Decompresses LZ-Huffman compressed data. The compressed data address must be SRC_ALIGNMENT_UNCOMPRESS_LH-byte aligned (4-byte aligned). The decompressed data address must be DST_ALIGNMENT_UNCOMPRESS_LH-byte aligned (2-byte aligned).
UncompressLRC Decompresses LZ-Range Coder compressed data. The compressed data address must be SRC_ALIGNMENT_UNCOMPRESS_LRC-byte aligned (4-byte aligned). The decompressed data address must be DST_ALIGNMENT_UNCOMPRESS_LRC-byte aligned (4-byte aligned).
UncompressDeflate Performs deflate compression data expansion appended to the cx common header.

UncompressZlib Decompresses deflate compressed data in ZLIB format.
UnfilterDiff Restores data converted using a difference filter. The compressed data address must be SRC_ALIGNMENT_UNFILTER_DIFF-byte aligned (4-byte aligned). The decompressed data address must be DST_ALIGNMENT_UNFILTER_DIFF-byte aligned (4-byte aligned).
Decompression With Error Checking
SecureUncompressAny Detects the compression format from the data header and executes the appropriate decompression process while checking for errors. Because they do not have headers, LZ backward compressed files (BLZ files) cannot be handled by this API. In addition, LH and LRC files that require a work buffer cannot be handled by this API. This function results in the linkage of the decompression code for all compression types. If you are using only certain compression formats, we recommend using the decompression functions specific to those being used.

SecureUncompressHuffman Decompresses Huffman compressed data while checking for errors.

SecureUncompressLRC Decompresses LZ-Range Coder compressed data. The compressed data address must be SRC_ALIGNMENT_UNCOMPRESS_LRC-byte aligned (4-byte aligned). The decompressed data address must be DST_ALIGNMENT_UNCOMPRESS_LRC-byte aligned (4-byte aligned).
