nn::dlp::CTR::ServerState Enumerated Type

Syntax

enum ServerState;

Value

Value Description
SERVER_STATE_INVALID The server is not initialized.
SERVER_STATE_INITIALIZED Initialization is complete.
SERVER_STATE_OPENED_SESSIONS Accepting session participation requests.
SERVER_STATE_DISTRIBUTING The state while in the process of distributing.
SERVER_STATE_COMPLETE_DISTRIBUTION The server has finished distributing a child program.
SERVER_STATE_REBOOTING_CLIENTS The server is rebooting clients. After confirming that all clients are disconnected, quit Download Play. If carrying out local communication after that, wait for client connections via UDS.
Note: Clients do not always reconnect.
SERVER_STATE_ERROR An error has occurred, and reinitialization must occur. There are two conditions under which the server enters this state: (1) the system enters Sleep Mode or wireless-disabled mode between calls to the Server::Initialize and Server::Finalize functions, or (2) a card is removed or a client that is not in the CLIENT_STATE_RECONNECTING_NETWORK state disconnects after a download session has started (between SERVER_STATE_DISTRIBUTING and SERVER_STATE_COMPLETE_DISTRIBUTION).
SERVER_STATE_PREPARING_FOR_SYSTEM_DISTRIBUTION This state is only used internally by the SDK. Preparing for system distribution.
SERVER_STATE_DISTRIBUTING_SYSTEM This state is only used internally by the SDK. Distributing the system.
SERVER_STATE_WAITING_RECONNECT This state is only used internally by the SDK. Waiting for clients that rebooted systems after system distribution to reconnect. Clients do not always reconnect.
SERVER_STATE_PREPARING_FOR_TITLE_DISTRIBUTION This state is only used internally by the SDK. The server is preparing to distribute a child program.
SERVER_STATE_DISTRIBUTING_TITLE This state is only used internally by the SDK. The server is distributing a child program.

Description

Server states. Following a call to the StartDistribute function, the distribution process proceeds automatically and transitions to the SERVER_STATE_COMPLETE_DISTRIBUTION state after the distribution has been completed.
However, there may be a transition to SERVER_STATE_ERROR during that time for any number of reasons (such as a communications error, Sleep Mode, or wireless-disabled mode). Use the GetState function or the GetEventDesc function to monitor the state.
Also, make sure that the user can cancel distribution while the server is in the SERVER_STATE_DISTRIBUTING state.
The DLP server waits indefinitely for rebooted clients to reconnect after a system update. However, some clients may not reconnect.
