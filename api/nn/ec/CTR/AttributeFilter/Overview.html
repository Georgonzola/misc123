nn::ec::CTR::AttributeFilter Structure

Syntax

struct AttributeFilter
{
   char filterType[ATTRIBUTE_FILTER_TYPE_LENGTH];
   char dataType[ATTRIBUTE_DATA_TYPE_LENGTH];
   char name[ATTRIBUTE_NAME_LENGTH];
   char value1[ATTRIBUTE_VALUE_LENGTH];
   char value2[ATTRIBUTE_VALUE_LENGTH];
};

Description

The structure for filtering item searches by attribute values. You can only use the ? (for matching any single character) and * (for matching any multiple characters) wildcards for matching when specifying the LIKE filter. value2 is only enabled when the filter type is BETWEEN, and is ignored for all other filter types. Using the filter type of BETWEEN matches attribute values between value1 and value2.

Member Variables

filterType char Specifies the filter type using a NULL-terminated string. Specify ==, !=, <=, <, >=, >, BETWEEN, or LIKE.
dataType char Specifies the data type using a NULL-terminated string. Specify string or integer.
When integer is specified, attribute values are specified by integers in the range of -2147483648 to 2147483647.
name char Specifies the attribute name to filter for using a NULL-terminated UTF-8-encoded string.
value1 char Specifies the attribute value to match using a NULL-terminated UTF-8-encoded string.
value2 char Specifies the attribute value to match using a NULL-terminated UTF-8-encoded string. Valid only when the filter type is BETWEEN.

Member Functions

S MakeItemAttributeFilter Creates filters for searching for specified items. Use this function if you need to re-obtain the newest information for specific items. Get ItemId using the ContentSet::GetItemId function and the ServiceItem::GetItemId function. If you are handling multiple data titles, filter the data titles using the ContentSetCatalogFilter::SetTitleFilters function and the ServiceItemCatalogFilter::SetTitleFilter function to uniquely specify items.
