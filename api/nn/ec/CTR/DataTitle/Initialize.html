nn::ec::CTR::DataTitle::Initialize Member Function

Syntax

ResultError Initialize(
     const TitleInfo & titleInfo
);

Parameters

Name Description
in titleInfo Title information for which to get information.

Return Values

Returns the execution result of the function. For more information about how to handle ResultError objects, see the ResultError reference. The results returned by the ResultError::GetResult function are shown below.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultDataTitleNotOwned The specified data title has never been initialized.
ResultDataTitleNotFound The specified data title was not found.
ResultFsMediaAccessFailedError An error occurred accessing the SD card.

Description

Initializes an instance of the DataTitle class.

Initializes the data title you are getting content information for. This function must be called before you use the member functions of the DataTitle class. The function returns ResultDataTitleNotOwned if the data title has never been initialized on this CTR system. The function returns ResultDataTitleNotFound if the data title has been initialized, but is not downloaded. This may occur when the data title has been deleted from the Data Management screen, or the SD card has been exchanged. If the data title information is needed to manage content, use the EcApplet::RequestInitializeSession function to get this information. Swapping the SD card after this function succeeds and before calling other member functions can cause the other member functions to return unexpected errors. If you detect that the SD card has been removed, destroy the initialized object instance and initialize a new instance.
