nn::ec::CTR::ECardCatalogFilter Class

Syntax

class ECardCatalogFilter : public nn::ec::CTR::detail::CatalogFilterImpl<ECardCatalogFilter>

Description

Filter for using ECardCatalog to search for ECardItem objects. Search by passing this class to the ECardCatalog::Retrieve function to filter the ECardItem search results.

Member Functions

ECardCatalogFilter Constructor for configuring a buffer for the filter. The configured buffer is used when calling the ECardCatalog::Retrieve function. You need the same number of buffers as you have configured filters. ECardCatalog::Retrieve returns ResultOutOfFilterMemory if the buffer is too small for the filters that were set. Fix the problem by such means as increasing the size of the buffer or setting fewer filters. The system consumes the buffer even if nothing is set for the filters.
SetReturnAttributes Configures the attribute name for the attribute value to get. Call the ECardCatalog::Retrieve function using the filter configured with the attribute names to get the attribute values corresponding to the relevant attribute names of the ECardItem objects. Use the ECardItem::GetAttributeByName function to access the retrieved attribute values. You can specify multiple attribute names in an array. ECardCatalogFilter only maintains references to the configured attribute name array and does not copy it, so be sure not to destroy the attribute name array until you are done using the configured filters.
SetAttributeFilters Configures the attribute filter. Call the ECardCatalog::Retrieve function using the filter configured with the attribute filters to get just the ECardItem objects that match the attribute filters. For more information about how to code attribute filters, see AttributeFilter. Currently, the only AttributeFilter::filterType that can be set in a ECardCatalogFilter is "==". The ECardCatalog::Retrieve object will return an error if a different filter type is set. You can specify multiple attribute filters in an array. Do not destroy the attribute filter array until you are done using the configured filter, because ECardCatalogFilter only maintains a reference to the specified attribute filter array and does not copy it.
SetTitleFilters Configures the title filter. Call the ECardCatalog::Retrieve function using a filter configured with the title filters to get just the ECardItem objects included in the configured data titles. There is usually no need to set title filters, because if none are configured, all of the data titles initialized in the Session object set in the ECardCatalog object will be used as title filters. Specifying a data title not initialized in the Session object as a title filter will cause the ECardCatalog::Retrieve function to return ResultSessionPreparedTitleNotFound and fail. You can specify multiple title filters in an array. ECardCatalogFilter only maintains a reference to the specified title filter array and does not copy it, so be sure not to destroy the title filter array until you are done using the configured filter.
SetECardTypeCode Configures the download code. Specify the download code retrieved from the CardInfo::GetTypeCode function. The item to be redeemed for the set download code can be searched when ECardCatalog::Retrieve is run using the selected filter. If no download code is set in the filter, it is treated as an invalid filter.
