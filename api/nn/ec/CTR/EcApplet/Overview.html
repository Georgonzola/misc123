nn::ec::CTR::EcApplet Class

Syntax

class EcApplet : private nn::ec::CTR::detail::SessionHolder

Description

Class for making requests of the EC applet.

Applications that call the member functions in the EC applet class must call the nn::applet::CTR::Enable function to enable the applet's features.

Also, graphics command requests must be stopped before making requests to the EC applet. Call the nngxWaitCmdlistDone function to end any rendering or drawing processes currently running.

To make requests to the EC applet for various types of processing, call the EcApplet::RequestInitializeSession function to initialize a session. (The EC applet also handles the process of initializing the session.)

An initialized session is necessary for the initialization of each class the EC library uses in communicating with the Nintendo eShop server.

When a request is made to the EC applet, it checks to see whether the system firmware is up to date. If it is not up to date, the applet generates ResultAppletNeedsSystemUpdate and a message appears prompting a system update. If a Session object has been successfully initialized in the system, the firmware is guaranteed to be the latest version.

Even if purchasing content or services from Nintendo eShop has been restricted by Parental Controls, the EC applet handles it by a method such as displaying the PIN code input screen. Applications do not need to keep track of Parental Controls.

When a purchase request is made to the EC applet, the user can add funds if the current funds in the user's account are insufficient. In the development environment, an error may occur if you add amounts less than 1000.

The EC applet may disconnect from or reconnect to the access point automatically while running. Note that the operations of the application may change depending on the AP connection status. Some actions are required by the application, such as checking the access point connection status again after a function sending a request to the EC applet returns. Unexpected operations may result if changes are made to the automatic connection status while the EC applet is running (for example, by the application calling the AC library API).

The operations when the POWER Button or HOME Button is pressed, or when the system is reset or transitioned into Sleep Mode while the EC applet is operating are as follows.

Pressing the POWER Button
The applet immediately notifies the application that the button was pressed, and performs a shutdown, except in the following circumstances.
During startup. During communication. While processing save data. During screen transition.Pressing the HOME Button
Blocks completely while the EC applet is running.

Sleep Mode
The system goes to sleep immediately in other than the following scenarios.
During startup. During communication. While processing save data. During screen transition.Software Reset
Blocks completely while the EC applet is running.

The operations in each of the blocked scenarios are as follows.
When the POWER Button is blocked, short presses of the POWER Button are ignored. Nothing is displayed. When the HOME Button is blocked, a message appears stating that transition to the HOME Menu is blocked.
When Sleep Mode is blocked, the system does not transition to Sleep Mode even if it is closed. Although the system continues to operate, the screen display and sound shut off. The system enters Sleep Mode after the blocked scenario ends.

Member Functions

S Preload Preloads the EC applet. Preloads the EC applet to make it start faster. However, the following cautions are needed. Preloading will not be finished even when returning from this function. Consequently, you must either call the EcApplet::WaitForPreload function when calling this function to preload, or use the EcApplet::IsRegistered function to confirm that preloading is complete before calling any request function. When a library applet other than the EC applet is preloading, you must cancel that preload before calling this function. You must not terminate the application while an applet is in a preloaded state. Call EcApplet::CancelPreload to cancel the preload before closing the application. After calling the Preload function, the application must not process the HOME Button until the IsRegistered function returns true. Specifically, if the IsExpectedToProcessHomeButton() function returns true, you must call the ClearHomeButtonState() function instead of the ProcessHomeButton() function.
S CancelPreload Cancels the preloading of the EC applet. When preloading has not completed, this function does nothing. Consequently, you must either call the EcApplet::WaitForPreload function before calling this function, or use the EcApplet::IsRegistered function to confirm that all preloading is complete. This function cancels preloading for all library applets. Note that calling this function cancels preloading for any other library applets in addition to EC applets.
S WaitForPreload Waits for the EC applet to be preloaded. The function does not return if it is called when it is not preloading. After calling the EcApplet::Preload function, you can only call this function once in the interval before the request functions or the EcApplet::CancelPreload function are called.
S IsRegistered Checks whether preloading of the EC applet is complete.
RequestInitializeSession Requests that the EC applet initialize a session.
RequestPurchaseContentSet Requests that the EC applet purchase and download a content set.
RequestPurchaseContentSets Requests that the EC applet purchase and download multiple content sets in a batch.
RequestDownloadContents Requests that the EC applet redownload content.
RequestDownloadAllOwnedContents Requests that the EC applet redownload all purchased content.
RequestRedeemContentSet Requests that the EC applet redeem a download voucher for a content set.
RequestRedeemUniqueContentSetAuto Calls the EC applet and displays the screen for entering a download code for an item. Once the download code has been entered, this function gets and redeems the content set (item) associated with the type of download code used and begins downloading it.
RequestPurchaseServiceItem Requests that the EC applet purchase a service item.
RequestPurchaseServiceItems Requests that the EC applet purchase multiple service items in a batch.
RequestRedeemServiceItem Requests that the EC applet redeem a download voucher for a service item.
RequestManageBalance Requests that the EC applet manage an account balance.
RequestAddBalance Requests that the EC applet add the shortfall to the balance.
