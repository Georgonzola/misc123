Description

Requests that the EC applet initialize a session.

Initializes the session by specifying the predetermined application ID, TIN, and title information of the data title and service titles handled by the application.

This function requires an Internet connection because it communicates with the Nintendo eShop server.

The EC applet downloads the metadata content of the title entered, and initializes the specified data title.

The EC applet does nothing if the specified data title does not exist on the Nintendo eShop server. Use the functions of the Session class to confirm the information of the initialized data title.

You cannot request initialization of a data title while the content of a data title is mounted. The MetaDataReader class mounts metadata content. When there is a valid instance of the MetaDataReader class, your request to initialize generates ResultDataTitleInUse and fails.

The EC applet does not perform any actions on the specified service title. It saves the specified service title information in the Session object.

Initializing a new session invalidates any old ones. The instance that was using the old session must be reinitialized.

If this function discovers any inconsistencies between the license information on the Nintendo eShop server and the local license information while it is running, it synchronizes the local information with the information on the Nintendo eShop server. When doing so, it may delete local licenses or data titles. If a data title or its license is deleted while there is a valid instance of that DataTitle or ServiceTitle class, that instance no longer works correctly. Destroy any instances of the DataTitle or ServiceTitle class that you initialized before calling this function, and create new instances after the function executes.

You can specify up to 64 titles in this function.

Sometimes there are requests to close the application during a call to the EC applet. Close the application immediately after receiving ResultAppletCloseApplicationRequested.

When you call this function, the EC applet checks whether a writable SD card is inserted in the system.
