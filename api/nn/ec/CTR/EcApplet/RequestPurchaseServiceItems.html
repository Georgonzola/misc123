nn::ec::CTR::EcApplet::RequestPurchaseServiceItems Member Function

Syntax

ResultError RequestPurchaseServiceItems(
     const TitleInfo & title,
     const ServiceItemId items[],
     size_t numItems
);

Parameters

Name Description
in title Title information for the data title you are purchasing and downloading a content set for.
in items[] Specifies the ServiceItemId array for the service items to be purchased in batch.
in numItems Number of elements in items.

Return Values

Returns the result of the function. For more information about handling ResultError objects, see the ResultError reference. The results returned by the ResultError::GetResult function are shown below.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultSessionInvalid Invalid session.
ResultTooManyItems The number of specified items exceeds the maximum number.
ResultAppletWrongPinCode The entered PIN code did not match the one configured on the system.
ResultAppletCloseApplicationRequested Application shutdown was requested. Close the application immediately.
ResultAppletCancelled The process was canceled by user operation.
ResultAppletPowerButton The process was canceled by the user pressing the POWER Button.
ResultAppletAlreadyPurchased Attempted to purchase a previously purchased item.
ResultAppletItemUnpurchasable Attempted to purchase an item that is not for sale.
ResultAppletItemNotFound The specified item does not exist.
ResultAppletWifiOffError Wi-Fi is off.
ResultAppletFailedConnectError Connection failed for another reason.
ResultAppletSdBroken The SD card is broken.
ResultAppletSdEjected The SD card was removed.
ResultAppletFailedError Another error occurred. (General error.)
ResultAppletFailed The same general error as ResultAppletFailedError.
ResultAppletRequiresReConnect The server connection has been ended, because there has been no user input for a long time.
ResultAppletNeedsBalanceUpdate Detected that the balance needs to be updated while displaying the applet. Call EcApplet::RequestInitializeSession() again.

Description

Requests that the EC applet purchase multiple service items in a batch.

This function requires an Internet connection because it communicates with the Nintendo eShop server.

Get the ItemId for the ServiceItemId structure passed to this function with the ServiceItem::GetItemId function or a similar one.

The RIV token for the ServiceItemId structure passed to this function contains an item code and reference ID, and the Nintendo eShop server validates them prior to purchase.

The string set in the ServiceDescription for the ServiceItemId structure passed to this function appears on the purchase confirmation screen of the EC applet. Specify the string you want to display.

The maximum number of service items that can be purchased in a batch is 20.

Sometimes there are requests to close the application during a call to the EC applet. Close the application immediately after receiving ResultAppletCloseApplicationRequested.

This function successfully ends, and the local license information is synchronized with the Nintendo eShop server. (The synchronized information is limited to the licenses acquired by this function.)

When a ResultError for which ResultError::RequiresNewSession returns true is returned, if the user proceeds with the purchase after reinitializing the session, sometimes the ItemId is no longer valid and an error occurs. To avoid this error, get the content set again.
