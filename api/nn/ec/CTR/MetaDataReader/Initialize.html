nn::ec::CTR::MetaDataReader::Initialize Member Function

Syntax

ResultError Initialize(
     const TitleInfo & titleInfo
);

Parameters

Name Description
in titleInfo Title information for which to get metadata.

Return Values

Returns the execution result of the function. For more information about how to handle ResultError objects, see the ResultError reference. The results returned by the ResultError::GetResult function are shown below.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultTitleInvalid Invalid title information was specified.
ResultMetaDataTitleNotFound The specified title was not found.
ResultMetaDataInvalidFormat Invalid format.
ResultFsMediaAccessFailedError An error occurred while accessing the SD card.

Description

Initializes an instance of the MetaDataReader class.

Initializes the data title for which to get metadata. The function returns ResultMetaDataTitleNotFound if the data title has not been downloaded.

This function must be called before you use other member functions of the MetaDataReader class.

The title information specified by titleInfo must be a data title. Calling this function to initialize an instance of MetaDataReader that consumes a resource that is shared by the system as a whole. You must not have more than one valid instance of MetaDataReader at any one time. Swapping the SD card after this function succeeds and before calling other member functions can cause the other member functions to return unexpected errors. If you detect that the SD card has been removed, destroy the initialized object instance and initialize a new instance.
