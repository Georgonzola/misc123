nn::ec::CTR::NeedsSystemUpdate Function

Syntax

ResultError NeedsSystemUpdate(
     bool * pOut
);

Parameters

Name Description
out pOut Specifies whether a system update is required.

Return Values

Returns the result of the operation.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultAcNotConnectedError Either not connected to the Internet, or the connection was lost.
ResultInfraReportError Infrastructure reported an error.
ResultInfraReceiveError Failed to receive from the infrastructure connection.
ResultInfraInvalidResponseError Invalid response from the infrastructure connection.
ResultInfraTimeoutError Infrastructure communication timeout.

Description

Returns whether a system update is needed.

When using the Nintendo eShop REST API, use this function to determine whether a system update is required before using the ShowPurchaseConfirmationDialog or ShowRedemptionConfirmationDialog functions to display the EC applet.

This check is needed because the system update may change the design/behavior of the EC applet. Make sure to call this function every time you log in to the independent server to make sure you display the latest EC applet.

This function requires an Internet connection because it communicates with a server.

To set the device to a state in which a system update is required for testing purposes in the development environment, use DevMenu to delete a dummy data title. For more information, see the DevMenu Reference.
