nn::ec::CTR Namespace

Description

Namespace for the CTR E-Commerce API. (It can also be written without the CTR portion.) The ec library is the library for purchasing and managing add-on content and service items. The following is a rough description of the process for purchasing and downloading add-on content. 1. Call Initialize to initialize the library. 2. Call the EcApplet::RequestInitializeSession function to have the EC applet initialize a Session object and establish a session with the eShop server. 3. Use the initialized Session object to initialize the ContentSetCatalog object. 4. Call the ContentSetCatalog::Retrieve function to search for content sets, and then call the ContentSetCatalog::GetContentSet function to get the ContentSet. 5. Call the ContentSet::GetItemId function to get the ItemId for the content set to purchase. 6. Call the EcApplet::RequestPurchaseContentSet function to have the EC applet purchase and download the content set. Use the DataTitle or MetaDataReader classes to manage purchased and downloaded add-on content. Use the Server class for operations such as getting the balance and server time. When handling download codes, use the ECardInfo class to get download code information; the ECardItem, ECardCatalogFilter, and ECardCatalog classes to search for redeemable items; and the EcApplet::RequestRedeemContentSet function to download the content being redeemed. The EC library features assume that a writable SD card has been inserted into the CTR system.
Before downloading or deleting add-on content, be sure that a writable SD card is inserted in the CTR system.
Perform this check inside the application before the downloading or deleting of add-on content, or call the EcApplet::RequestInitializeSession function and then monitor for removal of the SD card until the point of downloading or deleting.
Processing that uses the EC library halts if it detects that an SD card has been inserted or removed during such processing. For more information about how to detect SD card hot-swapping, see the API Reference Manual for the FS library.
Note that if you are only reading the add-on content you do not need to check for a writable SD card. The EC library returns all strings encoded in UTF-8, with the exception of the string set in ServiceDescription. The string set in ServiceDescription is encoded in UTF-16. Use the enc library to convert between UTF-8 and UTF-16. None of the classes or functions in the ec library are thread-safe. Do not call library functions from multiple threads at the same time. The EC applet uses a thread with a priority of 17,22. If you have other threads running while the applet is running, be careful about their priority settings. Another thread with a priority of 15 is created to run the applet. The listings for the results returned by each API function are not intended to cover all of the results returned by their respective API functions. Note that results other than those listed may also be returned.

nn::ec::CTR::ContentSetGroupCatalog Class for searching for content set groups. Content set groups provide information about content sets grouped by user-defined keyword attribute values. Set a filter to group content sets by using ContentSetGroupCatalogFilter, and then call ContentSetGroupCatalog::Retrieve to get a ContentSetGroup as information grouped by ContentSetGroupCatalog::GetContentSetGroup. Content sets are grouped by attribute values of the specified length. For example, assume that the attribute name Color has four content sets, having the values BLACK, BLUE, GREEN, and BLACK. Grouping these content sets in ContentSetGroupCatalogFilter::SetAttributeGroupFilter with a filter specifying the attribute name "Color" and length "2" yields the following two groups. A group named BL containing 3 content sets. A group named GR containing 1 content set. If you specify a length of 0 in ContentSetGroupCatalogFilter::SetAttributeGroupFilter, content sets are grouped by the attribute value. In this example, the following three groups are returned. A group named BLACK containing 2 content sets. A group named GREEN containing 1 content set. A group named BLUE containing 1 content set.

eShop
nn::ec::CTR::Server Class for getting information from the Nintendo eShop server. Using the Server class requires a Session object that has been initialized using the EcApplet::RequestInitializeSession function. After initializing the Server object, calling the EcApplet::RequestInitializeSession function again, such as to reinitialize, causes the old session to be destroyed, with any further calls to any object functions returning a ResultError that includes ResultSessionInvalid. In such cases, use the newly initialized Session to initialize a new instance of the Server class, and then use that object.
Managing
nn::ec::CTR::Session Class for managing sessions established with the eShop server. The Session object can only be initialized using the EcApplet::RequestInitializeSession function. Use the functions of the Session class to get information for initialized data titles. Even if data title information is specified in the EcApplet::RequestInitializeSession function, if initialized data title information cannot be found, the data on the eShop server cannot be used because it is not yet on sale or the sale period has ended. You must have an initialized Session object to initialize the other classes used to communicate with the eShop Server. When an EC library function returns a ResultError instance with a ResultError::RequiresNewSession method that returns true, the Session object that was in use has become invalid. Call EcApplet::RequestInitializeSession again to reinitialize the Session object. When a new Session object is initialized, old ones are automatically invalidated. A new Session object must be initialized when the removal of the SD card is detected, but you can avoid using the old Session object by calling Session::Invalidate to invalidate it explicitly. If service item information is passed to the EcApplet::RequestInitializeSession function, the initialization for the service item is always in a completed state because service items do not require initialization.

Structures

nn::ec::CTR::Money The structure that stores balance and price information. To display balance and price information on the screen, it is not enough to just display the string stored in the Money structure. You must also display any currency prefix and suffix. Use the Server::ConvertPrice function to convert the information in the Money structure into a PriceIndication structure. Make sure to display the converted PriceIndication on the screen.
nn::ec::CTR::PriceIndication The structure that stores balance and price notation strings. To display balance and price information on the screen, it is not enough to just display the string stored in the Money structure. You must also display any currency prefix and suffix. Use the Server::ConvertPrice function to convert the information in the Money structure into a PriceIndication structure. Then always display the converted PriceIndication on the screen.
nn::ec::CTR::ItemId The structure that indicates item IDs.
nn::ec::CTR::AttributeName The structure that stores the item's attribute name.
nn::ec::CTR::AttributeFilter The structure for filtering item searches by attribute values. You can only use the ? (for matching any single character) and * (for matching any multiple characters) wildcards for matching when specifying the LIKE filter. value2 is only enabled when the filter type is BETWEEN, and is ignored for all other filter types.
