nn::ec::CTR::ResultError Class

Syntax

class ResultError

Description

Class for getting processing results and error determinations.

The ResultError objects returned by the functions of the EC library must be properly handled by the application. The following pseudocode is an example of how to handle these errors.
ResultError resultError = SomeEcLibraryFunction();

// When you send a request to the EC applet, you must check for any
// requests to close the application before handling any other errors. 
if(resultError.GetResult() == ResultAppletCloseApplicationRequested())
{
    // Finalize the application. 
}
else if(resultError.IsFailure())
{
    if(SD card inserted or removed)
    {
        // If an SD card is inserted or removed while calling an EC library function, 
        // unexpected errors could occur. Before checking the ResultError object, determine whether an SD card
        // has been inserted or removed. If so, stop any processes that use the EC library. 
        // See the fs library reference for instructions on detecting the insertion or removal of an SD card. 
        
        // If the SD card is reinserted and you want to start again, you must execute
        // the EcApplet::RequestInitializeSession function to re-create the session. 
    }
    else
    {
        if(resultError.NeedsShowErrorCode())
        {
            // The error code must be displayed, so call the Error/EULA applet. 
            // You can get the error code to display in the Error/EULA applet using the resultError.GetErrorCode function. 
        }
        
        if(resultError.RequiresNewSession())
        {
            // A new session must be initialized. 
            // This error occurs when there is a discrepancy between the session and the information on the eShop Server,
            // such as when a data title is updated on the server. 

            // The current session must be destroyed and a new one initialized
            // by calling the EcApplet::RequestInitializeSession function again. 

            // The user will be notified by the EC applet, so the application does not need to give any notices to the user.  
        }
        else if(resultError.IndicatesServiceNotAvailable())
        {
            // The eShop Service is not available. 
            // Stop the process that is attempting to connect to the eShop Service. 

            // The user will be notified by the EC applet, so the application does not need to give any notices to the user.  
        }
        else
        {
            // If there are any errors that the application needs to handle, handle them here. 
            switch(resultError.GetResult().GetValue())
            {
                ...
            }

            // Fix any Result objects caused by bugs in the program, such as Result objects that are an ec module with a level of NN_LEVEL_USAGE or NN_LEVEL_PERMANENT,
            // so that they no longer occur in the development phase. 
            
            // If this code is reached for some other reason, go back to the state before the function was called. 
            // The program might run correctly if the user repeats the same actions. 

            // For example, if a DLC update or Nintendo Network ID sign-in is canceled, 
            // an error might be returned. 

            // The user is notified of such errors by the EC applet or the Error/EULA applet, so the application does not need to do so. 
            // If the state prior to calling the function was a content or item list, 
            // the state might have changed even if the function call failed, so update the list. 
        }
    }
}
