nn::ec::CTR::ServiceItemCatalog::Retrieve Member Function

Syntax

ResultError Retrieve(
     u32 startOffset,
     size_t numItems,
     ServiceItemCatalogFilter & filter
);

Parameters

Name Description
in startOffset Offset for getting service items from the search results that match the filter.
in numItems The number of service items to get.
in filter Filter to use for the search.

Return Values

Returns the execution result of the function. For more information about how to handle ResultError objects, see the ResultError reference. The results returned by the ResultError::GetResult function are shown below.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultSessionInvalid Invalid session.
ResultNewSessionRequired A new session is required.
ResultCatalogInvalidArgument An invalid filter was set for filter.
ResultOutOfFilterMemory The buffer set in ServiceItemCatalogFilter is too small.
ResultOutOfCatalogMemory The buffer configured for the catalog is too small.
ResultOutOfCatalogMemoryError The buffer set in the catalog is too small to hold the search results.
ResultOutOfSystemMemoryError Insufficient system memory.
ResultAcNotConnectedError Either not connected to the Internet, or the connection was lost.
ResultInfraNeedsReconnect Infrastructure reconnection requested.
ResultInfraReportError Infrastructure reported an error.
ResultInfraReceiveError Failed to receive from the infrastructure connection.
ResultInfraInvalidResponseError Invalid response from the infrastructure connection.
ResultInfraNeedsUpdateError Infrastructure updated, so the system must be updated.
ResultInfraTimeoutError Infrastructure communication timeout.
ResultInfraBusyError The infrastructure connection is busy.

Description

Searches for service items.

This function requires an Internet connection because it communicates with the eShop server. When this function is called, it first uses the buffer set by ServiceItemCatalogFilter to create a search query for the eShop server. The function returns ResultOutOfFilterMemory if the buffer for creating the search query is too small. Set a larger buffer in the ServiceItemCatalogFilter object.

The search results returned by the eShop server are extracted temporarily in system memory. The function returns ResultOutOfSystemMemoryError if there is not enough system memory for this operation. You can avoid running out of memory by adjusting the number of service items obtained for each call, and calling this function multiple times with the offset specified to search through the service items a few at a time. Use the ServiceItemCatalog::GetTotalNumMatched function to get the total number of service items that matched the filter. The function next copies the service item information from the search results extracted in system memory to the buffer set in the ServiceItemCatalog::ServiceItemCatalog constructor.

The acquired service item information is sorted by ascending order of item code. Only part of the Service Item information is copied if this buffer is too small to hold all of the content set information. Use the ServiceItemCatalog::GetNumRetrieved function to get the number of service items stored in the buffer. The function returns ResultOutOfCatalogMemory or ResultOutOfCatalogMemoryError if the buffer is too small to hold even one item. Note that applying additions and updates to the items registered to the eShop server can change the service items that match the filter and the service items that can be obtained with one search query. If you do not plan for additions and updates, adding and updating items may not be possible in your application after it is released. To avoid this problem, adequately verify that you can add and update items under the expected conditions of actual operations.

The other member functions of this class must not be called if this function does not return successfully.

If this function is called multiple times, the ServiceItem objects obtained from previous calls are invalidated.
