nn::ec::CTR::Session Class

Syntax

class Session

Description

Class for managing sessions established with the eShop server. The Session object can only be initialized using the EcApplet::RequestInitializeSession function. Use the functions of the Session class to get information for initialized data titles. Even if data title information is specified in the EcApplet::RequestInitializeSession function, if initialized data title information cannot be found, the data on the eShop server cannot be used because it is not yet on sale or the sale period has ended. You must have an initialized Session object to initialize the other classes used to communicate with the eShop Server. When an EC library function returns a ResultError instance with a ResultError::RequiresNewSession method that returns true, the Session object that was in use has become invalid. Call EcApplet::RequestInitializeSession again to reinitialize the Session object. When a new Session object is initialized, old ones are automatically invalidated. A new Session object must be initialized when the removal of the SD card is detected, but you can avoid using the old Session object by calling Session::Invalidate to invalidate it explicitly. If service item information is passed to the EcApplet::RequestInitializeSession function, the initialization for the service item is always in a completed state because service items do not require initialization.

Member Functions

GetNumPreparedTitles Returns the number of data titles initialized this session. If service item information is passed to the EcApplet::RequestInitializeSession function, that data is included in the count.
ListPreparedTitleInfos Lists information for data titles that have been initialized this session. If service item information is passed to the EcApplet::RequestInitializeSession function, that information is included.
FindPreparedTitleInfos Searches information for data titles that have been initialized this session. If service item information is passed to the EcApplet::RequestInitializeSession function, that information is included.
Validate Confirms the validity of the session.
Invalidate Invalidates the session. Explicitly invalidates a session. Use this function when you want to invalidate the session currently in use, such as when the SD card removal is detected. Calling this function has no effect if the session is already invalid.
