nn::ec::CTR::ShowPurchaseConfirmationDialog Function

Syntax

ResultError ShowPurchaseConfirmationDialog(
     const TitleInfo & appId,
     const PurchaseInfo & purchaseInfo
);

Parameters

Name Description
in appId The application ID.
in purchaseInfo The purchase confirmation information.

Return Values

Returns the result of the function. For more information about handling ResultError objects, see the ResultError reference. The ResultError::GetResult function returns the following results.

Value Description
Result::IsSuccess The user selected the purchase button in the applet, and control has returned to the application.
ResultAppletWrongPinCode The entered PIN code did not match the one configured on the system.
ResultAppletCloseApplicationRequested Application shutdown was requested. Close the application immediately.
ResultAppletCancelled The process was canceled by user operation.
ResultAppletPowerButton The process was canceled by the user pressing the POWER Button.
ResultAppletWifiOffError Wi-Fi is off.
ResultAppletFailedConnectError Connection failed for another reason.
ResultAppletFailedError Another error occurred. (General error.)
ResultAppletFailed The same general error as ResultAppletFailedError.
ResultAppletInvalidParameter The string set in purchaseInfo is invalid.

Description

Makes a request to the EC applet to show the purchase confirmation screen.

Set the information retrieved from the independent server to a parameter for display.

Sometimes there are requests to close the application during a call to the EC applet. Close the application immediately after receiving ResultAppletCloseApplicationRequested.

The EcApplet::InitializeSession function does not need to be called before executing this function.
