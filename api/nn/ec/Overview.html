nn::ec Namespace

Description

The namespace of the E-Commerce API.

Namespaces

nn::ec::CTR Namespace for the CTR E-Commerce API. (It can also be written without the CTR portion.) The ec library is the library for purchasing and managing add-on content and service items. The following is a rough description of the process for purchasing and downloading add-on content. 1. Call Initialize to initialize the library. 2. Call the EcApplet::RequestInitializeSession function to have the EC applet initialize a Session object and establish a session with the eShop server. 3. Use the initialized Session object to initialize the ContentSetCatalog object. 4. Call the ContentSetCatalog::Retrieve function to search for content sets, and then call the ContentSetCatalog::GetContentSet function to get the ContentSet. 5. Call the ContentSet::GetItemId function to get the ItemId for the content set to purchase. 6. Call the EcApplet::RequestPurchaseContentSet function to have the EC applet purchase and download the content set. Use the DataTitle or MetaDataReader classes to manage purchased and downloaded add-on content. Use the Server class for operations such as getting the balance and server time. When handling download codes, use the ECardInfo class to get download code information; the ECardItem, ECardCatalogFilter, and ECardCatalog classes to search for redeemable items; and the EcApplet::RequestRedeemContentSet function to download the content being redeemed. The EC library features assume that a writable SD card has been inserted into the CTR system. Before downloading or deleting add-on content, be sure that a writable SD card is inserted in the CTR system. Perform this check inside the application before the downloading or deleting of add-on content, or call the EcApplet::RequestInitializeSession function and then monitor for removal of the SD card until the point of downloading or deleting. Processing that uses the EC library halts if it detects that an SD card has been inserted or removed during such processing. For more information about how to detect SD card hot-swapping, see the API Reference Manual for the FS library. Note that if you are only reading the add-on content you do not need to check for a writable SD card. The EC library returns all strings encoded in UTF-8, with the exception of the string set in ServiceDescription. The string set in ServiceDescription is encoded in UTF-16. Use the enc library to convert between UTF-8 and UTF-16. None of the classes or functions in the ec library are thread-safe. Do not call library functions from multiple threads at the same time. The EC applet uses a thread with a priority of 17,22. If you have other threads running while the applet is running, be careful about their priority settings. Another thread with a priority of 15 is created to run the applet. The listings for the results returned by each API function are not intended to cover all of the results returned by their respective API functions. Note that results other than those listed may also be returned.
