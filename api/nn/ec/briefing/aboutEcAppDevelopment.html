Overview

This document explains the handling required for downloadable content when developing an application that uses downloadable content.

Notes

  • The correct serial number must be written to the CTR system for applications that use the EC library. You can use the Config tool to check the system's serial number. Verify that the serial number matches the serial number printed on the back of the system. If the serial number is not written correctly and the printed serial number starts with EJF, the system can be repaired. Contact Nintendo. Any other serial number is for a prototype system that cannot be repaired.
  • Data titles created with the version of E-Commerce Kit for CTR-SDK 3.2 or later are not compatible with those created with E-Commerce Kit for CTR-SDK 3.1. (Compatible with version 3.2 and later.)
    Note that for this reason, the demo data titles on the server have been replaced and some of the 3.1 demo programs will not work.
    Also, for the same reason, when independent data titles are already used with 3.1 and you are replacing with SDK 3.2 or later, the data title must be rebuilt and uploaded to the server.
  • The system might freeze if the user removes the SD card while the ES applet is running for a purchase or otherwise.
    This issue has been noted in Symptoms Due to Design Limitations of Specifications as follows: [FS] Removing a card while it is being accessed may cause a fatal error or cause the system to stop operating.
    This freeze is something that applications cannot overcome, so consider this to be an error that your application cannot handle.
  • The development and production shop environments have some differences.
    For example, the production environment eShop for the Korean region does not allow credit card payments, but the development environment shows the option.

Downloadable Content Development Flow

The following example shows the workflow when developing for downloadable content.

  1. Create the downloadable content data to be accessed from the application.
    For more information about how to create downloadable content data, see Creating Downloadable Content and Sample Demo for Making a Data Title.
  2. Use local import to import the downloadable content to your development machine, and implement a scene that uses the downloadable content.
    For more information about importing downloadable content locally, see Importing Locally. For more information about how to access downloadable content from the application, see Accessing Downloadable Content and Sample Demo for Managing Purchased downloadable Content.
  3. Implement a scene where the user can purchase, download, and manage downloadable content.
    For more information about how to upload data to the eShop server and how to purchase, download, and manage downloadable content from eShop in an application, see Importing from the eShop Server, the EC library reference documentation, and Sample Demo for Purchasing and Removing downloadable Content.
  4. Verify that downloadable content can be added and updated, so that downloadable content can be added later if it becomes necessary. See Adding and Updating Downloadable Content.

Creating Downloadable Content

The final step in creating downloadable content is the creation of a data title as a CIA file. For more information, see How to Create Data Titles.

Importing Downloadable Content
Save the downloadable content to an SD card to use it from an application. Import the downloadable content locally from the DevMenu, or by downloading from the eShop server after you upload your downloadable content to the eShop server.

Importing Locally
Import locally to use content without downloading from a server. Import the data title containing the downloadable content (the CIA file) from the DevMenu. For more information about how to use the DevMenu, see the DevMenu documentation. Check Cautions When Switching Between Local Development and Development Using the eShop Server before changing the import method.

Importing From the eShop Server

Import from the server to use content from the eShop server, much as in a production environment. Upload the downloadable content to the eShop server, and then develop or test the process of downloading the content and using it from your application.

For more information about how to upload to the eShop server, see the IMAS Operations Manual, which is distributed separately.

Use EcDevTool to add any money required for purchases.

Check Cautions When Switching Between Local Development and Development Using the eShop Server before changing the import method.

If you are going to revert downloadable content, first delete the license for the downloadable content using EcDevTool on the development hardware to which the content was downloaded. Error 005-5637 is returned if you revert without deleting the license for the downloadable content.

Cautions When Switching Between Local Development and Development Using the eShop Server

The handling is different for downloadable content downloaded and imported from the server and for downloadable content imported locally using the DevMenu.

Using both import methods at the same time can result in being unable to access the content.

When switching between local development and development on the eShop server, delete the license and data title in advance using EcDevTool.

Accessing Downloadable Content
Downloadable content is handled as a file archive, just as for Romfs packages in regular applications. To access imported downloadable content, call the nn::fs::MountAddOnContent function to mount the downloadable content file archive, and then use functions such as nn::fs::FileInputStream to access the files within the archive. If TryRead was successful, the loaded data is guaranteed to be correct file data.

Managing Downloadable Content
You can browse and delete downloadable content from the DevMenu, whether imported locally or from the eShop server.

Managing Rights to Downloadable Content

Purchasing downloadable content grants users a permanent right to use that content, allowing users to redownload any purchased content that was deleted locally without having to repurchase that content.

Use EcDevTool to delete rights to any purchased content.

Adding and Updating Downloadable Content

If downloadable content will be added and updated to your application after it has been released, you must verify the application under assumed cases of actual operation. If the application does not provide sufficient support for these features, the post-release addition and updating of downloadable content may not be possible. We recommend that you verify your applications under conditions that are as close to actual operations as possible, taking the following points into consideration.

  • Does the application work as expected when the assumed item information is set to the eShop server?
  • Does the application work as expected when the data title is updated in the eShop server?
  • Does the application work as expected when the maximum number of content pieces are registered to the eShop server?

Displaying Account Balances

There is a possibility of the account balance changing even while the application is running, for example if the Theme Shop or another device sharing the same Nintendo Network ID is used.

When purchases are made, the correct balance is shown in the EC applet, so the application does not need to show the balance all the time. However, note that the balance can change this way if you plan to show the balance in your application.

Notes on Changing Region or Country Settings

If you use the Config tool or System Settings to change the country settings on development hardware, you must also reflect the change in the eShop server account. You can run EcDevTool to apply changes to the country setting of the development hardware in the eShop account. It is not possible to purchase content if the country settings of the development hardware and eShop account do not match.

Although you can change the region of the development hardware by running the System Updater for each region, you must first delete the eShop account using EcDevTool. If you change the region without deleting the eShop account, you cannot connect to the eShop server. If you accidentally become unable to connect to the eShop server, run EcDevTool to delete the eShop account.
If you still cannot connect, revert to the original region and then run EcDevTool again.

Notes on Using Nintendo Network Accounts

If you register a Nintendo Network account and use the ECDK, the eShop account is managed automatically by the Nintendo Network account.
When this happens, you can no longer use EcDevTool to delete licenses and change the country setting.
Delete the eShop account to delete licenses.
To change the country setting, first go to the System Settings and Nintendo Network ID Settings, and then erase the Nintendo Network ID. Then use EcDevTool.
For more information, see EcDevTool.

Because of this restriction, if you do not need a Nintendo Network account for development, we recommend that you develop and debug your application without registering a Nintendo Network account.

For more information about the operation of eShop accounts when a Nintendo Network account has been registered, see Nintendo 3DS Overview - E-Commerce.

If a Nintendo Network ID has been registered to a 3DS system and that system is later initialized or the NNID is deleted, the fate of the license information for downloadable content will differ, depending on whether the October 2014 system update has been applied.

  • For a 3DS system to which the October 2014 system update has not been applied:
    • The license information for downloadable content remains, even if the system is initialized and the NNID is deleted.
  • For a 3DS system to which the October 2014 system update has been applied:
    • The license information for downloadable content is deleted when the system is initialized or the NNID is deleted.

For more information about service item rights data behavior, see ECDK: Developing Applications That Support Service Items.

Determining Whether Downloadable Content Is Available on the Release Date

If you are not selling downloadable content on the day the application is released, this library provides the following means of notifying the user of that fact within the application.

Determine whether downloadable content is available for sale by checking whether the result of calling the Session::GetNumPreparedTitles function is 0 after the EcApplet::RequestInitializeSession function succeeds when the data title has not been registered on the server.

If the data title is registered on the server, but downloadable content is not available for sale, set SELECTION_FILTER_PURCHASABLE in the ContentSetCatalogFilter search range, and make the determination by checking whether the result of the ContentSetCatalog::GetTotalNumMatched function is 0 after the ContentSetCatalog::Retrieve function succeeds.

Cautions When Calling the EC Applet

Mounting Prohibited

When calling the EC applet or when deleting content (by calling nn::ec::DataTitle::DeleteContents), first unmount all downloadable content and metadata content. If this content is mounted when these operations are performed, content management information cannot be updated. Making a purchase with these items mounted, for example, leaves the purchased content in an unusable state.

An error results if they are mounted when the function is called. When calling the applet, do not mount via a separate thread, either.

The nn::ec::MetaDataReader::Initialize function automatically mounts metadata content, so be sure to call the MetaDataReader object's Finalize function before calling the EC applet. Call nn::fs::Unmount to unmount any downloadable content mounted using the nn::fs::MountAddOnContent function.

Cautions for File Streaming

When calling the EC applet, allocate a buffer of sufficient size, in accordance with the file system reference, if you stream any files, such as via a sound thread. During streaming, downloading and writing operations for downloadable content are given lower priority, which causes the import process to take longer.

Samples

Following are samples for downloadable content.

Sample Demo for Creating a Data Title
SampleDemos/ec/DataTitle
This demo creates downloadable content and a data title that includes the downloadable content.
Documentation can be found in SampleDemos/documents/demos/ec/DataTitle.html.
SampleDemos/ec/DataTitleRaw
This demo creates data titles by using the tools directly.
Documentation can be found in SampleDemos/documents/demos/ec/DataTitleRaw.html.

Sample Demo for Managing Purchased Downloadable Content
SampleDemos/ec/ContentsListUp
This demo uses the EC library to manage (list, access, delete, and redownload) downloadable content.
Documentation can be found in SampleDemos/documents/demos/ec/ContentsListUp.html.

Sample Demo for Purchasing and Deleting Downloadable Content
SampleDemos/ec/Purchase
This demo uses the EC library to purchase and delete downloadable content. It also implements a feature for the batch re-downloading of purchased content.
Documentation can be found in SampleDemos/documents/demos/ec/Purchase.html.

Sample Demo That Redeems Download Code
SampleDemos/ec/EasyRedemption
This demo redeems download code using the EC library by entering a download code directly in the EC applet.
The application only has to make a DLC redemption request to the EC applet, which makes the redemption process very simple.
It also implements a feature for the batch re-downloading of purchased content.
Documentation can be found in SampleDemos/documents/demos/ec/EasyRedemption.html.
