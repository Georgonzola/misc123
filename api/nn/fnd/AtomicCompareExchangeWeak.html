nn::fnd::AtomicCompareExchangeWeak Function

Syntax

#include <nn/fnd.h>
template <typename IntOrPointer>bool AtomicCompareExchangeWeak(
     volatile IntOrPointer * p,
     IntOrPointer * pExpected,
     IntOrPointer value
);

Template Arguments

Name Description
IntOrPointer The target type. It must be either an integer (8-bit, 16-bit, 32-bit, or 64-bit) or a pointer.

Parameters

Name Description
in p Pointer to variables in memory that are subject to atomic operations.
inout pExpected Pointers to variables that store values to compare.
in value Value to assign.

Return Values

Returns true when the assignment is made and false when it is not.

Description

Replaces values atomically in weak comparisons.

Assigns value to *p when *p and *pExpected are atomically compared at the byte level and are equal. Returns true when the assignment was successful. In all other cases, assigns the value loaded in *p to *pExpected, and then returns false.

This function makes weak comparisons, so sometimes even when *p and *pExpected are equal, false is returned without an assignment being made to *p. Typically this function is used in a while loop.

When you want to atomically load variables in memory p, update them with the F function and write them back. You can do this by setting appropriate values, such as those loaded with the AtomicLoad function, as the initial value of the tmp variable, and then doing something like this: while (!AtomicCompareExchangeWeak(p, &tmp, F(tmp))) {}.

Using the dedicated AtomicFetchAdd function is more efficient when doing basic arithmetic conversions atomically.
