Return Values

Description

Gets a list of friend relationships. Friend relationships are represented as a bitwise OR of attributes. Friends that have established a friend relationship with the local host at least once have the nn::friends::CTR::ATTRIBUTE_FLAG_ESTABLISHED bit set to 1. Friends that meet all of the following conditions have the nn::friends::CTR::ATTRIBUTE_FLAG_REMOTE_ACCESSIBLE bit set to 1.

A friend relationship has been established on the friend server. (It does not include the local friend state.) The friend has not deleted the friend relationship, or the deleted friend relationship has not been synchronized yet from the friend server.
The friend list on the local system does not explicitly show the user that the friend has deleted their friend relationship, and it appears to the user as if that person is always offline. The functionality is the same to the fullest extent, so developers do not need to worry about it when using the friend presence library or the NEX library. When accessing the latest data about users in friend lists on independent servers, use the nn::friends::CTR::ATTRIBUTE_FLAG_REMOTE_ACCESSIBLE bit to determine whether the latest friend relationships can be evaluated only locally, and it is difficult to hide the deletion of friend relationships. pAttributeFlagsListSpecifies a pointer to the buffer that stores obtained relationship information. pFriendKeyListSpecifies a pointer to a list of friend keys. sizeSpecifies the number of buffer elements in the friend key list. Returns the result of the operation. Result::IsSuccessIndicates success. ResultInvalidPointerAn invalid pointer was passed as an argument. ResultTooLargeThe value passed as the argument is too large. ResultNotInitializedThe friend presence library is not initialized. OtherOther results can be returned depending on the internal state of the daemon.
