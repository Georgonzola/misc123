nn::friends::CTR::GetFriendScreenName Function

Syntax

Result GetFriendScreenName(
     char16 (*pScreenNameList)[SCREEN_NAME_SIZE],
     const FriendKey * pFriendKeyList,
     size_t size = 1,
     bool replaceForeignCharacters = true,
     u8 * pFontRegionList = NULL
);

Parameters

Name Description
out pScreenNameList[SCREEN_NAME_SIZE] Specifies a pointer to a buffer that stores acquired screen names.
in pFriendKeyList Specifies a pointer to a list of friend keys.
in size Specifies the number of buffer elements in the friend key list.
in replaceForeignCharacters Specifies whether to use questions marks (?) in place of non-ASCII characters in the screen names of friends from different font regions.
in pFontRegionList Specifies a pointer to a buffer that stores the font region of a friend's screen name. Specify NULL if not needed.

Return Values

Returns the result of the operation.

Value Description
Result::IsSuccess The process succeeded.
ResultInvalidPointer Invalid pointer passed as an argument.
ResultTooLarge The value passed as the argument is too large.
ResultNotInitialized The friend presence library is not initialized.
Other Another result may be returned, depending on the internal state of the daemon and other factors.

Description

Gets a list of friend screen names.

By calling this function, you can get the name of a friend's Mii directly without using the Mii library. Even if you have not yet established a friend relationship with this person and cannot get his or her Mii, the display name that you have tentatively registered is obtained.

The actual name of a friend's Mii is obtained even when it is in the list of blocked users, but the obtained name is replaced with "???" if it contains profanity.

Returns an empty display name in this buffer if there is no friend corresponding to the key passed in the arguments. In this case the function still returns success.

Some Mii nicknames might contain characters that are not supported by the application. Make sure that the presence of these unsupported characters does not corrupt the screen or impede progress in the application. For workarounds, see the section about creating Mii nicknames and creator names in 3DS Guidelines.
