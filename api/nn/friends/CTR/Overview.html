nn::friends::CTR Namespace

Description

Namespace of the CTRfriends library. (It can also be written without the CTR portion.)

The success and failure values returned by functions in the namespace may be expanded. Unless you are handling specific results, capture the entire success or failure using the nn::Result::IsSuccess or nn::Result::IsFailure function.

The value stored in the output parameter when a function fails is not fixed.

The friend library must be initialized to make most function calls (excluding calls to functions such as nn::friends::CTR::Initialize, nn::friends::CTR::Finalize, and nn::friends::CTR::IsInitialized). In non-release builds, the library stops on an assertion if you attempt to call functions that require library initialization without first initializing the library.

typedef Definitions

PrincipalId Represents a principal ID.
LocalFriendCode Represents a local friend code.
FriendKey Structure for comprehensively handling the principal IDs and local friend codes of users.
Profile Structure storing the profile information of a user.
MiiData Structure storing Mii data.
GameMode Structure that stores information about the game the local host is currently playing. This information is required in order for friends to join in.
MyPresence Structure storing the local user's presence information.
FriendPresence Structure storing friend presence information.
EventNotification Structure storing notifications of changes in the online states of the local host and friends.
ApproachContext This structure contains the friend presence account information of the current user, to enable other users to register the user as a friend.

GetFriendPresence Gets a list of friend presence information. Cannot acquire the invitation flag and game mode of a friend who has a different matchmaking game ID specified.

Returns an empty presence in this buffer if there is no friend corresponding to the key passed in the arguments. The function still returns success in this case.

The function retrieves an empty presence when a non-existent friend is specified. This state is represented by a nn::friends::CTR::FriendPresence structure that sets isValid to false. If the friend is offline, or has made the online status private, the obtained presence sets isValid to true, but isOnline is false, and the gameMode structure is filled with zeros.

GetFriendAttributeFlags Gets a list of friend relationships. Friend relationships are represented as a bitwise OR of attributes. Friends that have established a friend relationship with the local host at least once have the nn::friends::CTR::ATTRIBUTE_FLAG_ESTABLISHED bit set to 1. Friends that meet all of the following conditions have the nn::friends::CTR::ATTRIBUTE_FLAG_REMOTE_ACCESSIBLE bit set to 1.
