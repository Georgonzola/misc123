nn::fs::CommitSaveData Function

Syntax

Result CommitSaveData(
     const char * archiveName = "data:"
);

Parameters

Name Description
in archiveName The name of the archive. (If this argument is omitted, data: is specified.)

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultNotFound The specified archive has not been mounted.
You must not allow this error to occur in retail products.
Other An application-side bug, or an unexpected error.

Description

Commits a change to the save data. Commits the save data region that was mounted using the specified archive name, and thereby finalizes changes to save data.
If the archive is unmounted without calling this function, or the program stops, all changes to the save data since the previous commit are rolled back. If this function returns successfully, it is guaranteed that changes to the save data were made correctly. If the program is interrupted while this function is running, there are only two possibilities: either the save data was rolled back to the previous commit, or all changes were applied. There is no chance of corruption or the data being only partially written.
