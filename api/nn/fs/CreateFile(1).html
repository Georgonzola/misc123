nn::fs::CreateFile Function

Syntax

void CreateFile(
     const char * pathName,
     s64 size
);

Parameters

Name Description
in pathName Path of the file to create.
in size Size of the file to create.

Return Values

None.

Description

Creates a file.

Note: This function uses about 1 KB of buffer space on the stack because it converts the path name from a multibyte string into a wide string. Pay careful attention to the stack size.

This function is implemented to show an error screen when errors occur internally, regardless of whether such errors are fatal. Control does not return from the function in this case. Use nn::fs::TryCreateFile on everything other than ROM archives.

For more information about error handling, see Handling Errors During File and Directory Operations.

Important

After the size has been extended with this function, if nothing has been written to that region, in principle the process of reading that region will return nn::fs::ResultVerificationFailed. However, if this function is called for save data or expanded save data, the first 512 bytes (or the entire file if the size is 512 bytes or smaller) are guaranteed to be zero-initialized. By using this function and writing to the file in the order Create file > Write data and flush > Write file signature to front and flush, you can check that the file signature has been loaded correctly and guarantee that the file is written correctly.
