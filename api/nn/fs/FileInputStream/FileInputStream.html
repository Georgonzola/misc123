Description of FileInputStream ( )

In the overloaded version without arguments, no file is opened. You must call the TryInitialize function separately to manipulate files.

In the overloaded version with the path name specified, it opens the file at the specified path with read-only access and enables reading.

Description of FileInputStream ( const wchar_t * )

It opens the file at the specified path with read-only access and enables reading.

Description of FileInputStream ( const char * )

It opens the file at the specified path with read-only access and enables reading.

Note: This function uses about 1 KB of buffer space on the stack because it converts the path name from a multibyte string into a wide string. Pay careful attention to the stack size.
