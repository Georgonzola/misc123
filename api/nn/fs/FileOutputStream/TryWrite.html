nn::fs::FileOutputStream::TryWrite Member Function

Syntax

virtual Result TryWrite(
     s32 * pOut,
     const void * buffer,
     size_t size,
     bool flush
);

Parameters

Name Description
out pOut Stores the actual number of bytes that were written.
in buffer Pointer to the buffer that contains the data to write.
in size Maximum number of bytes to write to the file.
in flush Specifies whether to flush data.

Return Values

Returns the execution result.

Description

Writes the specified amount of data from a buffer to a file.

Up to size bytes of data are written to the file from the region at the address specified by buffer.

Data is not flushed if false is specified for flush. If you are making multiple consecutive write operations to the same file, call the TryFlush function after all operations are complete with flush = false. Then create a buffer to be managed by the system in low-level storage.

Do not write-access the memory of the specified buffer region during this function call (for example, from another thread). File write operations may be incorrect and the program could stop working in a future version of the SDK.

Error handling differs, depending on the archive. For more information, see Handling Errors During File and Directory Operations.
