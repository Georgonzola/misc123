nn::fs::GetAddOnContentRequiredMemorySize Function

Syntax

Result GetAddOnContentRequiredMemorySize(
     s32 * pOut,
     bit32 uniqueId,
     u32 contentIndex,
     size_t maxFile,
     size_t maxDirectory,
     bool useCache
);

Parameters

Name Description
out pOut Returns the calculated memory size.
in uniqueId Specifies the UniqueId.
in contentIndex Specifies the ContentIndex.
in maxFile The maximum number of files that can be opened simultaneously.
in maxDirectory The maximum number of directories that can be opened simultaneously.
in useCache Set to true to enable caching of metadata to memory, or false to disable metadata caching.

Return Values

Returns the execution result.

Value Description
ResultNotFound The specified add-on content was not found.
You must not allow this error to occur in retail products.
ResultMediaNotFound An SD card could not be found or recognized.
ResultAlreadyExists The specified archive name is already mounted.
You must not allow this error to occur in retail products.
ResultWriteProtected The SD card is write-protected.
ResultMediaAccessError An error occurred while accessing the SD card because of some cause such as a loose connection.
ResultBadFormat The SD card has an invalid format. The SD card must be formatted.
ResultVerificationFailed Either verification failed or some form of tampering was detected. You must re-download add-on content.
Other An unexpected or fatal error occurred.

Description

Gets the size of memory needed to mount an add-on content archive.

Returns the size of memory needed to mount an add-on content archive.
Implicitly uses 0 as the Variation value for the add-on data.
The returned memory size allows only the number of files specified by maxFile and the number of directories specified by maxDirectory to be opened simultaneously.
By setting the useCache argument equal to true, this size includes the amount of memory required to cache the metadata of the archive in memory.

If you call this function in a state where you do not have rights to the target content index (that is, the nn::ec::ContentInfo::IsOwned function returns false), it returns a nn::fs::ResultAccessDenied-type error.
