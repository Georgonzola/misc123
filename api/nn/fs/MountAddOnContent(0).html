nn::fs::MountAddOnContent Function

Syntax

Result MountAddOnContent(
     const char * archiveName,
     bit32 uniqueId,
     bit8 variation,
     u32 contentIndex,
     size_t maxFile,
     size_t maxDirectory,
     void * workingMemory,
     size_t workingMemorySize,
     bool useCache
);

Parameters

Name Description
in archiveName Specifies the archive name.
in uniqueId Specifies the UniqueId.
in variation Specifies the Variation (overloaded version of this function implicitly specifies 0 when omitting this parameter).
in contentIndex Specifies the ContentIndex.
in maxFile The maximum number of files that can be opened simultaneously.
in maxDirectory The maximum number of directories that can be opened simultaneously.
in workingMemory Start address of memory region used for archive operation
in workingMemorySize Size of memory region used for archive operation
in useCache Specify true to enable caching of metadata to memory, or false to disable metadata caching.

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultNotFound The specified add-on content was not found.
You must not allow this error to occur in retail products.
ResultMediaNotFound An SD card could not be found or recognized.
ResultAlreadyExists The specified archive name is already mounted.
You must not allow this error to occur in retail products.
ResultNotEnoughSpace An insufficient memory size was specified to workingMemory.
You must not allow this error to occur in retail products.
ResultMediaAccessError An error occurred while accessing the SD card because of some cause such as a loose connection.
ResultBadFormat The SD card has an invalid format. The SD card must be formatted.
ResultVerificationFailed Either verification failed or some form of tampering was detected. You must re-download add-on content.
Other An unexpected or fatal error occurred.

Description

Mounts an add-on content archive.

Mounts the add-on content of the specified archive name.
The numbers of files and directories you can simultaneously open from the add-on content archive are specified by maxFile and maxDirectory, respectively.

The region passed to workingMemory must be at least the size calculated by the GetAddOnContentRequiredMemorySize function.
You can cache archive metadata to memory by setting the useCache parameter to true. This reduces the time taken to open files and scan directories. However, it also increases the amount of memory required.

Mounting an add-on content archive consumes a resource that is shared by the system as a whole.
Currently, you must have no more than two instances of add-on content mounted at one time.
