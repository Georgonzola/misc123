Description of MountAddOnContent ( const char *, bit32, bit8, u32, size_t, size_t, void *, size_t, bool )

Mounts the add-on content of the specified archive name.
The numbers of files and directories you can simultaneously open from the add-on content archive are specified by maxFile and maxDirectory, respectively.

The region passed to workingMemory must be at least the size calculated by the GetAddOnContentRequiredMemorySize function.
You can cache archive metadata to memory by setting the useCache parameter to true. This reduces the time taken to open files and scan directories. However, it also increases the amount of memory required.

Mounting an add-on content archive consumes a resource that is shared by the system as a whole.
Currently, you must have no more than two instances of add-on content mounted at one time.
