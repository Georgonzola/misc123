nn::fs::MountSaveData Function

Syntax

Result MountSaveData(
     const char * archiveName,
     bit32 uniqueId
);

Parameters

Name Description
in archiveName The archive name.
in uniqueId The unique ID of the other application.

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultAlreadyExists The specified archive name is already mounted.
You must not allow this error to occur in retail products.
ResultOperationDenied Indicates that the operation was denied because the save data is currently being mounted.
You must not allow this error to occur in retail products.
ResultNotFormatted Indicates that the data region is not formatted. The other application's format is prohibited, so the application itself must do the formatting.
ResultBadFormat Invalid format. The other application's format is prohibited, so the application itself must do the formatting.
ResultVerificationFailed Either verification failed or some form of tampering was detected. The other application's format is prohibited, so the application itself must do the formatting.
Other An application-side bug, or an unexpected error.

Description

Mounts the save data archive that is in another application.

The save data region of another application is mounted as a save data archive. The other application is specified by specifying its unique ID.

The system will search multiple storage media types when mounting, but unformatted storage media is not searched.

There is no ironclad guarantee that the save data this function is supposed to mount actually exists. For this reason, you must prepare adequate error handling.

There are restrictions on the unique IDs that this function can specify. For proper access, you must make the necessary descriptions in the RSF files of the programs that call this function.

For more information, see the ctr_makerom reference.

The following page has hints on how to handle mount errors. Please use this as a reference.

  • Handling Mount Errors: Save Data Archives
