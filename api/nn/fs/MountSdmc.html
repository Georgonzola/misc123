nn::fs::MountSdmc Function

Syntax

Result MountSdmc(
     const char * archiveName = "sdmc:"
);

Parameters

Name Description
in archiveName The archive name. (When omitted, sdmc: is specified.)

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultMediaNotFound An SD card could not be found or recognized.
ResultAlreadyExists The specified archive name is already mounted.
You must not allow this error to occur in retail products.
ResultArchiveInvalidated The SD card may have been removed during the mount operation.
ResultMediaAccessError An error occurred while accessing the SD card because of some cause such as a loose connection.
ResultBadFormat The SD card has an invalid format. The SD card must be formatted.
Other An application-side bug, or an unexpected error.

Description

Mounts an archive that directly accesses the SD card (for debugging only). Mounts the archive that directly accesses the inserted SD Memory Card. The function fails if an SD Memory Card is not inserted or the SD Memory Card is not formatted. To mount a previously mounted SD Memory Card that has been pulled out and reinserted, you must first unmount the card using the nn::fs::Unmount function, and then execute the nn::fs::MountSdmc function again.
Note: This type of archive is only for debugging and cannot be accessed from retail versions.

To use this function, you must add the - Debug attribute to the file system access permissions in your RSF file. For more information, see the ctr_makerom reference.
