Overview
Debugging can sometimes be difficult when using a non-Try type API function with a ROM archive. For example, if an error occurs, the display might forcibly change to an error display screen and you will not be able to see what was being displayed. This page gives useful information for dealing with this situation.

The following example shows results that may be displayed on the error display screen.
RSL : 0xC8804464
      Level  : -7
      Summary: 4
      Module : 17
      Desc   : 100
Module=17: Indicates a file system error. Desc: This value is used to determine which error occurred.
From the list, you can determine that the error was nn::fs::ResultNotFound.

List of Descriptions
File system errors are handled by taking multiple results together. (Reference: Handling Results)
The value of Desc has a range of meanings even if the error is of the same type.
