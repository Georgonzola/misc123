Description of CopyTexture2DResourceBlockToLinear ( const Texture2DResource *, s32, u32, s32, const Texture2DResource *, s32, u32, DownScalingMode, gdBool )

The following pixel formats can be specified for the source and destination.
NATIVE_FORMAT_RGBA_8888
NATIVE_FORMAT_RGB_888
NATIVE_FORMAT_RGBA_4444
NATIVE_FORMAT_RGBA_5551
NATIVE_FORMAT_RGB_565
The source memory layout must be either LAYOUT_BLOCK_8 or LAYOUT_BLOCK_32. The destination memory layout must be LAYOUT_LINEAR. The copied data is automatically clipped if the source memory region (including srcMipLevelIndex, srcOffsetY, and srcCountRow) is bigger than the destination region (including srcMipLevelIndex, srcOffsetY, and srcCountRow). An error occurs if the source region is smaller than the destination region. When converting the pixel format, the source pixel size must be the same size or smaller than the destination pixel size. In other words, conversions that increase the pixel size are not possible. This process calls the nngxAddB2LTransferCommand function.
This function adds commands to the command list. For more information, see the Programming Manual.
