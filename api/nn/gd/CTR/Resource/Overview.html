nn::gd::CTR::Resource Class

Syntax

class Resource

Description

A resource function.

The term resource refers to two-dimensional regions like pixels (Texture2DResource), in addition to regions like vertex buffers and index buffers (VertexBufferResource. When a resource is created, memory is allocated to the resource. You create resources using the Resource::CreateTexture2DResource function or the Resource::CreateVertexBufferResource function. When a resource is released, the allocated memory is also released.


Note: Interaction with the nngx command list

You must be careful when releasing the initialData passed as an argument to Resource::CreateTexture2DResource or Resource::CreateVertexBufferResource. If Texture2DResourceDescription.m_MemLocation is set to Memory::VRAMA or Memory::VRAMB, you must be careful when releasing initialData. (You cannot release it immediately after the creation of the resource.) If Texture2DResourceDescription.m_MemLocation is set to a VRAM location, DMA transfer requests for the initialData data accumulate in the command buffer. A copy of the actual data is used when commands execute. For this reason, the user application must retain initialData until the commands have finished executing.
Note: If initialData is created in Memory::FCRAM, data is transferred using the memcpy function, so it is okay to release initialData immediately after creating the resource.


The Texture2DResource case

You cannot directly bind a Texture2DResource to the pipeline stages. To use a Texture2DResource on one of the pipeline stage, you must create one of the following objects from it.

a Texture2D using TextureStage::CreateTexture2D
a TextureCube using TextureStage::CreateTextureCube
a RenderTarget using OutputStage::CreateRenderTarget
a DepthStencilTarget using OutputStage::CreateDepthStencilTarget
This mechanism allows the user to have different ways of using the same resource. For example, after you have bound a Texture2DResource resource to a RenderTarget and completed the render task, you could use the result as a 2D texture.
You can use the following functions to bind a Texture2DResource to the pipeline stages.

TextureStage::SetTexture(TextureUnitId, Texture2D &texture2D)
TextureStage::SetTextureUnit0(TextureCube &textureCube)
OutputStage::SetRenderTarget(const RenderTarget *renderTarget)
OutputStage::SetDepthStencilTarget(const DepthStencilTarget *depthStencil)


The VertexBufferResource case

As for the VertexBufferResource, because there is only one way of using each type of vertex buffer, you can directly bind it to the VertexInputStage.
The following functions can bind a VertexBufferResource to the pipeline stages.

VertexInputStage::SetIndexBuffer(VertexBufferResource *indexBuffer, IndexFormat format, u32 offset)
VertexInputStage::SetVertexBuffers(u32 startSlot, u32 numBuffers, VertexBufferResource **vertexBuffers, u32 *offsets)
