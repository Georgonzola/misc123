nn::gd::CTR::ShaderStage::CreateShaderPipeline Member Function

Syntax

static nnResult CreateShaderPipeline(
     Shader * vertexShader,
     Shader * geometryShader,
     ShaderPipeline ** shaderPipeline,
     ShaderPipelineUnmanagedRegisters * unmanagedRegister = NULL
);

Parameters

Name Description
in vertexShader Specifies a pointer to the vertex shader object to attach.
in geometryShader Specifies a pointer to the geometry shader object to attach. Specify NULL if not used.
out shaderPipeline Specifies the address storing the created shader pipeline pointer.
in unmanagedRegister The specified register command is not generated. Directly output by the SetFloatConstantBuffer function.

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultNullParameter NULL was specified for vertexShader or shaderPipeline.
ResultOutOfMemory Could not allocate memory.
ResultInvalidShaderSignature The geometry object is not correct. There were 96 or more floating point constant registers.

Description

Creates a shader pipeline with an attached shader object.

By default, after a new shader or the current shader is updated, subsequent rendering functions automatically create (recreate) commands for some or all shader variables. In several cases, you can improve performance by directly generating shader variable commands rather than automatically creating them. In such cases, you can define shader variables and then create direct commands as unmanaged variables to directly output.
