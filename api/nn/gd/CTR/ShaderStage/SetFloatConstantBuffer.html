nn::gd::CTR::ShaderStage::SetFloatConstantBuffer Member Function

Syntax

static nnResult SetFloatConstantBuffer(
     ShaderType shaderType,
     u32 firstRegisterIndex,
     u32 registerCount,
     f32 * constantBufferSrc
);

Parameters

Name Description
in shaderType Specifies the shader type. (SHADER_TYPE_VERTEX or SHADER_TYPE_GEOMETRY)
in firstRegisterIndex Specifies the index of the start register. (For example, if 3 is specified for the index, it starts from register c3.)
in registerCount Specifies the register count to be set.
in constantBufferSrc Specifies a pointer to the fixed point constant data to be set.

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultNullParameter NULL was specified for constantBufferSrc.
ResultOutOfRangeParameter registerCount was incorrect.

Description

Immediately creates a floating point constant register command in the command buffer.

In some cases, performance may be increased by directly generating shader variable commands. To use this function, you must first use the CreateShaderPipeline function to specify unmanaged shader variables. (Shader variables that are not set for direct output are automatically re-uploaded in render functions when updates or shader settings are made. )
The following command formats can be entered into the command buffer by the SetFloatConstantBuffer function.
0x00000000 [Reserved]
0x00000004 [Reserved]
0x00000008 constantBufferSrc[3]
0x0000000c [Reserved]
0x00000010 constantBufferSrc[2]
0x00000014 constantBufferSrc[1]
0x00000018 constantBufferSrc[0]
0x0000001c constantBufferSrc[7]
0x00000020 constantBufferSrc[6]
0x00000024 constantBufferSrc[5]
0x00000028 constantBufferSrc[4]
0x0000002c constantBufferSrc[11]
0x00000030 constantBufferSrc[10]
0x00000034 constantBufferSrc[9]
0x00000038 constantBufferSrc[8]
0x0000003c ...
0x00000040 ..
0x00000044 .
