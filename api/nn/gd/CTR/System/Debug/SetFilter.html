nn::gd::CTR::System::Debug::SetFilter Member Function

Syntax

static nnResult SetFilter(
     u32 filterFlag,
     u32 drawStartIndex,
     s32 drawCount
);

Parameters

Name Description
in filterFlag Specifies the status or effect to be changed.
in drawStartIndex Specifies the number of draw executions to put on standby before enabling the effect. If 0 is specified for drawStartIndex, the effect is enabled the next time a draw is executed.
in drawCount Specifies the number of draw executions at which to enable the effect.

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates that the operation succeeded.

Description

Sets a filter capable of making several status changes upon the next draw. The filterFlag parameter can be specified by using an OR operation on the System::Filter enum members.

This function can be used when the compile mode is Debug or Development. It does nothing for Release builds. This function has been designed so that the user can test performance, the state of the visual image, and feature effects. The state and features specified by the filterFlag parameter can be switched to beginning after the drawing specified by drawStartIndex for the number of drawings specified by drawCount. The gd API automatically restores the user-defined state after the effect is no longer changed.
