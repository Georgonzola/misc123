nn::gr::CTR::MakePostVertexCacheEntryTypeCommand Function

Syntax

bit32 * MakePostVertexCacheEntryTypeCommand(
     bit32 * command,
     const PostVertexCacheEntryType type
);

Parameters

Name Description
in command The start address for storing the rendering command.
in type Specifies the number of entries to change.

Return Values

Returns the address that follows the end of the stored rendering command.

Description

Changes the post-vertex cache to the specified number of entries.

This generates a command to change the number of entries in the post-vertex cache to 32 or 16.
By default, it is set to 32.

The advantage of changing the number of entries to 16 is that doing so allows you to increase the maximum number of input vertex attributes to the geometry shader to 15.
Decreasing the number of entries also lowers the cache hit rate and therefore increases the cost of processing on the GPU.
Unless you need to do this, though, we recommend that you use 32 entries.
