nn::gx::CTR::UpdateBufferLight Function

Syntax

void UpdateBufferLight(
     const void * pBuffer,
     size_t size
);

Parameters

Name Description
in pBuffer Pointer to the start of the buffer.
in size The buffer size.

Return Values

None.

Description

Applies to the graphics processor the updates that have been made to the contents of the vertex buffer and texture buffer in main memory.

When the GPU directly references texture, vertex, and other buffer data that has been expanded, copied, or overwritten into device memory by the CPU, use this function to reflect those CPU-side changes in the GPU. This function does not invalidate the CPU cache.

Specifically, internal processing consists of storing the cache, but Nintendo does not support using this function as a generic means of storing the cache.

Only use this function to reflect CPU changes to the vertex and texture buffers in the GPU.

This function does not invalidate the CPU cache like the nn::gx::CTR::UpdateBuffer function, but it is faster. We recommend using this function to apply dynamic buffer changes by the CPU to the GPU. Use the nn::gx::CTR::UpdateBuffer function only when the cache must be invalidated, such as when the CPU accesses the GPU rendering results.

This function blocks so that it can call system processes. There is a fixed processing overhead for each process call. To improve performance, call this function as infrequently as possible. In other words, try to collect updates into a large batch before calling it, rather than breaking the updates into small batches.
