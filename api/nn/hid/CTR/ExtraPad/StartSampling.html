nn::hid::CTR::ExtraPad::StartSampling Member Function

Syntax

static nn::Result StartSampling(
     s32 samplingThreadPriority,
     s32 period
);

Parameters

Name Description
in samplingThreadPriority Sets the priority for the sampling thread. The higher the priority, the more stable you can expect the sampling to be.
in period Sets the sampling period. Set a value in the range from 8 to 32 milliseconds. The shorter the sampling period, the higher the load.

Return Values

Returns the execution result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Process succeeded and sampling of the Nintendo 3DS Circle Pad Pro has started.
MakeResultNoConnection The Nintendo 3DS Circle Pad Pro could not be found.
MakeResultInvalidCombination Inappropriate argument.
MakeResultNotInitialized Not initialized.

Description

Starts sampling the Nintendo 3DS Circle Pad Pro. When this function succeeds, a thread for sampling is created internally using application resources.

It takes approximately 50 to 200 milliseconds for the Circle Pad Pro to be connected successfully or up to approximately 100 milliseconds for the connection to fail. If this function is called when sampling has already started, it disconnects and then connects The sampling thread is released by a call to the StopSampling function.

On SNAKE, connection is always successful.
The sampling frequency for the C Stick, ZL Button, and ZR Button can be set to any value from 10 to 21 milliseconds. It is technically possible to set the sampling frequency to a value from 8 to 9 or 22 to 32 milliseconds, but the function gets the values at a slightly different time from when they are actually sampled on the C Stick. For this reason, set the sampling frequency to a value from 10 to 21 milliseconds.
