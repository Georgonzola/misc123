nn::http::Initialize Function

Syntax

Result Initialize(
     uptr bufferAddress = 0,
     size_t bufferSize = 0
);

Parameters

Name Description
in bufferAddress Pointer to the start of the POST data buffer. Memory for the buffer must be BUFFER_ALIGNMENT (4096-byte) aligned.
in bufferSize Size of the POST data buffer.

Return Values

Returns the execution result of the function. Returns one of the following Result values.


Value Description
Result::IsSuccess Indicates that the operation succeeded. The HTTP library can be used.
ResultClientProcessMaxErr Error indicating that the maximum number of simultaneous client processes are already in use. Trying again later may succeed (if another module calls Finalize in the meantime).
ResultIpcSessionMaxErr Error indicating that the maximum number of simultaneous IPC sessions are already connected. Trying again later may succeed (if another IPC session succeeds in the meantime).
ResultAlreadyInitializedErr Error indicating that initialization is already complete. The HTTP library can already be used.
Other Indicates an unexpected error. (For more information, see http_Result.h.)

Description

Initializes the HTTP library. Processes that link against the HTTP library must call this before using the library.

You must use this function's arguments to set the POST data buffer before you can use the AddPostData-related functions of nn::http::Connection (the functions that register POST data before Connect runs).
If you are not using the AddPostData-related functions you do not need to set anything in the argument. (In other words, even if you are using POST operations, nothing needs to be set in the argument if you use only SendPostData-related functions.)

The POST data buffer is used to temporarily store the registered POST data.
For this reason, the buffer size must be at least as large as the maximum total amount of POST data that is registered at any one time.
(The POST data registered in nn::http::Connection is stored temporarily in the POST data buffer, sent via POST when nn::http::Connection::Connect executes, and freed when nn::http::Connection::Finalize executes. Accordingly, the size of the POST data buffer must be at least as large as the maximum total amount of POST data registered to the nn::http::Connection object at any one time before Finalize.)
