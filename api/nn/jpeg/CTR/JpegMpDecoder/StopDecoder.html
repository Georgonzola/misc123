nn::jpeg::CTR::JpegMpDecoder::StopDecoder Member Function

Syntax

#include <nn/jpeg.h>
void StopDecoder();

Parameters

None.

Return Values

None.

Description

Requests to stop decoding that is being performed by another thread.

This function sets a "stop request flag" inside the decoder object. The decoding functions (StartJpegDecoder, StartJpegDecoderShrink and StartMpDecoderLR) monitor this flag and stop decoding when the flag is set.

Because decoder objects are not thread-safe, as a basic rule multiple threads cannot access the same decoder object.

However, during the periods of time when the application can check whether the decoder object is valid (meaning the object has been initialized and has not ended), this function can be called on the decoder object from a different thread.

Generally it takes some time to decode a JPEG image. Applications sometimes use a separate thread for decoding to maintain the response speed of the main thread.

For example, say you have a screen that shows a list of images, and you want to stop the decoding that has already begun for a certain image. To do this, you can call this function from a different thread (the main thread or an alarm handler) that has a higher priority than the decoding thread.

The stop request flag is cleared immediately before the decoding functions begin their main processing. The following conditions apply, so there is no certainty that simply calling this function once will stop ongoing decoding.

1. Decoding cannot be stopped before the decoding thread calls a decoding function or before control reaches the main decoding process within the function.
2. After main processing within decoding has completed, execution cannot be stopped. (Decoding succeeds unless another error occurs.)
3. StartMpDecoderLR clears flags each time two images are decoding. Decoding cannot be stopped during the period after the first image finishes decoding and before the second image begins decoding.

To deal with situations 1 and 3, you must continue to call this function (for example, in every game frame) until you have verified that decoding has been stopped, or has ended, whether in success or failure.

If a request to halt a decoding function is received, the function returns 0, as it does when decoding fails. (The error code returned by GetLastError is JPEG_DECODER_ERROR_STOPPED unless another error occurs.)

Stop requests cannot be canceled. After decoding has stopped, it cannot be resumed from the same point at which it stopped.
