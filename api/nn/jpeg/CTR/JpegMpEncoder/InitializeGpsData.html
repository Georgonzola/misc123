nn::jpeg::CTR::JpegMpEncoder::InitializeGpsData Member Function

Syntax

#include <nn/jpeg.h>
static void InitializeGpsData(
     GpsData * pGps
);

Parameters

Name Description
out pGps Specifies the GPS IFD (the structure used for storing GPS data).
The stored data format is easy for the library to process and is different from the binary format that results from encoding.

Return Values

None.

Description

Initializes the structure used for registering GPS IFD data.

This section describes the procedure for storing GPS information in the image and storing it.

Have the application maintain the GPS IFD structure for storing information and initialize it with this function. Call the SetGpsLatitude and SetGpsLongitude functions to store GPS information in the structure.
Depending on the function, some structure members, such as SetGpsSatellites, only maintain a pointer to the data to store. In this case, you must maintain the data to store until encoding ends. After all data has been stored in the structure, be sure to confirm the data by calling the SetGpsData function.
The encoder object only maintains a pointer to this structure. This structure must be maintained until encoding ends. Call the encoding function and encode the image. Whether it succeeds, after the encoding function exits, the structure and data to store can be released.
To reuse this structure, be sure to reinitialize it with this function.

After initialization, the structure only stores the GPS tag version. (Version 2.2 is the default for this library.)
To change the version, call the SetGpsVersionId function. To instruct the system not to save the version, call the ClearGpsVersionId function.
