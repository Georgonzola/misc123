nn::jpeg::CTR::JpegMpEncoder::SetMpTypeFlags Member Function

Syntax

#include <nn/jpeg.h>
void SetMpTypeFlags(
     u32 flags = 0,
     u16 image1 = 0,
     u16 image2 = 0
);

Parameters

Name Description
in flags Specify either a single MpTypeFlag or a bitwise OR of them. It is also possible to specify 0.
If a representative image already exists, MP_TYPE_FLAG_REPRESENTATIVE_IMAGE (the representative image flag) is ignored even if it is set.
The default is 0.
in image1 Specifies the entry number for dependent image 1. The default is 0.
in image2 Specifies the entry number for dependent image 2. The default is 0.

Return Values

None.

Description

Registers values to embed in the MP entry: the various flags for the individual image type management data and the dependent image entry numbers.

This function is enabled if the StartMpEncoderFirst or StartMpEncoderNext function is used as the encoding function. It is invalid when using StartMpEncoderLR. (The default value is registered.)

Of the individual image type management information fields, you can specify the dependent parent image flag, the dependent child image flag, and the representative image flag.
Of the other fields, the reserved (0) and data format (0: JPEG) fields are fixed by the library and cannot be specified.
The type code is specified by the StartMpEncoderFirst function.

You can skip this function and still use the StartMpEncoderFirst and StartMpEncoderNext functions without any problems. By default the dependent parent image flag and dependent child image flag are cleared to 0, with the representative image flag set to 1 for the starting image only and cleared to 0 for all others.

In more detail, the default functionality is as follows.

If you skip this function and use StartMpEncoderFirst, this image (the starting image) becomes the representative image (meaning that its representative image flag is set to 1). If you skip this function and use StartMpEncoderNext, and if no representative image yet exists, this image becomes the representative image.
If a representative image does exist, the representative image flag for this image is cleared to 0.

Call this function before calling the encoding function. The values specified by this function are cleared after the encoding function completes, regardless of whether the encoding function succeeded or failed. When encoding multiple times, you must call this function before each encoding function call. The values specified by this function are cleared if you reinitialize the encoder object using the Initialize function.
