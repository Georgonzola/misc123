nn::jpeg::CTR::JpegMpEncoder::StartMpEncoderFirst Member Function

Syntax

#include <nn/jpeg.h>
size_t StartMpEncoderFirst(
     u8 * dst,
     size_t limit,
     const void * src,
     u32 width,
     u32 height,
     u32 quality,
     PixelSampling dstPixelSampling,
     PixelFormat srcPixelFormat,
     bool addThumbnail,
     u32 numImages = 2,
     MpTypeCode typeCode = MP_TYPE_CODE_MULTI_VIEW_DISPARITY_IMAGE,
     bool addImageUidList = false,
     bool addTotalFrames = false
);

Parameters

Name Description
out dst Specifies the buffer that stores the encoded result.
Do not destroy this buffer until after encoding completes for the specified total number of images. (You can destroy the buffer if encoding has been canceled.)
in limit Specifies the size of dst in bytes. Encoding fails if this size is exceeded.
If StartMpEncoderNext fails during the encoding of the specified total number of images, it might be because this byte size was insufficient.
in src Specifies the buffer of image data to encode. Must be aligned to SRC_ALIGNMENT (4 bytes).
in width Specifies the image width in pixels (equal to or less than 65,536).
in height Specifies the image height in pixels (equal to or less than 65,536).
in quality Specifies the encoding quality.
Values can be set from 1-100, with 100 being the highest quality and the largest size.
in dstPixelSampling Specifies the image's output format (its pixel sampling).
in srcPixelFormat Specifies the input pixel format of the image to encode.
in addThumbnail Specifies whether to add a thumbnail.
in numImages Specifies the total number of images included in one MPO.
If the MP type (given by the typeCode argument) is a stereoscopic image, this parameter must be set to at least 2.
For Baseline MP primary images, this parameter must be set to no more than 3.
The default value is 2.
in typeCode Specifies the MP type.
For the values that can be specified, see Description.
The default is MP_TYPE_CODE_MULTI_VIEW_DISPARITY_IMAGE (stereoscopic image).
in addImageUidList Specifies whether to append a unique ID list for individual images.
Specify true to append it, or false to not append it.
The appended values are the unique image IDs specified by the SetImageUid function immediately preceding the call to this function or the StartMpEncoderNext function.
The value is NULL (all bytes cleared to 0) if SetImageUid has not been called or if the image is the starting image.
The default is false.
in addTotalFrames Specifies whether to append the total number of frames taken.
Specify true to append it, or false to not append it.
When the MP type (the value of the typeCode parameter) is MP_TYPE_CODE_BASELINE_MP_PRIMARY_IMAGE, this specification is ignored and the total number of frames taken is not appended.
The appended value is usually the total number of images, but if 0 is specified as the individual image number by SetMpIndividualNum immediately before the call to this function and to StartMpEncoderNext, that image is not counted.
Starting from the second image, images of MP type MP_TYPE_CODE_LARGE_THUMBNAIL_IMAGE_CLASS_1 or MP_TYPE_CODE_LARGE_THUMBNAIL_IMAGE_CLASS_2 (images for monitor display) are not counted either.
The default is false.

Return Values

If the call is successful, the function returns the number of bytes in the generated MPO. (However, if the total number of images is greater than 1, the MPO is not complete.)
Returns 0 if unsuccessful. Use the GetLastError function to get the reason for failure.

Description

JPEG-encodes an image and stores it as the starting image in MP format.

With this function, you specify the total number of images in one set of MP format data (a multiple-picture object, or MPO) and encode the first image. Call the StartMpEncoderNext function to encode the second image and all subsequent images up to the total.

The encoding results of subsequent runs of StartMpEncoderNext overwrite the APP2 (including MP index IFD, MP Entry, and MP Attribute IFD) of the first image, encoded by this function. If the process fails while encoding the specified total number of images, or if encoding stops, you must start the encoding again from the first image.

Unlike with StartMpEncoderLR, you can save various types of data for every image by calling functions like SetUserMakerNote, SetMpTypeFlags, and SetDateTime. To register such data to the second or subsequent images, you must call those functions before calling the StartMpEncoderNext function.

The SetThumbnailSize and SetInputBufferWidth function arguments can also be specified separately for each image. To specify these values for the second or subsequent images, you must call those functions before calling the StartMpEncoderNext function.

The widths and heights of images that this function can encode depend on the output format of the image (in other words, the dstPixelSampling parameter). The same is true of the StartJpegEncoder function.

Values that can specified for the typeCode parameter:
You can specify any of the following values.MP_TYPE_CODE_MULTI_VIEW_DISPARITY_IMAGE (stereoscopic image) MP_TYPE_CODE_MULTI_VIEW_PANORAMA_IMAGE (panoramic image) MP_TYPE_CODE_MULTI_VIEW_MULTI_ANGLE_IMAGE (multi-angle image) MP_TYPE_CODE_UNDEFINED (undefined type) MP_TYPE_CODE_BASELINE_MP_PRIMARY_IMAGE (Baseline MP primary image) Specify MP_TYPE_CODE_BASELINE_MP_PRIMARY_IMAGE (Baseline MP primary image) to use the Baseline MP primary image format.
Specify any other value to use the Extended MP format.

MP Attribute IFDs are not stored in the Baseline MP format, so the following functions have no effect.
