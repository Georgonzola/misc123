nn::jpeg::CTR::JpegMpEncoder::StartMpEncoderNext Member Function

Syntax

#include <nn/jpeg.h>
size_t StartMpEncoderNext(
     const void * src,
     u32 width,
     u32 height,
     u32 quality,
     PixelSampling dstPixelSampling,
     PixelFormat srcPixelFormat,
     bool addThumbnail,
     MpTypeCode typeCode = MP_TYPE_CODE_MULTI_VIEW_DISPARITY_IMAGE,
     bool omitPixelDimensions = false
);

Parameters

Name Description
in src Specifies the buffer of image data to encode. Must be aligned to SRC_ALIGNMENT (4 bytes).
in width Specifies the image width in pixels (equal to or less than 65,536).
in height Specifies the image height in pixels (equal to or less than 65,536).
in quality Specifies the encoding quality.
Values can be set from 1-100, with 100 being the highest quality and the largest size.
in dstPixelSampling Specifies the image's output format (its pixel sampling).
in srcPixelFormat Specifies the input pixel format of the image to encode.
in addThumbnail Specifies whether to add a thumbnail.
in typeCode Specifies the MP type.
For the values that can be specified, see Description.
The default is MP_TYPE_CODE_MULTI_VIEW_DISPARITY_IMAGE (stereoscopic image).
in omitPixelDimensions If the MP type (the typeCode parameter) is MP_TYPE_CODE_LARGE_THUMBNAIL_IMAGE_CLASS_1 or MP_TYPE_CODE_LARGE_THUMBNAIL_IMAGE_CLASS_2 (the types for images for display on monitors), this parameter specifies whether to omit registering the actual image size (PixelXDimension, PixelYDimension) to APP1.
This specification is ignored for other MP types, because the size must always be recorded for those types.
Specify true to omit recording it, or false to record it.
The default is false.

Return Values

If the call is successful, the function returns the number of bytes of the generated MP. (However, this MPO is not complete until the call has succeeded the required number of times.)
Returns 0 if unsuccessful. Use the GetLastError function to get the reason for failure.

Description

JPEG-encodes an image and appends it to the encoded results from the immediately previous call to either StartMpEncoderFirst or this function.

If processing fails, this function also fails. (You must re-encode from the first image.)

If there is one less image than the total number specified by StartMpEncoderFirst, you must call this function (the total number - 1) times to complete the MPO (multiple-picture object). Calling more than the required number of times results in failure. If the function fails before encoding is complete or if encoding is canceled, there is no further need to call this function.

If the encoder object is initialized during this process, the encoding result cannot be appended, so do not call Initialize before you call this function.

The encoding result of this function is appended to the previous image encoding result, inside the range of the buffer and byte size specified by StartMpEncoderFirst. The data is padded so that the byte offset from the start of the buffer to the location where data is appended is a multiple of 2. Consequently, sometimes 1 byte of zeroes is inserted between the EOI marker of the previous individual image and the SOI marker for this individual image. No padding is added after appending.

The data registered by functions like SetUserMakerNote, SetMpTypeFlags, and SetDateTime and the specifications for SetThumbnailSize and SetInputBufferWidth are cleared after the function ends, whether or not it is successful. When encoding multiple times, you must call these functions before encoding each image.

When the starting image MP type is MP_TYPE_CODE_BASELINE_MP_PRIMARY_IMAGE, the baseline MP format is used and the MP Attribute IFD is not stored. Just as with StartMpEncoderFirst, calls to functions that register the MP Attribute IFD have no effect.

Values that can specified for the typeCode parameter:
You can specify any of the following values.MP_TYPE_CODE_MULTI_VIEW_DISPARITY_IMAGE (stereoscopic image) MP_TYPE_CODE_MULTI_VIEW_PANORAMA_IMAGE (panoramic image) MP_TYPE_CODE_MULTI_VIEW_MULTI_ANGLE_IMAGE (multi-angle image) MP_TYPE_CODE_UNDEFINED (undefined type) MP_TYPE_CODE_LARGE_THUMBNAIL_IMAGE_CLASS_1 and MP_TYPE_CODE_LARGE_THUMBNAIL_IMAGE_CLASS_2 (image for monitor display) If something other than an image for monitor display is specified, it must be of the same MP type as the first image.

When encoding an image for monitor display, only the following tags are recorded in the APP1.
You cannot use the GetMpRegionsToBuildJpegData function to rebuild JPEG data right after encoding images for monitor display.

IFD0 Exif IFD Pointer Maker Note of the EXIF Private Tag (when storing manufacturer notes) PixelXDimension (*) PixelYDimension (*) ImageUniqueID (when storing unique image IDs) IFD1 JPEGInterchangeFormat (when attaching thumbnails) JPEGInterchangeFormatLength (when attaching thumbnails)

(*) The MP format standard recommends that these tags not be recorded if the images are the same as the original monitor images. The application must check whether the size of the images for encoding (the width and height parameters) are the same as the original monitor images. They are not recorded if the omitPixelDimensions parameter is set to true.
