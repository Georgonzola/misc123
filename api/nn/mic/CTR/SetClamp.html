nn::mic::CTR::SetClamp Function

Syntax

Result SetClamp(
     bool enable
);

Parameters

Name Description
in enable When true, clamping is enabled. (The default is true.)

Return Values

Returns the execution result.

Value Description
Result::IsSuccess Indicates success.

Description

Configures whether microphone data is clamped.

The ideal is for the full range of microphone data input to be accepted, but in actual practice variations in microphones occur.
A microphone's guaranteed input range is defined as the range for which input is guaranteed, regardless of the microphone.

When this function is called with a true argument, clamping is performed for the value of the microphone's guaranteed input range.
Always use clamping to avoid problems that could be caused by differences in microphones.

When a broader range is necessary, you can disable clamping.
Note, however, that , you might not be able to get values outside the microphone's guaranteed input range.

For the microphone's guaranteed input range based on the sampling type, see the following.
nn::mic::CTR::TYPE_8BIT_GUARANTEED_INPUT_MIN
nn::mic::CTR::TYPE_8BIT_GUARANTEED_INPUT_MAX
nn::mic::CTR::TYPE_16BIT_GUARANTEED_INPUT_MIN
nn::mic::CTR::TYPE_16BIT_GUARANTEED_INPUT_MAX
nn::mic::CTR::TYPE_SIGNED_8BIT_GUARANTEED_INPUT_MIN
nn::mic::CTR::TYPE_SIGNED_8BIT_GUARANTEED_INPUT_MAX
nn::mic::CTR::TYPE_SIGNED_16BIT_GUARANTEED_INPUT_MIN
nn::mic::CTR::TYPE_SIGNED_16BIT_GUARANTEED_INPUT_MAX

If sampling is already taking place when this function is called, the setting gets applied from after the current position of writing.
