nn::nfp::CTR::CreateApplicationArea Function

Syntax

nn::Result CreateApplicationArea(
     const ApplicationAreaCreateInfo & createInfo
);

Parameters

Name Description
in createInfo Initialization information.

Return Values



Value Description
ResultSuccess Creation succeeded.
ResultInvalidArgument Invalid argument.
ResultInvalidPointer An invalid pointer to the parameter was specified.
ResultInvalidOperation The library is not in the correct state.
ResultSleep The system is in a sleep state.
ResultWifiOff The Wi-Fi switch is set to off.
ResultNotSupported Non-NFP tag, or invalid tag format. Check the tag being used.
ResultAlreadyCreated The application area has already been created.
ResultTagNotFound Target tag not found. The tag may have been replaced with another tag during the write process.
ResultNeedRetry The writing process failed. Retrying may succeed.
ResultOperationFailed An error occurred while writing to the tag. The tag data may have been corrupted.
ResultConnectCanceled Lost the connection with the NFC reader and writer due to an internal error.
Other Unexpected error.

Description

Creates the application area on a tag. The application area must be created before using a blank tag for the first time.

Before calling this function, the tag must be mounted using Mount(). Also, the structure specified for the createInfo parameter must be initialized with the InitializeCreateInfo function. The Flush function is also executed when the application area is created. As a result, calling this function might block for roughly one to two seconds.

Make sure that you check the characterID of CommonInfo(), available with the GetNfpCommonInfo() function, to ensure that the amiibo can be used by that application.

When planning to use NFP tags across multiple platforms, note that the byte order differs by platform.

As an example, the 3DS is little-endian, whereas the Wii U is big-endian.
