nn::nfp::CTR::Disconnect Function

Syntax

nn::Result Disconnect();

Parameters

None.

Return Values



Value Description
ResultSuccess Connection successful.
ResultInvalidOperation The library is not in the correct state.
ResultSleep The system is in a sleep state.
ResultWifiOff The Wi-Fi switch is set to off.
ResultIrFunctionError The IrDA module may be malfunctioning. This result can only occur on the CTR. Equivalent to ir::CTR::ResultFatalError.
Removing the IrDA module from the debugger forcibly triggers it.
Other Unexpected error.

Description

Breaks the connection to the NFC reader and writer.

When called on SNAKE, halts the detection of tags just as if StopDetection() were called, if in the middle of detecting tags. If it is not detecting tags, it does nothing and returns success. It can return a failure, however, when the NFP library has not been initialized, when in Sleep Mode, or when wireless is turned off.
