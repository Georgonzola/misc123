nn::nfp::CTR::GetConnectResult Function

Syntax

nn::Result GetConnectResult(
     nn::Result * pConnectResult
);

Parameters

Name Description
out pConnectResult Gets the result when the Connect function finishes.

Return Values



Value Description
ResultSuccess The process succeeded.
ResultInvalidOperation The library is not in the correct state.
ResultInvalidPointer Invalid argument.

Description

Returns the result of the Connect function. When called on SNAKE, the function succeeds without doing anything.

Results obtained by this function are just the results of the Connect function called immediately prior. ResultSuccess is returned even if a disconnect occurs immediately after a short-term connection succeeds.
Consequently, if the connection is lost immediately after a successful call to the Connect function, you might encounter a situation where this function returns ResultSuccess even though the application considers the call to the Connect function to have failed.
In such instances, treat it as a disconnection after a successful connection.

The following results are returned to pConnectResult.

ValueDescription
ResultSuccessThe connection succeeded or a disconnection occurred that was unrelated to calling the Connect function.
ResultConnectCanceledThe connection process was canceled or failed due to an internal error.
ResultTimeOutErrorThe connection timed out unsuccessfully.
ResultUpdateRequiredNFC reader/writer needs to be updated. This error can be issued by corrupting the firmware with the FangateFwUpdater tool.
ResultIrFunctionErrorThe IR module might be malfunctioning. This result can only occur on CTR. Equivalent to ir::CTR::ResultFatalError.
ResultNfcTargetErrorNFC reader/writer might be malfunctioning. This can be forcibly issued by enabling Fangate Failure Emulation on the Config tool.
Anything elseAn unexpected error.
