nn::nfp::CTR::GetNfpRegisterInfo Function

Syntax

nn::Result GetNfpRegisterInfo(
     RegisterInfo * pRegData
);

Parameters

Name Description
out pRegData Pointer to the RegisterInfo object in which the retrieved data was stored.

Return Values



Value Description
ResultSuccess Obtained successfully.
ResultInvalidPointer An invalid pointer to the parameter was specified.
ResultInvalidOperation The library is not in the correct state.
ResultSleep The system is in a sleep state.
ResultWifiOff The Wi-Fi switch is set to off.
ResultNeedRegister The tag has not been registered. It must be registered in amiibo Settings.
Other Unexpected error.

Description

Gets initial NFP tag registration info.

To call this API function, the tag must be mounted with the Mount() function.

If a ResultNeedRegister error occurs, it means the initial information has not been registered. You can prompt the user to register the information by calling up the amiibo Settings from the StartAmiiboSettings() function. In such cases, specify AMIIBO_SETTINGS_NICKNAME_OWNER as the mode for the Input structure.

Tag nicknames may sometimes include characters that are not supported in the application. Handle this so that it does not corrupt the screen display or hinder the progress of your application. For more information, see the section about the display of nicknames and creator names in the 3DS Guidelines.
