nn::nfp::CTR::MountRom Function

Syntax

nn::Result MountRom();

Parameters

None.

Return Values



Value Description
ResultSuccess Mount succeeded.
ResultInvalidOperation The library is not in the correct state.
ResultSleep The system is in a sleep state.
ResultWifiOff The Wi-Fi switch is set to off.
ResultNotSupported Not an NFP tag. Check the tag being used.
ResultTagNotFound The target tag was not found. The tag might have been removed.
ResultInvalidFormatVersion Unsupported tag version. Notify the user that the tag is not supported.
Other Unexpected error.

Description

Mounts a tag. Can only access the ROM data on the tag.

Run after the tag is detected. This API function can block for a maximum of 500 milliseconds when called.

Unlike the Mount, this function succeeds even when the tag data is corrupted. You can get information from the ROM region with the GetNfpRomInfo function. The GetNfpCommonInfo, GetNfpRegisterInfo, and OpenApplicationArea functions cannot be used if the tag is not mounted with the Mount function. After the tag has been mounted with the MountRom function, you need to unmount it with the Unmount function before it can be remounted with the Mount function.

On the CTR, if this function is called the moment you disconnect from the NFC reader/writer, or the moment a tag is lost, the function might return ResultInvalidOperation.
When that happens, call the Connect function again after explicitly disconnecting by calling the Disconnect function.
