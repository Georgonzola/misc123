nn::nfp::CTR::OpenApplicationArea Function

Syntax

nn::Result OpenApplicationArea(
     bit32 accessId
);

Parameters

Name Description
in accessId The ID to use to access the application area.

Return Values



Value Description
ResultSuccess Mount succeeded.
ResultInvalidOperation The library is not in the correct state.
ResultSleep The system is in a sleep state.
ResultWifiOff The Wi-Fi switch is set to off.
ResultAccessIdMisMatch Cannot access the application area because the access IDs do not match.
ResultNeedCreate No application area. The application area must be created.
Other Unexpected error. The tag must be mounted using the Mount() function beforehand. If the ResultAccessIdMisMatch error occurs, it means that another application is already using that amiibo. You can delete those settings by calling amiibo Settings from StartAmiiboSettings(). In such cases, specify AMIIBO_SETTINGS_ERASE_GAMEDATA as the mode parameter for the Input structure.

Description

Enables access to the application area.
