nn::nfp::CTR::ReplaceOtherRegionName Function

Syntax

nn::Result ReplaceOtherRegionName(
     bit16 * outName,
     const bit16 * inputName,
     size_t bufferSize,
     bit8 fontRegion,
     bit16 replacement = 0x3F00
);

Parameters

Name Description
out outName Specifies the place to output the converted nickname.
in inputName Specifies the original nickname.
in bufferSize Specifies the size of the outName buffer in bytes.
in fontRegion Specifies the FontRegion for the amiibo.
in replacement Specifies the replacement character. Uses the question mark (?) (U+003F) when omitted.

Return Values



Value Description
ResultSuccess The event was set successfully.
ResultBufferIsSmall Invalid argument. outName must have at least NICKNAME_LENGTH_MAX + 1 elements.
ResultInvalidPointer An invalid pointer was specified.
ResultInvalidOperation The library is not in the correct state.
Other Unexpected error.

Description

Converts the amiibo nickname for display in the current region.

amiibo are mobile accessories that move between systems, regardless of region and platform. Consequently, an amiibo with a nickname registered in amiibo Settings for a different region might be held to the reader.

As a result, characters for all regions might be input for amiibo nicknames. If the characters are not supported by the application, they must be displayed instead as question marks (?) or as U+E06B characters.

For this function, when the console's region does not match the fontRegion specified in the arguments, all non-ASCII characters included in inputName are replaced by replacement, one by one.

The same buffer can be specified for outName and inputName.
