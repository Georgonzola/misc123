nn::nfp::CTR::Restore Function

Syntax

nn::Result Restore();

Parameters

None.

Return Values



Value Description
ResultSuccess The tag was successfully restored.
ResultInvalidOperation The library is not in the correct state.
ResultSleep The system is in a sleep state.
ResultWifiOff The Wi-Fi switch is set to off.
ResultNotSupported The target tag is not an NFP tag. Check the tag being used.
ResultNotBroken Tag data does not need to be restored because it is not corrupted.
ResultTagNotFound Target tag not found. The tag may have been replaced with another tag during the write process.
ResultOperationFailed An error occurred while writing to the tag. The tag data may not have been restored correctly.
ResultNeedRetry The writing process failed. Retrying may succeed.
ResultConnectCanceled Lost the connection with the NFC reader and writer due to an internal error.
ResultBackupError Failed to access the backup data. This NFP tag cannot be restored.
Other Unexpected error.

Description

Restores a corrupted tag using the backup data.

Call this function when the Mount() function returns ResultNeedRestore. The Flush() function is automatically called, so calling this API function can block for one or two seconds.

Backup data is created when tags are mounted and just before data is written to them. As a result, when a restore is needed after a write to a tag fails, the data written to the tag after the restore is not the data prior to the failed write operation. It is the data as it would have been written properly if the write operation was successful.

When this function call succeeds, the tag is automatically mounted, as it would be after Mount() had been run.
