nn::nfp::CTR::SetDeactivateEvent Function

Syntax

nn::Result SetDeactivateEvent(
     nn::os::Event * pEvent
);

Parameters

Name Description
inout pEvent Sets an event to use for the notification for when the tag is lost.

Return Values



Value Description
ResultSuccess The event was set successfully.
ResultInvalidPointer An invalid pointer to the parameter was specified.
ResultInvalidOperation The library is not in the correct state.
Other Unexpected error. Call this function after the NFP library is initialized and before the StartDetection function starts tag detection. When a detected tag is removed, a notification with the event specified in this function is sent. This event also occurs when tag detection is stopped by the StopDetection or Finalize function while tag detection is in progress. The event is managed within the API function, so there is no need for the user to initialize or release it. Note: The event is initialized to isManualReset = false .

Description

Sets an event indicating that a tag was lost (deactivated).
