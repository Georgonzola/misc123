nn::nfp::CTR::WriteApplicationArea Function

Syntax

nn::Result WriteApplicationArea(
     const void * pData,
     u32 dataSize,
     const TagId & tagId
);

Parameters

Name Description
in pData The data to write.
in dataSize The size of the data to write.
in tagId The UID of the tag to write the data to.

Return Values



Value Description
ResultSuccess Read successful.
ResultInvalidArgument Invalid argument. The size of the data to write exceeds the size of the dedicated application area.
ResultInvalidPointer An invalid pointer to the parameter was specified.
ResultInvalidOperation The library is not in the correct state.
ResultSleep The system is in a sleep state.
ResultWifiOff The Wi-Fi switch is set to off.
ResultUidMisMatch The specified UID does not match the tag's UID.
Other Unexpected error.

Description

Writes data to the application area on a tag.

Before using this function, you must mount the tag and call OpenApplicationArea. This function writes to a buffer in memory. To actually write to a tag, you must call Flush.

Note that if the amount of data you specify to write to the tag with this function is less than the maximum amount of data that can be saved to the tag, the data will be written starting from the beginning of the area, and the remaining part of the area will be deleted and filled with random numbers.
