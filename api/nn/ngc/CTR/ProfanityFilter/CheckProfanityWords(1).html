nn::ngc::CTR::ProfanityFilter::CheckProfanityWords Member Function

Syntax

virtual nn::Result CheckProfanityWords(
     bit32 * pCheckResults,
     ProfanityFilterPatternList nPatternCode,
     const wchar_t ** ppWords,
     size_t nWordCount
);

Parameters

Name Description
out pCheckResults Specifies the buffer that stores the results of the check. This parameter requires an array with nWordCount elements.
nPatternCode
in ppWords Specifies a NULL-terminated string array containing the words to check. Encode it as UTF16-LE (little-endian).
in nWordCount Specifies the number of strings to check. The maximum number is nn::ngc::MAX_WORD_NUM.
in nPatterns Specifies which pattern list to check against.

Return Values

Description

Checks a number of specified words against the specified pattern lists to determine whether strings contain profanity and should not be displayed.

This function checks single words, such as screen names. The maximum number of characters (including the NULL terminator) per word is nn::ngc::MAX_WORD_LENGTH. This function cannot be used to process text added by the user. To check text input, use the ProfanityFilter::MaskProfanityWordsInText function instead.

This operation may take a long time. (It blocks.) Processing time can vary depending on the version of the system. We recommend running this function in a separate thread. Running it in the main thread can reduce the frame rate. You can use this function to check a maximum of nn::ngc::MAX_WORD_NUM strings at one time. Checking multiple strings at the same time requires less time than checking each string separately.

The results of the check on each string are stored in the pCheckResults. Check by taking the bitwise AND of this array and the result of left-shifting the value 1 by the value specified in nPatterns. The bit field has a value of 0 if there are no profanity patterns specified in any pattern lists.

To determine which pattern lists you need to check against, see the UGC guidelines in the latest 3DS Guidelines.

Because the @ symbol could denote an email address, this function always detects strings containing this symbol as possible disallowed words. On the other hand, strings containing lots of numbers that might be used to display phone numbers are not detected as possible disallowed words by this function. For this check, use the nn::ngc::CTR::CountNumbers function.
