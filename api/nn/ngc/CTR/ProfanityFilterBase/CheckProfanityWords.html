Description of CheckProfanityWords ( bit32 *, const wchar_t **, size_t )

This function checks single words, such as screen names. The maximum number of characters (including the NULL terminator) per word is nn::ngc::MAX_WORD_LENGTH. This function cannot be used to process text added by the user. To check text input, use the ProfanityFilterBase::MaskProfanityWordsInText function instead.

This operation may take a long time. (It blocks.) Processing time can vary depending on the version of the system. We recommend running this function in a separate thread. Running it in the main thread can reduce the frame rate. You can use this function to check a maximum of nn::ngc::MAX_WORD_NUM strings at one time. Checking multiple strings at the same time requires less time than checking each string separately.

The results of the check on each string are stored in pCheckResults. This array is structured as a bit field, so you can locate which pattern set contains a problematic string. Check the results by calculating the bitwise AND of (1) this bit field, and (2) the result of left-shifting the value 1 by the value of the nn::ngc::CTR::ProfanityFilterPatternList enumerated type constant. The bit field has a value of 0 if no strings are registered as profanity in any pattern lists.

To determine which pattern lists you need to check against, see the UGC guidelines in the latest 3DS Guidelines.

Because the @ symbol could denote an email address, this function always detects strings containing this symbol as possible disallowed words. On the other hand, strings containing lots of numbers that might be used to display phone numbers are not detected as possible disallowed words by this function. For this check, use the nn::ngc::CTR::CountNumbers function.
