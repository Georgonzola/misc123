nn::ngc::CTR::ProfanityFilterBase::MaskProfanityWordsInText Member Function

Syntax

virtual nn::Result MaskProfanityWordsInText(
     int * pProfanityWordCount,
     bool bCommunicateWithOtherRegions,
     wchar_t * pText
) = 0;

Parameters

Name Description
out pProfanityWordCount Specifies a pointer to the number of times profanity appeared in a text. If this information is not required, you can specify NULL.
in bCommunicateWithOtherRegions Specifies whether the application exchanges data with other regions. This can change the list of items subject to checking. (Not currently referenced.)
pText Specifies the pointer to the text that is the subject of the check. Places where profanity appears in the text are overwritten by an asterisk. For this reason, you must specify a buffer that the application can write to. The maximum number of characters than can be checked is nn::ngc::MAX_TEXT_LENGTH (512), including the null character. If a string is longer than this value, the part that exceeds this maximum length is not inspected.

Return Values

Description

Gets the hardware's region and language settings, and checks the specified text against the pattern list specified in the UGC guidelines of the 3DS Guidelines to determine whether the text contains profanity and should not be displayed.

This function processes text strings equal to or longer than a particular length. The maximum number of characters (including the null terminator) for the text is nn::ngc::MAX_TEXT_LENGTH. It cannot process short single words, such as screen names. To check words, use the ProfanityFilterBase::CheckProfanityWords function instead.

When this function is called, it checks the specified strings to determine whether they contain profanity and masks those parts with asterisks (*).

This operation may take a long time. (It blocks.) Processing time can vary depending on the version of the system. We recommend running this function in a separate thread. Running it in the main thread can reduce the frame rate.
