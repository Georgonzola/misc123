nn::olv::Initialize Function

Syntax

nn::olv::Result Initialize(
     nn::olv::InitializedData * initializedData,
     const nn::olv::InitializeParam * initializeParam
);

Parameters

Name Description
out initializedData The class that stores the initialization results.
in initializeParam The class that stores the parameters used for initialization.

Return Values

Returns the result of the operation.
For information about handling the result, see the section about Error Handling or the descriptions of the nn::olv::Result values.


Value Description
ResultSuccess Succeeded.
ResultUsageError A usage error.
ResultFatalError A fatal error.
ResultLibraryPermanentError A permanent library error.
ResultLibraryTemporaryError A temporary library error.

Description

Initializes the OLV library.

You must call this function before calling any of the functions in the nn::olv namespace.
Before calling this function, you must call the nn::fs::Initialize, nn::cfg::CTR::Initialize, and nn::act::Initialize functions to initialize those libraries.
Note that the working buffer specified for the initializeParam argument with the nn::olv::InitializeParam::SetWork() function must not be cleared until the nn::olv::Finalize() function is called.
This function is not thread-safe. Make sure not to execute any OLV library functions or class methods on another thread while calling this function.

OLV Access Key Settings
To use the OLV library at the development stage, configure and build the application with the unique ID issued by Nintendo, and then set the OLV access key issued by Nintendo using the CTR Config tool before starting the application. This prevents people who are not involved in application development from posting to the viewable communities.
