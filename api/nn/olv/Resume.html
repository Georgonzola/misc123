nn::olv::Resume Function

Syntax

nn::olv::Result Resume(
     nn::olv::ResumedData * resumedData,
     const nn::olv::ResumeParam * resumeParam
);

Parameters

Name Description
out resumedData The class that stores the recovery process results.
in resumeParam The class that stores the parameters used for the recovery process.

Return Values

Returns the result of the operation.
For information about handling the result, see the section about Error Handling or the descriptions of the nn::olv::Result values.


Value Description
ResultSuccess Succeeded.
ResultUsageError A usage error.

Description

Handles recovery operations.

Call this function after initializing the OLV library, when resuming at the application start event after calling the nn::olv::StartPortalApp() function, or when resuming at the application start event after transitioning to the HOME Menu.
This function is not thread-safe. Make sure not to execute any OLV library functions or class methods on another thread while calling this function.

You can receive data passed from the Miiverse application when starting or resuming an application using an application jump from the Miiverse application.
When you want to receive this data, get the system parameter from the Miiverse application and specify it for the resumeParam parameter in the nn::olv::ResumeParam::SetSystemArgs() function.
Use the nn::applet::CTR::GetInitialParam function to get the system parameter from the Miiverse application when starting an application. Use the nn::applet::CTR::WaitForStarting function to get the system parameter from the Miiverse application when resuming an application from the Miiverse application. When doing this, make sure that the nn::applet::CTR::AppletId value returned by the n::applet::CTR::GetInitialParam or nn::applet::CTR::WaitForStarting function matches the nn::olv::PORTAL_APP_APPLET_ID value to ensure that the system parameter originated from the Miiverse application.
You can get the data passed from the Miiverse application from the resumedData parameter.
Note: The appropriate parameters must be specified in the application banner spec (BSF) file to make it possible to start an application from Miiverse and to make it possible for the application to get the system parameter from the Miiverse application. For more information about which parameters to specify, see the ctr_makebanner reference.
Note: See the StartPortalApp demo for an example.

When confirming the behavior of nn::olv::Resume() upon an application jump from a Miiverse post in a development environment, the following methods can be used to confirm, because communities favorited by a user can be viewed within Miiverse even if the application is not running.
1. Start the Miiverse application while the application is running.
2. Add the application's community as a Favorite (by tapping the star icon).
3. Start the Miiverse application while the application is not running.
4. Transition to the application's community from the Favorite Communities.
5. Click Launch from the relevant post to start the application.
