nn::olv::StartPortalApp Function

Syntax

nn::olv::Result StartPortalApp(
     const nn::olv::StartPortalAppParam * startParam
);

Parameters

Name Description
in startParam The class that stores the parameters used when starting the Miiverse application.

Return Values

Returns the result of the operation.
For information about handling the result, see the section about Error Handling or the descriptions of the nn::olv::Result values.


Value Description
ResultSuccess Succeeded.
ResultUsageError A usage error.
ResultLibraryPermanentError A permanent library error.
ResultLibraryTemporaryError A temporary library error.

Description

Starts the Miiverse application.

If this function succeeds, you must call the nn::applet::CTR::WaitForStarting() function to wait for the start event of the application.
Note that the working buffer specified for the startParam argument with the nn::olv::StartPortalAppParam::SetWork() function must not be cleared until either the nn::olv::Resume() or the nn::olv::Finalize() function is called after the start event of the application.
This function is not thread-safe. Make sure not to execute any OLV library functions or class methods on another thread while calling this function.

The PARTNER-CTR series Debugger and the IS-CTR series debugger both have less available memory for the Miiverse application than the actual retail product. You may find that errors occur when starting the Miiverse application to check it.
(This issue does not occur on the various development devices or on IS-SNAKE Devkit (with the standard 2 GB of memory or more).)

If you are in an environment where you are using the development menu in PARTNER-CTR Debugger, first load the application into emulator memory and then press the A Button to start the application. (Communities may not be displayed properly if you run the Miiverse application by dragging the file into the window, selecting File > Load, or using the L command.

Depending on the system mode setting, this function might not work correctly. For more information, see the documentation under CTR Tools > Configuration.
