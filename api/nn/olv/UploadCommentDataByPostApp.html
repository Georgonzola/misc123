nn::olv::UploadCommentDataByPostApp Function

Syntax

nn::olv::Result UploadCommentDataByPostApp(
     nn::olv::UploadedCommentData * uploadedData,
     const nn::olv::UploadCommentDataByPostAppParam * uploadParam
);

Parameters

Name Description
out uploadedData The class for storing the results of starting the posting applet and uploading comments.
You can specify NULL if not needed.
in uploadParam The class for storing the parameters used when starting the posting applet and uploading comments.

Return Values

Returns the result of the operation.
For information about handling the result, see the section about Error Handling or the descriptions of the nn::olv::Result values.


Value Description
ResultSuccess Succeeded.
ResultUsageError A usage error.
ResultCanceled Canceled.
ResultCanceled is returned if the posting applet is closed without the user uploading the comments.
ResultLibraryPermanentError A permanent library error.
ResultLibraryTemporaryError A temporary library error.

Description

Starts the posting applet and uploads comments.

This function can only be used when a connection to the OLV library has already been made with the nn::olv::Connect() function.
Before calling this function you must use the nn::cfg::CTR::IsAgreedEula function and confirm that the user has agreed to the EULA.
If the EULA has not been agreed to, this function returns an error. For information about what to do when the EULA has not been agreed to, see the 3DS Guidelines.
This function is not thread-safe. Make sure not to execute any OLV library functions or class methods on another thread while calling this function.

Depending on the system mode setting, this function might not work correctly. For more information, see the documentation under CTR Tools > Configuration.
