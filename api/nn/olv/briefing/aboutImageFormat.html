Overview

This section describes the image formats handled by the OLV library.

Format for Handwritten Messages

The library supports 1-bit color BMP images for the handwritten messages that can be used for posts and comments.
The images must adhere to the following data format.

  • BMP image
    Content Data Size Comments
    File type 42 4D 2 'BM'

    Width 40 01 00 00 4 320 pixels
    Height 78 00 00 00 4 120 pixels

    Compression format 00 00 00 00 4 0 (no compression)

    Color palette (white) FF FF FF 00 4
    Color palette (black) 00 00 00 00 4
    Image data (Optional) 4800
    Padding 00 00 2

Format for Attached Images and Attached Thumbnails

The library supports baseline JPEG compression and MPO as the image formats for image and thumbnail attachments that can be used for posts and comments.
The specification for thumbnail attachments can be omitted.
The images must adhere to the following data format.

  • JPEG or MPO image.
  • Baseline JPEG compression format.
  • Combination of width x height and image format.
    Image AttachmentThumbnail Attachment
    Width x HeightImage FormatWidth x HeightImage Format
    Method 1800 x 450JPEG171 x 96JPEG
    Method 2400 x 240JPEG171 x 103JPEG
    Method 3400 x 240MPO171 x 103JPEG
    Method 4320 x 240JPEG171 x 128JPEG
    Method 5640 x 480JPEG171 x 128JPEG
  • Size of attached image
    • Maximum of 512 KB.
  • Size of attached thumbnail
    • Maximum of 128 KB.
    • Note: If you specify a thumbnail attachment, the maximum size is 512 KB for the image and the thumbnail combined.

Format for Stamps and Stamp Masks

The library supports 1-bit color BMP images for the stamps and the stamp masks that the Post application uses for posts and comments.
Stamps and stamp masks must be either small (100 x 100) or large (200 x 200).
The stamp mask is the image data that specifies the transparent and opaque parts of the stamp image. Specify white for the opaque parts and black for the transparent parts.
The images must adhere to the following data format.

Format for Icons

The library supports 32-bit color TGA images for the icons that can be used for communities.
The images must adhere to the following data format.

  • TGA image
    • 32-bit color
    • No RLE compression
    • No color map
    • 48 x 48 (width x height)
    • Stored from lower left.
    • Data size (including header and footer)
      • 9260 bytes (Can be 9234 if the footer is omitted)
    • Note: The 18-byte header must be a byte string like the following.
      00 00 02 00 00 00 00 00 00 00 00 00 30 00 30 00 20 08 (hexadecimal).
