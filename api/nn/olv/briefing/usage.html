Starting Applications From Posts in the Miiverse Application

Under the following conditions, you can start an application from a post displayed in the Miiverse application.

  • The banner spec file (BSF file) for the application has EnableMiiverseJumpArgs enabled. (For the specification method, see the ctr_makebanner reference.)
  • The FLAG_APP_STARTABLE flag was set on the post when it was posted with nn::olv::UploadPostDataByPostApp.

When the application starts, it can receive the following parameters associated with the post.
  • Community ID
  • Post ID
  • Application data (only if set)
The method of retrieving the parameters differs depending on whether the application was started or resumed. For more information, see the descriptions at the following links.
  • When Application Was Started
  • When Application Was Resumed

How to Use

Before using the OLV library, the following libraries must be initialized.

  • nn::fs
  • nn::cfg
  • nn::act

To use the OLV library, an access key must be set.
Access keys are distributed by Nintendo along with a unique ID.
Set the access key using the Config tool.

The OLV access key is accessed along with a unique ID, and used to check posting to and viewing the Miiverse.
The key and ID are to prevent posting to and viewing of unrelated titles on the development server using the OLV library and Miiverse application.

The following steps are required before calling nn::olv::Connect.

  • Use the nn::cfg::CTR::IsAgreedEula function to check whether the EULA has been agreed to.
  • Use the nn:ac library to connect to the Internet.
  • Initialize the nn:http library.

Preparing the Library for Use

Link libnn_olv.fast.a or libnn_olv.small.a to your project.
In addition, add the following code to your program.

#include < nn/olv.h >

Initializing and Finalizing the OLV Library

Set the parameters in an nn::olv::InitializeParam object.

nn::olv::InitializeParam initParam;
u8* work;

// Get the required working buffer size.
u32 workSize = initParam.GetWorkSize();

// Set the working buffer. Do not free the working buffer until the OLV library is no longer in use.
work = new u8[workSize];
if (work == NULL)
{
    // Failed to allocate the working buffer.
}
initParam.SetWork(work, workSize);

Next, call the nn::olv::Initialize function to initialize the OLV library.
nn::olv::InitializedData initData;
// Initialize the OLV library.
nn::olv::Result result = nn::olv::Initialize(&initData, &initParam);
if (result.IsFailure())
{
    // Error handling.
}

Call the finalization function when you no longer need the OLV library.
// OLV library finalization.
nn::olv::Finalize();

OLV Library Connection Process

Set the parameters in an nn::olv::ConnectParam object.

nn::olv::ConnectParam connectParam;
// Set the service token for Miiverse obtained from the nn::act library.
connectParam.SetServiceToken(serviceToken);

Next, connect the library by calling nn::olv::Connect.
nn::olv::ConnectedData connectData;
// Run the OLV library connection process (initiates communication).
nn::olv::Result result = nn::olv::Connect(&connectData, &connectParam);
if (result.IsFailure())
{
    // Error handling.
}

Data that needs to be excluded based on the Blocked Users list is excluded on the server side. The developer is not required to implement any specific functionality. The following functions exclude data based on the Blocked Users list.

Use nn::olv::DownloadedPostData::TestFlags to determine post data attributes by the flags.
An example is shown below. 
// The postDataList variable is expected to be populated with the post data downloaded by calling the nn::olv::DownloadPostDataList function.
if (postDataList.TestFlags(nn::olv::DownloadedPostData::FLAG_EMPATHY_ADDED))
{
    // This post includes a Yeah.
}
if (postDataList.TestFlags(nn::olv::DownloadedPostData::FLAG_SPOILER))
{
    // This post is a spoiler. (Does not include spoilers under review.)
}

Use the nn::olv::DownloadedPostData::DownloadExternalImageData function to get the image data attached to post data.
An example is shown below. 
// The postDataList variable is expected to be populated with the post data downloaded by calling the nn::olv::DownloadPostDataList function.
if (postDataList[0].TestFlags(nn::olv::DownloadedPostData::FLAG_WITH_EXTERNAL_IMAGE_DATA))
{
    // Get the size.
    const u32 externalImageDataSize = postDataList[0].GetExternalImageDataSize();

    // Allocate memory.
    u8* externalImageData = new u8[externalImageDataSize];
    if (externalImageData == NULL)
    {
        // Failed to allocate memory.
    }

    u32 downloadSize = 0;

    // Get the image attachment. (Initiates communication.)
    nn::olv::Result result = postDataList[0].DownloadExternalImageData(
                                externalImageData, &downloadSize, externalImageDataSize);
    if (result.IsFailure())
    {
        // An error occurred. Handle it as required.
    }
}

Use the nn::olv::DownloadedPostData::GetBodyText function to get the text data from the post data.
An example is shown below. 
// The postDataList variable is expected to be populated with the post data downloaded by calling the nn::olv::DownloadPostDataList function.
if (postDataList[0].TestFlags(nn::olv::DownloadedPostData::FLAG_WITH_BODY_TEXT))

If an application is started by the Miiverse application when no other application is running, get the system parameters from the nn::applet::GetInitialParam function and pass them to the nn::olv::Resume function. An example is shown below. 
nn::applet::AppletId senderId = 0;

nn::olv::ResumedData resumeData;
nn::olv::ResumeParam resumeParam;

// Call the function to resume the OLV library.
if (senderId == nn::olv::PORTAL_APP_APPLET_ID)
{
    // Specify any system arguments passed by the Miiverse application.
    resumeParam.SetSystemArgs(systemArgs, (u32)systemArgsSize);
}
nn::olv::Result result = nn::olv::Resume(&resumeData, &resumeParam);
if (result.IsFailure())
{
    // An error occurred. Handle it as required.
}
else
{
    if (resumeData.GetMode() == nn::olv::ResumedData::MODE_START_POST)
    {
        // If it was started from a post in the Miiverse application, you can get the community ID, the post ID, and the application data.

        // Get the community ID.
        u32 communityId = resumeData.GetCommunityId();

        // Get the post ID.
        char postId[nn::olv::POST_ID_BUFF_LENGTH];
        resumeData.GetPostId(postId, nn::olv::POST_ID_BUFF_LENGTH);

        if (resumeData.TestFlags(nn::olv::ResumedData::FLAG_WITH_APP_DATA))
        {
            // Get the size of the application data.
            u32 appDataSize = resumeData.GetAppDataSize();

            // Get the application data.
            u8 appData[nn::olv::APP_DATA_BUFF_LENGTH];
            resumeData.GetAppData(appData, nn::olv::APP_DATA_BUFF_LENGTH);
        }
    }
}
