nn::os::AutoStackManager::Destruct Member Function

Syntax

virtual void Destruct(
     void * pStackBottom,
     bool isError
) = 0;

Parameters

Name Description
in pStackBottom The stack to release.
in isError Indicates whether it is being released because an error occurred.

Return Values

None.

Description

Releases the memory for the stack.

This function is a pure virtual function. This function itself cannot be called. You must implement a derived class in accordance with the specifications described here.

Releases the memory region passed to pStackBottom. pStackBottom is one of values returned by the Construct function until this function is called.

When isError is true, it indicates that it became necessary to free a tentatively allocated stack after an error was triggered during the creation of a thread within the Thread::StartUsingAutoStack or Thread::TryStartUsingAutoStack function.

In this instance, pStackBottom is the return value for the Construct function called immediately prior by the same thread.

When isError is false, it indicates the freeing of a stack in conjunction with the termination of the thread that is using the stack indicated by pStackBottom.

When the instance of the derived class is specified by Thread::SetAutoStackManager, the context called differs depending on the value of isError.

When isError is true, it is called from within the Thread::StartUsingAutoStack or Thread::TryStartUsingAutoStack function.

When isError is false, the context for the thread using pStackBottom is called.

Note that when isError is false, this function itself is using pStackBottom as a stack.

Before freeing pStackBottom, you must either change the stack such that it uses another memory region or free the memory without using the stack.

When isError is false, there is no issue with returning with the stack in an unusable state, as the stack is not used after returning from this function.
