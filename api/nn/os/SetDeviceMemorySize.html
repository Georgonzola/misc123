nn::os::SetDeviceMemorySize Function

Syntax

#include <nn/os.h>
Result SetDeviceMemorySize(
     size_t size
);

Parameters

Name Description
in size Specifies the new device memory size. The size parameter must be a multiple of DEVICE_MEMORY_UNITSIZE (4096). It is possible to reset a different size of device memory, but when doing so both the size before changing and the size after changing must be multiples of 1048576 (1 MB).

Return Values

Returns the execution result.

Description

Changes the size of the device memory.

Changes the size of the device memory. Device memory is not yet allocated immediately after the application has started, so here you specify the necessary size of memory and newly allocate the region.

When reconfiguring the memory, if the size of the region will be smaller after the change, the unneeded portion is deallocated from the end of the region and the remainder of the contents (from the start of the memory region for the full reconfigured size) is retained. If the memory size is enlarged by the reconfiguration, a new region is added to the end of the existing allocated region. (In this case, the memory sizes before and after the change must both be a multiple of 1 MB.)

If 0 is specified for the size, the entire memory region is freed. If the region is freed and then reallocated, the contents of the region are not guaranteed.

If you specify a size of memory that does not adhere to the designated multiple, nn::os::ResultMisalignedSize will return.
