Overview

The stereoscopic field of view has been increased with SNAKE because of the super-stable 3D feature. The super-stable 3D feature can be used with SNAKE systems. It detects the user's face, tracks and predicts the position of the user's eyes, and adjusts the stereoscopic effect of the 3D screen based on the user's position. The super-stable 3D feature is run automatically, so developers do not need to do anything to make it happen.
However, if you want to know the location of the user's eyes as obtained by the super-stable 3D feature, you'll need to call the QTM API.

Notes

  • The super-stable 3D feature can only be used on SNAKE systems.
    CTR systems do not receive any benefit from this feature.
  • The super-stable 3D feature is implemented using the camera images and gyro values.
  • The super-stable 3D feature is disabled when game applications need to access the camera. The disabling occurs from the time the camera library is initialized to the time it is finalized.
  • Game applications that need to obtain gyro values can do so while leaving the super-stable 3D feature enabled.

Development Flow

It is possible to obtain the position of the user's eyes by calling the GetTrackingData function. The super-stable 3D feature calculates the position of the user's eyes based on camera images and gyro values. It uses gyro values to capture the quick motions that are typical of handheld devices.

The GetTrackingData function returns the position of the user's eyes at the point in time when the function was called.

There is some important information about the camera operations to keep in mind. At the time when the system is processing camera images to obtain the eye position, 30 ms will have elapsed since the image was captured. Using this value as is results in outdated positional information from the perspective of the game application. The GetTrackingData function runs a predictive process based on the system time when called, and the previous eye position to calculate the current position.

When the GetTrackingData function call is made in an irregular cadence, the predictive precision between multiple frames will also experience some irregularity. However, if GetTrackingData is called at the same time in every rendering frame, the effect of the timing variance is reduced, smoothing the position data.

GetTrackingData gets two statuses about the user's position.:

  • isValid flag
    Set to true if the user is in front of the camera and the user's position is known with a high degree of certainty.
    Set to false if the user cannot be found or if the positional information has a low degree of certainty. Even if isValid is false, GetTrackingData returns the eye position. In this case, the eye position is derived from the user position (assumed to be directly in front of the system) as well as the gyro values. This functionality is designed to cover cases when the user cannot be found, but is assumed to be directly in front of the system, and is moving the system while keeping his or her head in the same place.
  • isClipped flag
    Indicates that the user position has been clipped because it is either too close or too far from the extents of the camera's field of view. Clipping refers to a phenomenon wherein the eye coordinates cannot be followed past a certain point (the clip position). As a verb, clip indicates the act of correcting the eye position using this clip position. The clip position is the same for all SNAKE systems. The constraints on this clip position make it possible to handle edge cases with the assumption that all SNAKE consoles are guaranteed to have the same maximum values, regardless of any individual differences among their cameras' field of view.

Using 3D Rendering

If you want to use the user position for rendering a 3D scene, refer to the DynamicStereoCamera class.

Prohibitions

Prohibition Against Game Content That Requires Face Tracking

Face tracking is not guaranteed to detect properly on all people in every conceivable condition. As a result, using the tracking effect as a control that is required in order to advance through a game is prohibited. Consider face tracking to be a feature that enhances the 3D effect.

Prohibition Against Initializing the QTM Library in Software That Does Not Support Face Tracking

It is possible to verify whether you have done this by seeing whether NINTENDO:QtmInitialize is displayed in the Middleware List within the Basic Information (CIA) tab in Master Editor.
