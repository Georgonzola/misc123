nn::rdt::Receiver::Initialize Member Function

Syntax

nn::Result Initialize(
     const ReceiverConfig & config
);

Parameters

Name Description
in config Data structure storing initialization parameters. For more information, see ReceiverConfig.

Return Values

Returns the result of initialization. Specifically, this function may return the following: a value for which the nn::Result::IsSuccess() function returns true, or a result code returned by the UDS library (such as ResultAlreadyInitialized, ResultNullPointer, or ResultInvalidSize).

Description

Initializes the instance. If initialization succeeds, this function implicitly consumes two nn::uds::EndpointDescriptor objects. If initialization fails, the function returns without consuming any nn::uds::EndpointDescriptor objects. Memory passed to the instance by this call must be allocated before calling Finalize. The nn::uds::Attach function is called implicitly with nn::uds::ATTACH_BUFFER_SIZE_DEFAULT specified as the receive buffer size. If the application does not supply a large enough buffer to nn::uds::Initialize, initialization fails, returning nn::uds::ResultOutOfResource, because the nn::uds::Attach function uses the buffer passed to nn::uds::Initialize.
