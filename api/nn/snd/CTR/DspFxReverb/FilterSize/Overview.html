nn::snd::CTR::DspFxReverb::FilterSize Structure

Syntax

struct FilterSize
{
   u32 m_Comb0;
   u32 m_Comb1;
   u32 m_AllPass;
};

Description

This structure represents the filter size.

m_Comb0 and m_Comb1 represent the size of comb filters used on back-end reverberations (see the description in nn::snd::CTR::DspFxReverb::Param for more information). The smaller these values, the more sounds with certain frequencies will take on a metallic timbre. The initial value of m_Comb0 is 19 * 160, and the initial value of m_Comb1 is 23 * 160.

m_AllPass represents the size (delay length) of the all-pass filter used on back-end reverberations. The larger this value, the more detailed the back-end reverberations can become. The initial value is 13 * 160.

160 has its origin in the number of samples that can be processed in one audio frame (approximately 5 msec). As for the initial values, such as 19, 23, 13, etc., a value setting that is relatively prime is preferable. When you do not use a relatively prime value, it becomes easier for cycles to occur in the reverb.

When you change these values while an effect is executing, there is the possibility that noise will occur.

Values larger than 0 must be set for m_Comb0, m_Comb1, and m_AllPath.

Member Variables

m_Comb0 u32 Size of comb filter 0. (Default = 19 * 160.)
m_Comb1 u32 Size of comb filter 1. (Default = 23 * 160.)
m_AllPass u32 Coefficient for the full band pass filter. (Default = 13 * 160.)
