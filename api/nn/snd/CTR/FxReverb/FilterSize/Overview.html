nn::snd::CTR::FxReverb::FilterSize Structure

Syntax

struct FilterSize
{
   u32 m_Comb0;
   u32 m_Comb1;
   u32 m_AllPass;
};

Description

This structure represents the filter size.

m_Comb0 and m_Comb1 represent the size of comb filters used on back-end reverberations. (For more information, see the description in Param.)

If this value is small, only a specific frequency is prominent and the sound becomes metallic. The initial value of m_Comb0 is 19 * 160, and the initial value of m_Comb1 is 23 * 160.

m_AllPass indicates the size (delay length) of the full band pass filter applied to subsequent reverb sounds. When this value is large, the subsequent reverb sounds can be more detailed. The initial value is 13 * 160.

160 originates in the number of samples processed in one audio frame (about 5 milliseconds). As for the initial values, such as 19, 23, 13, and so on, a value setting that is relatively prime is preferable.

If you are setting values that are not coprime, periods appear more easily in the reverb sound.

When you change these values while an effect is executing, noise may occur.

m_Combo0, m_Comb1, and m_AllPath must be set to values greater than 0. If 0 is specified and the SetParam function is called, operations halt on an assert failure in Debug and Development builds.

The SetParam function returns false in Release builds.

Member Variables

m_Comb0 u32 Size of comb filter 0.
m_Comb1 u32 Size of comb filter 1.
m_AllPass u32 Coefficient for the full band pass filter.
