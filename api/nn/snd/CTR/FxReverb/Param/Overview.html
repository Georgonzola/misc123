nn::snd::CTR::FxReverb::Param Structure

Syntax

struct Param
{
   u32 m_EarlyReflectionTime;
   u32 m_FusedTime;
   u32 m_PreDelayTime;
   f32 m_Coloration;
   f32 m_Damping;
   FilterSize * m_pFilterSize;
   f32 m_EarlyGain;
   f32 m_FusedGain;
   bool m_UseHpfDamping;
};

Description

The data structure for reverb parameters.

Reverb can be divided into two large groups: "early reflection" and "subsequent reverb sounds."

"Early reflection" can be adjusted with m_EarlyReflectionTime and m_EarlyGain.

m_EarlyReflectionTime signifies the time until the early reflection returns. Larger values indicate longer times from when the sound reflects until it returns. Units are in milliseconds and must be larger than 5, set in multiples of 5. The initial value is set to 60.

m_EarlyGain signifies the output gain for early reflection. Set a value from 0.0.f to 1.0.f. When set to 0.0f, the early reflection cannot be heard. The initial value is set to 0.6f.

The back-end reverberation can be adjusted with m_FusedTime, m_FusedGain, m_PreDelayTime, m_Coloration, m_Damping and m_pFilterSize.

m_FusedTime signifies the time that the subsequent reverb sound continues to play. Larger values can express spaces where sounds are easily reflected. Units are in milliseconds, and the initial value is set to 4000.

m_FusedGain signifies the output gain for subsequent reverb sound. Set a value from 0.0.f to 1.0.f. When set to 0.0f, the subsequent reverb sound cannot be heard. The initial value is set to 0.4f.

m_PreDelayTime is the lag time before the subsequent reverb sound. Larger values cause the subsequent reverb sound to be heard later. Units are in milliseconds and must be larger than 5, set in multiples of 5. The initial value is 100.

m_Coloration signifies the coefficient for the full band pass filter of the subsequent reverb sound. The larger this value, the more detailed the back-end reverberation, and the more complex the reverberation becomes.

Set a value from 0.0.f to 1.0.f. When set to 0.0f, the full band pass filter is passed through. This value is initially 0.5f.

m_Damping indicates how the filter is applied to subsequent reverb sound. You can select either a low-pass filter or high-pass filter (specified by m_UseHpfDamping) as the filter type. Set a value from 0.0.f to 1.0.f. If using the low-pass filter, the larger the value for m_Damping, the lower the cutoff frequency. (If set to 0.0f, the low-pass filter is not applied.)

If using the high-pass filter, the larger the value for m_Damping, the higher the cutoff frequency. (If set to 0.0f, the high-pass filter is not applied and the phase is reversed.)

The default value for m_Damping is 0.4. The default value for m_UseHpfDamping is false (low-pass).

See the FilterSize description for each member variable of m_pFilterSize.

The necessary memory that you can get with the GetRequiredMemSize function is dependent on m_EarlyReflectionTime, m_PreDelayTime, and m_pFilterSize.

When you change the following values while an effect is executing, noise may occur.
m_EarlyReflectionTime
m_PreDelayTime

In addition, if you continually change the following values even while an effect is executing, noise does not occur.
m_FusedTime
m_Coloration
m_Damping
m_EarlyGain
m_FusedGain

Call SetParam to change parameters while an effect is being executed.

Member Variables

m_EarlyReflectionTime u32 Early reflection time (in milliseconds).
m_FusedTime u32 Attenuation time (in milliseconds) for subsequent reverb sounds.
m_PreDelayTime u32 Subsequent reverb sound pre-delay time (in milliseconds).
m_Coloration f32 Coefficient for the full-band pass filter for subsequent reverb sound.
m_Damping f32 Damping for the filter applied to subsequent reverb.
m_pFilterSize FilterSize * Filter size. If you specify NULL, the default value of the FilterSize structure is set.
m_EarlyGain f32 Output gain for the initial reflection.
m_FusedGain f32 Output gain for subsequent reverb sound.
m_UseHpfDamping bool Flag indicating whether to apply a high-pass filter to subsequent reverb.
