nn::snd::CTR Namespace

Description

The namespace of the CTR sound library. (It can also be written without the CTR portion.)

Sound processing is carried out while synchronizing with the DSP by sound threads that include the nn::snd::CTR::WaitForDspSync and nn::snd::CTR::SendParameterToDsp functions. Sound operations and parameter settings are configured after using the nn::snd::CTR::WaitForDspSync function to wait for an interrupt from the DSP that occurs approximately every 4.889 milliseconds, and then applying those operations and parameter settings to the DSP using the nn::snd::CTR::SendParameterToDsp function. The nn::snd::CTR::WaitForDspSync and nn::snd::CTR::SendParameterToDsp functions must be called alternately.

The snd library uses an nn::os::Event object held internally to perform the above synchronization.

The nn::dsp::CTR::Initialize function and functions for loading DSP components (such as the nn::dsp::CTR::LoadDefaultComponent function), must be called before the nn::snd::CTR::Initialize function.

Because there is no method of recovery if an API that returns Result returns an error, treat such an error as a fatal error.

Effect
SetEffect Configures the effect. Configures the delay effect. To process data that is to be output by AUX, you can use this function to apply delay and reverb effects in addition to configuring advance processing with the RegisterAuxCallback function. Note that you can only configure one delay (FxDelay) and one reverb (FxReverb) for each bus. When you configure more than one, only the settings for the last one are valid. The older configured effects are cleared. If you have called ClearEffect to clear older configurations as noted previously, you can deallocate the memory used for those effects using the FxDelay::ReleaseWorkBuffer and FxReverb::ReleaseWorkBuffer functions. Before calling this function, you must call the FxDelay::SetParam, FxDelay::GetRequiredMemSize, and FxDelay::AssignWorkBuffer functions from the FxDelay class to finish preparing for the delay effect.
ClearEffect Clears an effect setting. Clears an effect configured with the ttSetEffect/tt function. Any callback set with the RegisterAuxCallback function is not cleared. (You can clear it with the ClearAuxCallback function.)
