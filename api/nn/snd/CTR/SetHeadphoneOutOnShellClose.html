nn::snd::CTR::SetHeadphoneOutOnShellClose Function

Syntax

Result SetHeadphoneOutOnShellClose(
     bool forceout
);

Parameters

Name Description
in forceout Forces headphone output if true.

Return Values

Returns the result of the operation.

Value Description
Result::IsSuccess Indicates that the operation completed normally.
ResultComponentNotLoaded The dsp component has not been loaded.
Other An unexpected error occurred.

Description

Sets whether to force headphone output when the cover is closed.

When the cover is closed and sleep is denied, sound output defaults to headphone output regardless of whether headphones are inserted. This function can change that functionality.

When the flag argument is set to false, even if sleep is denied, sound is output from the speakers unless headphones are inserted. If this argument is set to true, sound is always output from the headphones.

This function must be called with the dsp component loaded (after the nn::dsp::CTR::LoadDefaultComponent and nn::dsp::CTR::LoadComponent functions are called). After this function is called, the settings are maintained until the nn::dsp::CTR::UnloadComponent function is called. (It is not necessary to keep track of things like HOME Menu transitions.) However, when this function is called from callback functions set by the nn::applet::CTR::SetSleepQueryCallback function (which are called when the cover is closed), the settings are applied from the next time the cover is closed and need to be set earlier.

These settings are only enabled when the application that called the function and the library applet that application called are active.

If this function is called with the argument set to false and sleep is not denied, sound is output from the speakers for the interval between when the cover is closed and when the system actually transitions into the sleep state. (The duration of this interval depends on the load on the system.) When calling this function with the argument set to false, always deny sleep.
