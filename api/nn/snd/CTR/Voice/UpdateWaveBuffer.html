nn::snd::CTR::Voice::UpdateWaveBuffer Member Function

Syntax

void UpdateWaveBuffer(
     WaveBuffer * pBuffer
);

Parameters

Name Description
in pBuffer Pointer to a buffer information structure.

Return Values

None.

Description

This updates buffer information appended to a voice.

You must call this function immediately after updating the specified WaveBuffer information (Note: The parameters may be updated even if this function is not called). The only variables that can be modified are sampleLength and loopFlag. Functionality is not guaranteed if any other parameters are modified.

This function has no effect if a pointer to a WaveBuffer that has not been appended is specified in the parameter. The actual update takes place when nn::snd::CTR::SendParameterToDsp is called. For this reason, the WaveBuffer pointer specified in the parameter still may not be updated even if it has been appended (for example, if the state of the WaveBuffer before the update is nn::snd::CTR::WaveBuffer::STATUS_DONE).

Note the following when modifying sampleLength.

  • As described above, the update takes place when nn::snd::CTR::SendParameterToDsp is called. Even if sampleLength is smaller than the current value, the latter portion of the buffer (the region after the new value) might be referenced immediately after this function is called.
  • If the newly specified sampleLength value is smaller than the current playback position, playback stops immediately, or a buffer jump is performed.
