nn::uds::CTR::CreateNetwork Function

Syntax

nn::Result CreateNetwork(
     u8 subId,
     u8 maxEntry,
     bit32 localId,
     const char passphrase[],
     size_t passphraseLength,
     u8 channel,
     const void * pData,
     size_t dataSize
);

Parameters

Name Description
in subId Specifies the ID for identifying the communication mode. Set this parameter to a value between 0x00 and 0xFE.
in maxEntry Specifies the maximum number of nodes that can connect to the network. This value can be up to NODE_MAX, including the local device.
in localId Specifies the local communication ID. Specify the value generated by CreateLocalCommunicationId.
in passphrase[] Specifies the string serving as the seed for the passphrase used to encrypt the wireless layer. With UDS, communication can only be established if localId and passphrase match.
in passphraseLength Specifies the passphrase length. Specify a value that is at least IR_PASSPHRASE_LENGTH_MIN, but no larger than IR_PASSPHRASE_LENGTH_MAX.
in channel Specifies the channel to use for communication. Specify channel 0 (automatic), 1, 6, or 11. When this function runs on production devices, the channel is always selected automatically.
in pData Specifies a pointer to the application data to set in the beacon.
in dataSize Specifies the size of the application data to set in the beacon. Specify a value no greater than NET_DESC_APPDATA_SIZE_MAX.

Return Values

Returns the result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates that the node successfully established local communication and began operating as a host.
ResultNotInitialized Indicates that the library is not initialized. Call the Initialize function.
ResultInvalidState Indicates that the function cannot be executed in this state. Returned when executed in a state other than STATE_DISCONNECTED. If the system goes to sleep while this function is being called and an internal error occurs, this value is returned and then transitions to STATE_ERROR.
ResultOutOfRange Indicates that an argument was outside of the allowed range. The function might succeed if you re-execute with an appropriate value for the argument.
ResultWirelessOff Indicates that the device entered wireless off mode. Reinitialization is required.
ResultTooLarge Indicates that dataSize is greater than NET_DESC_APPDATA_SIZE_MAX.
ResultInvalidPointer Indicates that pData is NULL. NULL is allowed when dataSize is 0.
Other Indicates failure for other reasons.

Description

Sets application data in the beacon and builds a new network. This function is provided for debugging purposes, and can be used to specify the channel to use for wireless communication during development. Note that the channel argument is ignored on production hardware.

Approximately 800 ms is needed to complete this because auto-selection of channels and other operations must be performed.

This API function allows you to set application data in the beacon when building the network. To update the beacon's application data after the network has been built, use the SetApplicationDataToBeacon function. The maximum size of this application data is NET_DESC_APPDATA_SIZE_MAX bytes. The data is not encrypted. It can be collected by any general-purpose computing device, such as a desktop computer. Other devices can get this data by calling Scan while they are connected as a Client or Spectator.

This function is thread-safe.
