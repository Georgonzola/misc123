nn::uds::CTR::GetApplicationDataFromBeacon Function

Syntax

nn::Result GetApplicationDataFromBeacon(
     void * pBuffer,
     size_t * pDataSize,
     size_t bufferSize
);

Parameters

Name Description
out pBuffer Specifies the location storing the data.
out pDataSize Specifies the size of the data.
in bufferSize Specifies the size of the buffer. The sender can freely change the size of the data. In general, specify NET_DESC_APPDATA_SIZE_MAX bytes.

Return Values

Returns the result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultNotInitialized Indicates that the library is not initialized. Call the Initialize function.
ResultInvalidState Indicates that the function cannot be executed in this state. Returns that it was executed in a state where it was not connected to the network.
ResultTooLarge Indicates that the bufferSize was smaller than the received data. The desired size is NET_DESC_APPDATA_SIZE_MAX.
ResultWirelessOff Indicates that the device entered wireless off mode. Reinitialization is required.
Other Indicates failure for other reasons.

Description

Gets the data set in the beacon.

The host gets the data set using the SetApplicationDataToBeacon function. This function can only be run when connected to the network. There is no notification if the data is updated.

This function is thread-safe.
