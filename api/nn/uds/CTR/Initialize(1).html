nn::uds::CTR::Initialize Function

Syntax

nn::Result Initialize(
     nn::os::Event * pStatusUpdateEvent,
     void * receiveBuffer,
     const size_t bufferSize,
     nn::cfg::UserName * pUserName
);

Parameters

Name Description
out pStatusUpdateEvent Event used to notify about connection state updates. This parameter is initialized as an auto-reset event within the library.
in receiveBuffer Pointer to the start of the receive buffer used by the UDS library. Specify a buffer aligned to BUFFER_ALIGNMENT (4096 bytes). Device memory cannot be used. Access to the memory region specified by the buffer is prohibited until the Finalize function completes.
in bufferSize The size of the receive buffer. Specify a value that is a multiple of BUFFER_UNITSIZE (4096 bytes).
in pUserName Specifies the user name. Handle the name according to UGC guidelines in the 3DS Guidelines. If NULL, uses System Settings.

Return Values

Returns the result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates that initialization succeeded.
ResultAlreadyOccupiedWirelessDevice Indicates that communication is already taking place. New UDS communication cannot begin.
ResultOutOfResource System resources are insufficient.
ResultWirelessOff Wireless-disabled mode.

Description

Initializes the UDS library. Initialization fails if any other communication features are already in use.

This process could block for a long time, because the UDS library retains control of the communication device until background communications are ended and Finalize runs.
Always call the Finalize function to release the communication device after you are finished using the UDS library.
Handle the user name specified by pUserName according to UGC guidelines in the 3DS Guidelines.

This function is thread-unsafe.
