nn::uds::CTR::Receive Function

Syntax

nn::Result Receive(
     const EndpointDescriptor & endpointDesc,
     void * pBuffer,
     size_t * pReceivedSize,
     size_t bufferSize,
     bit8 option = 0x00
);

Parameters

Name Description
in endpointDesc Specifies a descriptor indicating the endpoint to use. The port and sender must be attached in advance using the Attach function.
out pBuffer Specifies the location to store received data. Specify a buffer aligned to RECEIVE_DATA_ALIGNMENT (4 bytes).
out pReceivedSize Specifies the size of the received data. The UDS maximum received data size is UDS_PACKET_PAYLOAD_MAX_SIZE bytes.
in bufferSize Specifies the size (in bytes) of the receive buffer (pBuffer). The size of the receive buffer must be a multiple of RECEIVE_DATA_UNITSIZE (4 bytes).
in option Specifies the receive options. Specifying NO_WAIT causes the function to return immediately, even if no data has been received. If not specified, the function does not return until either it receives data or an error occurs.

Return Values

Returns the result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultNotInitialized Indicates that the library is not initialized or the specified endpoint does not exist.
ResultInvalidState Indicates that the function cannot be executed in this state. Returns that it was executed in a state where it was not connected to the network.
ResultNotAuthorized Indicates that an endpoint that was not attached was specified.
ResultTooLarge Indicates that the bufferSize was smaller than the received data. The best size is UDS_PACKET_PAYLOAD_MAX_SIZE.
ResultOutOfRange Indicates that an argument was outside of the allowed range. The function might succeed if you re-execute with an appropriate value for the argument.
ResultWirelessOff Indicates that the device entered wireless off mode. Reinitialization is required.
ResultMisalignedSize Invalid bufferSize. The function might succeed if you re-execute with an appropriate value for the argument.
ResultMisalignedAddress The value of pBuffer is invalid. The function might succeed if you re-execute with an appropriate value for the argument.
Other Indicates failure for other reasons.

Description

‏Receives data. (No functionality is provided for getting the address of the sender.)

Other than the parameters, this function is the same as ReceiveFrom. For more information, see the ReceiveFrom function reference.

This function is thread-safe.
