nn::uds::CTR::SendTo Function

Syntax

nn::Result SendTo(
     const EndpointDescriptor & endpointDesc,
     const void * data,
     size_t dataSize,
     u16 destNodeId,
     u8 port,
     bit8 option = 0x00
);

Parameters

Name Description
in endpointDesc Specifies a descriptor indicating the endpoint to use.
in data Specifies a pointer to the data to send. Specify a buffer aligned to SEND_DATA_ALIGNMENT (4 bytes). Specify a buffer for storing data that is the size of dataSize rounded up to a multiple of SEND_DATA_UNITSIZE (4 bytes).
in dataSize Specifies the size of the data to send (in bytes). The maximum size of the data to send is UDS_PACKET_PAYLOAD_MAX_SIZE bytes.
in destNodeId Specifies the recipient. Specify BROADCAST_NODE_ID to broadcast the data.
in port Specifies the port to assign. Port 0x00 is reserved by the system and cannot be used.
in option Specifies the send options.

Return Values

Returns the result of the function. Returns one of the following Result values.

Value Description
Result::IsSuccess Indicates that the operation succeeded.
ResultNotInitialized Indicates that the library is not initialized. Call the Initialize function.
ResultInvalidState Indicates that the function cannot be executed in this state. Returned when this function is executed by a local device acting as other than a host or client.
ResultInvalidNode The target node does not exist on the network. You might be disconnected.
ResultTooLarge Indicates that dataSize is greater than UDS_PACKET_PAYLOAD_MAX_SIZE.
ResultOutOfRange Indicates that an argument was outside of the allowed range. The function might succeed if you re-execute with an appropriate value for the argument.
ResultBufferIsFull Indicates that packets exceeding the transmission capacity of the wireless device were sent when the NO_WAIT option was specified. This result can occur more frequently when the signal strength is bad.
ResultWirelessOff Indicates that the device entered wireless off mode. Reinitialization is required.
ResultMisalignedAddress Indicates that the value of data is invalid. The function might succeed if you re-execute with an appropriate value for the argument.
Other Indicates failure for other reasons.

Description

Sends data to the specified port on the specified partner's system.

This function blocks until it has finished sending data. The send method can be specified by setting option to NO_WAIT or FORCE_DIRECT_BC. To specify multiple options, you can OR multiple values.

This function is thread-safe.
