nn::ulcd::CTR::DynamicStereoCamera::ComputeViewsAndProjections Member Function

Syntax

#include <nn/ulcd.h>
f32 ComputeViewsAndProjections(
     nn::math::Matrix44 * projL,
     nn::math::Matrix34 * viewL,
     nn::math::Matrix44 * projR,
     nn::math::Matrix34 * viewR,
     const nn::math::VEC2 * leftEyeTangent,
     const nn::math::VEC2 * rightEyeTangent,
     const Setting & setting,
     const nn::math::PivotDirection pivot = nn::math::PIVOT_UPSIDE_TO_TOP
) const;

Parameters

Name Description
out projL Specifies the location where the calculated results for the left projection matrix are stored.
out viewL Specifies the location where the calculated results for the left view matrix are stored.
out projR Specifies the location where the calculated results for the right projection matrix are stored.
out viewR Specifies the location where the calculated results for the right view matrix are stored.
in leftEyeTangent Tangent value of the left eye position. Uses the value retrieved by the QTM library. This class works the same as the original StereoCamera class if null is specified.
in rightEyeTangent Tangent value of the right eye position. Uses the value retrieved by the QTM library. This class works the same as the original StereoCamera class if null is specified.
in setting Custom camera settings.
in pivot Upward direction of the camera within the frame of the upper screen (used for rotation).

Return Values

Returns the final calculated stereo factor. A value of less than 1.0, represents a reduction to the parallax limit, stereo factor, or the distance between the left and right cameras by the 3D depth slider.

Description

Calculates the view matrices and projection matrices for the left and right cameras, based on the settings. This function estimates the optimal distance between the LCD screen and the user to provide a realistic stereoscopic effect, or an artistic effect that deviates from reality by using RealisticFovFactor. It is also possible to provide a motion parallax effect by controlling the camera based on the position of the user's eyes. If the left and right eye positions are set to null, this function works the same as the original functions such as nn::ulcd::CTR::StereoCamera::CalculateMatrices. When developing on CTR, these parameters must be set to null. After the projection matrix is calculated, the camera rotation is calculated based on the pivot parameter. No rotation takes place if nn::math::PIVOT_NONE is specified. The process stops if an invalid value is entered. The default argument of nn::math::PIVOT_UPSIDE_TO_TOP is used if no argument is specified.
