nn::ulcd::CTR::DynamicStereoCamera Class

Header File: nn/ulcd.h

Syntax

class DynamicStereoCamera

Description

Class that supports the creation of camera matrices for rendering images that take binocular parallax into account, based on the face tracking feature. This class provides a new stereoscopic image based on eye position information retrieved from the face tracking feature of the QTM library. When the face tracking feature is not used, this class works the same as the original nn::ulcd::CTR::StereoCamera class.

Structures

nn::ulcd::CTR::DynamicStereoCamera::OccupancyVolume The OccupancyVolume structure represents the space that the camera occupies. This structure can be obtained from the output of the nn::ulcd::CTR::DynamicStereoCamera::GetOccupancyVolume function. It provides information about the range of possible movement by the camera when using the DynamicPerspective feature. It can be used for things such as camera and object hit detection.
nn::ulcd::CTR::DynamicStereoCamera::Setting Stores data used to configure the rendering method. This Setting structure is passed to the nn::ulcd::CTR::DynamicStereoCamera::ComputeViewsAndProjections function, but the data is not retained in the nn::ulcd::CTR::DynamicStereoCamera class.

Member Functions

DynamicStereoCamera Instantiates the object with default parameters (default constructor). After the class object is generated, it must be configured with at least one of the nn::ulcd::CTR::DynamicStereoCamera::SetView functions and one of the nn::ulcd::CTR::DynamicStereoCamera::SetWindowFromSize, nn::ulcd::CTR::DynamicStereoCamera::SetWindowFromTangent, nn::ulcd::CTR::DynamicStereoCamera::SetWindowFromViewport, or nn::ulcd::CTR::DynamicStereoCamera::SetWindowFromProjectionMatrix functions to define the space. The nn::ulcd::CTR::DynamicStereoCamera::ComputeViewsAndProjections function can then be called to get the view matrix and projection matrix thereafter.
SetView Sets the view space using a view matrix as input. The view matrix is generated using a function such as nn::math::MTX34LookAt.
SetWindowFromSize Sets the virtual view volume based on the window width and three kinds of depth information. The window origin is set along with the view space. The window height direction is calculated from the screen aspect ratio. The planes, in order of distance from the camera, are assumed to be as follows: near clipping plane, window plane, and far clipping plane.
SetWindowFromTangent Sets the virtual view volume based on the window tangent values and three kinds of depth information. The window origin is set from the tangent values passed as arguments and is unrelated to the view space settings. The planes, in order of distance from the camera, are assumed to be as follows: near clipping plane, window plane, and far clipping plane.
SetWindowFromViewport Sets the virtual view volume based on the screen horizontal FOV, the viewpoint position, and three kinds of depth information. The planes, in order of distance from the camera, are assumed to be as follows: near clipping plane, window plane, and far clipping plane.
SetWindowFromProjectionMatrix Sets the virtual view volume directly from a projection matrix. The projection matrix is generating using functions such as nn::math::MTX44Frustum and nn::math::MTX44PerspectiveDeg.
ComputeViewsAndProjections Calculates the view matrices and projection matrices for the left and right cameras, based on the settings. This function estimates the optimal distance between the LCD screen and the user to provide a realistic stereoscopic effect, or an artistic effect that deviates from reality by using RealisticFovFactor. It is also possible to provide a motion parallax effect by controlling the camera based on the position of the user's eyes. If the left and right eye positions are set to null, this function works the same as the original functions such as nn::ulcd::CTR::StereoCamera::CalculateMatrices. When developing on CTR, these parameters must be set to null. After the projection matrix is calculated, the camera rotation is calculated based on the pivot parameter. No rotation takes place if nn::math::PIVOT_NONE is specified. The process stops if an invalid value is entered. The default argument of nn::math::PIVOT_UPSIDE_TO_TOP is used if no argument is specified.
GetOccupancyVolume Gets the camera occupancy space between the camera plane and the near clipping plane from the DynamicStereoCamera setting.
