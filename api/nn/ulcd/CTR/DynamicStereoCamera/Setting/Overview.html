nn::ulcd::CTR::DynamicStereoCamera::Setting Structure

Header File: nn/ulcd.h

Syntax

struct Setting
{
   f32 StereoFactor;
   f32 ParallaxDistanceLimit;
   f32 DynamicPerspectiveAmplitudeX;
   f32 DynamicPerspectiveAmplitudeY;
   f32 DynamicPerspectiveClampingLeft;
   f32 DynamicPerspectiveClampingRight;
   f32 DynamicPerspectiveClampingBottom;
   f32 DynamicPerspectiveClampingTop;
   f32 RealisticFovFactor;
   f32 WideFovMotionFactor;
   bool TiltEnabled;
};

Description

Stores data used to configure the rendering method. This Setting structure is passed to the nn::ulcd::CTR::DynamicStereoCamera::ComputeViewsAndProjections function, but the data is not retained in the nn::ulcd::CTR::DynamicStereoCamera class.

The StereoFactor member adjusts the parallax amount. Setting this value to 0 turns off the parallax and the display becomes 2D. Setting it to 1 provides the standard stereoscopic display. This parameter is multiplied by the 3DVolume value. The default value is 1.

The value specified in ParallaxDistanceLimit is the distance in millimeters of the far plane. If an object is placed far away, the parallax of the left and right images approaches the distance between the user's eyes (62 mm), but the parallax effect is too strong and does not provide an optimal 3D viewing experience. This parameter can be used to configure a maximum parallax value that provides a better viewing experience. However, note that limiting the amount of parallax also reduces the overall parallax amount in the scene. The DynamicPerspectiveAmplitudeX and DynamicPerspectiveAmplitudeY values are multiplied by the x-coordinates and y-coordinates of the user's eyes. For example, if you set a value of 0 for the X direction, horizontal movement will no longer be applied. The default value is 1. The DynamicPerspectiveClampingLeft, DynamicPerspectiveClampingRight, DynamicPerspectiveClampingBottom and DynamicPerspectiveClampingTop values limit the range in which user motion is applied in each respective direction. For example, if you set a value of 0 for the Left direction, movement to the left will no longer be applied. The default value is 1.

The RealisticFovFactor parameter adjusts the relationship between the realistic camera that represents the eyes of a person at the optimal viewing distance from the screen, and the custom camera that is set in the nn::ulcd::CTR::DynamicStereoCamera::SetView, nn::ulcd::CTR::DynamicStereoCamera::SetWindowFromSize, nn::ulcd::CTR::DynamicStereoCamera::SetWindowFromTangent, and nn::ulcd::CTR::DynamicStereoCamera::SetWindowFromProjectionMatrix functions. When this value is set to 1, the realistic camera is used for rendering. When set to 0, the custom camera is used. The default value is 1.

The WideFovMotionFactor parameter can be used to specify how to move the camera based on the user's movement. When set to 0, the camera moves proportionally to the distance to the screen. For example, if the camera and screen used for rendering is half the distance of the realistic camera and screen, the camera movement is half of the eye movement. When set to 1, the camera movement is the same as the eye movement. Note that this parameter is ignored when realistic camera is used. The default value is 0. The TiltEnabled flag determines whether stereoscopic camera calculations account for the tilt of the head that can be calculated from the positions of both eyes. This is enabled by default.

Member Variables

StereoFactor f32 Parallax adjustment factor. Specify 0 for no parallax. Specify 1 for standard stereoscopic display.
ParallaxDistanceLimit f32 Specifies the maximum parallax at the far plane.
DynamicPerspectiveAmplitudeX f32 This factor adjusts the amount of horizontal head movement. Specify 0 for no horizontal movement. Specify 1 to use the horizontal movement of the head as is.
DynamicPerspectiveAmplitudeY f32 This factor adjusts the amount of vertical head movement. Specify 0 for no vertical movement. Specify 1 to use the vertical movement of the head as is.
DynamicPerspectiveClampingLeft f32 This is used from clamping in the left direction. The value is used in a linear calculation, where 1 is the limit of the value that can be retrieved from nn::qtm::CTR::GetTrackingData, and 0 is the center. Specify a value in the range from 0 to 1.
DynamicPerspectiveClampingRight f32 This is used from clamping in the right direction. The value is used in a linear calculation, where 1 is the limit of the value that can be retrieved from nn::qtm::CTR::GetTrackingData, and 0 is the center. Specify a value in the range from 0 to 1.
DynamicPerspectiveClampingBottom f32 This is used from clamping in the down direction. The value is used in a linear calculation, where 1 is the limit of the value that can be retrieved from nn::qtm::CTR::GetTrackingData, and 0 is the center. Specify a value in the range from 0 to 1.
DynamicPerspectiveClampingTop f32 This is used from clamping in the up direction. The value is used in a linear calculation, where 1 is the limit of the value that can be retrieved from nn::qtm::CTR::GetTrackingData, and 0 is the center. Specify a value in the range from 0 to 1.
RealisticFovFactor f32 Realistic FOV adjustment factor. Specify 0 to use the realistic camera, or 1 to use the custom camera.
WideFovMotionFactor f32 Specifies how to move the camera based on the face position when the realistic camera is not being used.
TiltEnabled bool Flag that indicates whether stereoscopic camera calculations account for the tilt of the head.

Member Functions

Setting Instantiates the object with default parameters (default constructor).
