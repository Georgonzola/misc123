nn::webbrs::CTR Namespace

Description

The namespace for the CTR Internet Browser API.
(It can also be written without the CTR portion.)

The webbrs Library

The webbrs library is for specifying URLs and starting the Internet browser applet.
Use this library when you want to guide users to webpages from applications. For more information about the features of the Internet browser itself, see the 3DS Overview.
To display webpages using the Internet browser, you must comply with the guidelines stipulated in 3DS Guidelines: Internet Browser. Also refer to that document when using this library.
Applications that use this library must call the nn::applet::CTR::Enable function to enable the applet features. In addition, graphics command requests must be stopped when the applet is called. End rendering (by calling the nngxWaitCmdlistDone function, for example).

Transition After Calling the Internet Browser From the Application

The Internet browser applet differs from other applets in that when the user closes the browser that was called from an application, that application is interrupted and the process transitions to the HOME Menu.
In this case, the application processing and the restrictions in effect are the same as those used when the HOME Menu is started by pressing the HOME Button. For more information, see the section about the HOME Button in the 3DS Programming Manual: System.
If the POWER Button is pressed while the Internet browser is running or the process has transitioned from the browser to the HOME Menu, the nn::applet::IsExpectedToCloseApplication function returns true. When processing has returned to the application, promptly begin handling that and transition to the termination process.

What Needs Checking When the Internet Browser Starts

You can start the Internet browser by calling the StartWebBrowser function. At that time, check the following items.

Has a correct URL been passed?

The specified URL must start with either http:// or https://. You can check this by using the IsURLValid function.
If this condition is not satisfied, the StartWebBrowser function returns the error ResultInvalidUrl.

Note that even if the IsURLValid function returns true, the URL might still be invalid for another reason.
Confirm that the specified page can display in the Internet browser.

Is the Internet browser installed?

CTR systems that were part of the initial shipment after sales began did not include the Internet browser.
The Internet browser is installed when the user performs a system update, but otherwise the Internet browser is not installed in these CTR systems.

Use the IsWebBrowserAvailable function to determine whether the Internet browser is installed.

If the Internet browser is not installed when the StartWebBrowser function is called, the error ResultUnavailableWebBrowser is detected internally and is a fatal error.
Consider the possibility of not being able to return to normal after the call, and implement saving the save data when starting the browser in the same way as for HOME Menu transitions in the application.

Local Communications

During local communication the Internet browser cannot start.
If local communication is taking place when the StartWebBrowser function is called, the Internet browser does not start and the function returns the error ResultLocalCommMode.

Parental Control Restrictions

If Parental Control restrictions have been set for the Internet browser, the user must enter a four-digit PIN after the browser starts.
If the correct PIN is not entered, the Internet browser quits without displaying a webpage and the process transitions to the HOME Menu.

There is no way to notify the application about this state.
If you want to avoid this situation, have the application check for Parental Control restrictions using the IsRestrictWebBrowser function and prevent the browser from starting if browser use is restricted.

Consent to the EULA

If the user has not consented to the EULA, the user is asked to accept the EULA immediately after the Internet browser has started. If the user has not consented to the EULA at this point, the Internet browser quits without displaying any webpage and the system transitions to the HOME Menu.

There is no way to notify the application about this state.
If you want to avoid this situation, have the application check for EULA consent using the nn::cfg::IsAgreedEula function and prevent the browser from starting if the user has not consented.

System modes where the Internet browser cannot be started

When the system mode is set to dev1, dev2, or snake dev1, the Internet browser cannot be started.
For more information, see the documentation under CTR Tools > Config.

Classes

Result
nn::webbrs::CTR::ResultInvalidUrl Indicates that the Internet Browser cannot handle the specified URL.
nn::webbrs::CTR::ResultUnavailableWebBrowser Indicates that the Internet Browser has not been installed and cannot be used.
nn::webbrs::CTR::ResultLocalCommMode Indicates that the Internet Browser cannot start because local communications are taking place.

Structures

nn::webbrs::CTR::Config The Config structure storing the data to pass to the Internet browser.
nn::webbrs::CTR::Parameter The Parameter structure passed when the Internet browser runs.

Constants

URL_LENGTH_MAX const size_t : Maximum number of characters in a URL.

Functions

StartWebBrowser : Launches the Internet browser.
IsWebBrowserAvailable Checks whether the Internet browser has been installed.
IsRestrictWebBrowser Checks whether starting the Internet browser is restricted by the Parental Controls.
IsURLValid Checks whether the URL begins with http:// or https://.
