nn::y2r::CTR::SetReceiving Function

Syntax

void SetReceiving(
     void * pDst,
     size_t imageSize,
     s16 transferUnit,
     s16 transferStride = 0
);

Parameters

Name Description
in pDst Image data transfer destination. Must be aligned to BUFFER_ALIGNMENT bytes. The buffer must be allocated from device memory.
in imageSize Specifies the total size of the data to transfer. This size must be an integer multiple of transferUnit.
in transferUnit Specifies the size of a single transfer. To increase performance, we recommend setting a size equivalent to 8 lines.
in transferStride Specifies an offset value (separate from transferUnit) to add to the transfer destination address for each individual transfer. If you want to add an offset per each individual line, you must specify a size equivalent to one line for transferUnit.

Return Values

None.

Description

Configures loading of RGB data. The actual transfer operation takes place after the StartConversion function is called. The system panics if pDst is set to anything other than device memory.

You must use this function to configure transfer settings before you start converting data with the nn::y2r::CTR::StartConversion function.

You cannot specify VRAM as the location to which image data will be transferred.
For more information about accessing VRAM, see CTR Programming Manual: Basic Graphics and other documents.
