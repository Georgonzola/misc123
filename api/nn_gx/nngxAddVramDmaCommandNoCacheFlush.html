nngxAddVramDmaCommandNoCacheFlush Function

Syntax

NNGX_APICALL void NNGX_APIENTRY nngxAddVramDmaCommandNoCacheFlush(
     const GLvoid * srcaddr,
     GLvoid * dstaddr,
     GLsizei size
);

Parameters

Name Description
in srcaddr Transfer source address (in main memory).
in dstaddr Transfer destination address (in VRAM).
in size Size of the transfer data.

Return Values

No values are returned.

Description

Issues a DMA transfer command to the current command list.

DMA transfer command requests accumulate in the current command list. They are used when transferring data from main memory to VRAM without using the GL functions. This operation transfers size bytes of data from the address specified in srcaddr to the address specified in dstaddr.

There are no restrictions on the alignment of values that can be set in srcaddr, dstaddr, or size. However, both srcaddr and dstaddr can be transferred most efficiently by using pointers that have 8-byte alignment.

This function performs the same operations as the nngxAddVramDmaCommand function, except that this function does not flush the cache of the region specified by srcaddr. This function assumes that nngxAdd3DCommand has already been called once for this region, or that cache operations are no longer necessary because, for example, nngxUpdateBufferLight has been explicitly called. Operation is not guaranteed if this function is used when this assumption is not true.

This function generates the following errors.

GL_ERROR_8090_DMPA valid command list is not currently bound.
GL_ERROR_8091_DMPThe size argument is negative.
