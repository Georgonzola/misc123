nngxEnableCmdlistCallback Function

Syntax

NNGX_APICALL void NNGX_APIENTRY nngxEnableCmdlistCallback(
     GLint id
);

Parameters

Name Description
in id ID of the accumulated command that enables the callback.

Return Values

No values are returned.

Description

Enables a callback upon completion of the current command list.

When the idth command accumulated in the command list finishes, the callback function configured by the nngxSetCmdlistCallback function is invoked. You can configure each command accumulated in the command list. When id is set to -1, a callback is invoked upon execution of the last command accumulated in the command list. Exit callbacks are disabled for all commands by default.

If calling callback functions on command requests in progress is enabled, the callback function is called asynchronously after the execution of the next command request starts. In other words, the execution of the callback function happens in parallel with the processing executed by the next command request. When the callback function's process ends, however, the system ensures that the driver does not call the next callback function.

If nngxClearCmdlist or nngxClearFillCmdlist is called on a command list configured to invoke a callback, the callback setting is cleared. The setting is not cleared, however, if an id of -1 is specified.

This function generates the following errors.

GL_ERROR_8012_DMPOccurs when id is 0, a negative number other than -1, or a value greater than or equal to the number of elements allocated in nngxCmdlistStorage.
