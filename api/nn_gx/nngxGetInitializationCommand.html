nngxGetInitializationCommand Function

Syntax

NNGX_APICALL GLsizei NNGX_APIENTRY nngxGetInitializationCommand(
     GLsizei datasize,
     GLvoid * data
);

Parameters

Name Description
in datasize Size of the buffer to which the data pointer points, in bytes.
in data Pointer to the buffer that stores the command buffer.

Return Values

Returns the size of the initialization commands, in bytes.

Description

Gets initialization register setting commands that are executed by the nngxInitialize function.

This function was added to counter problems related to rendering that occur while returning from the HOME Menu or elsewhere when commands to set registers are directly created without using the graphics library supported by the SDK.
Accordingly, there is normally no need to use it.

When 0 is specified for the data argument, no commands are obtained.
Even when this function is called before initialization with the nngxInitialize function, no commands are obtained (a value of 0 is returned).

First, specify 0 in data and call this function, get the size needed for command allocation as a return value, prepare a buffer of that size, and then call this function again.

When values other than 0 are specified in data, the return value is 0.

This function generates the following errors.

GL_ERROR_80B4_DMPdatasize is smaller than the obtained command size.
