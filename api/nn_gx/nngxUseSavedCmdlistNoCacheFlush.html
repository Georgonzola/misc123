nngxUseSavedCmdlistNoCacheFlush Function

Syntax

NNGX_APICALL void NNGX_APIENTRY nngxUseSavedCmdlistNoCacheFlush(
     GLuint cmdlist,
     GLuint bufferoffset,
     GLsizei buffersize,
     GLuint requestid,
     GLsizei requestsize,
     GLbitfield statemask
);

Parameters

Name Description
in cmdlist The command list that is saved.
in bufferoffset The saved information obtained with the nngxStopCmdlistSave function.
in buffersize The saved information obtained with the nngxStopCmdlistSave function.
in requestid The saved information obtained with the nngxStopCmdlistSave function.
in requestsize The saved information obtained with the nngxStopCmdlistSave function.
in statemask Specifies a state flag that creates a complete command (bitwise OR).

Return Values

No values are returned.

Description

Adds the saved command to the current command request. (No cache flush.)

This function provides the same functionality as when calling nngxUseSavedCmdlist using GL_FALSE specified in copycmd. However, this function uses different error codes and does not flush the added 3D command buffer. If you do not need to flush the 3D command buffer to be added when calling nngxUseSavedCmdlist and passing GL_FALSE for copycmd, use of this function instead can eliminate the cost of flushing the cache.

Make sure that the application guarantees that the cache is flushed for the 3D command buffer region to be added.

For information about other parameters and features of this function, see nngxUseSavedCmdlist.

This function generates the following errors.

GL_ERROR_8092_DMP0 is bound to the current command list.
GL_ERROR_8093_DMPInvalid value specified for cmdlist
GL_ERROR_8095_DMPThis function has added saved commands that exceed the maximum size of the current command list’s 3D command buffer and command requests
