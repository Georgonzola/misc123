DevMenu

Overview

DevMenu has the following three display modes.
    1. Program List Mode
      Displays the program imported to the various media.
      Can start the CTR card.
      Can start and delete imported applications.
    2. SDMC and HIO Mode
      Displays files on the SD card or on the PC.
      Can import CIA files.
    3. Expanded Save Data Mode
      Displays expanded save data.
      Can delete expanded save data.

Cautions When Launching From Cards

TWL cards cannot be started from DevMenu.
Application region-checking is not performed by DevMenu.

Button Assignments

ButtonOperation
↑↓Moves the cursor.
←→Switches modes.
ASelect / Run
BBack / Cancel
XDelete (when possible)

As a specially prepared command, you can delete all imported SDK tools by pressing L + R + Left + B + START.
When you do this, DevMenu itself is deleted. To revert to the original state, run System Updater.

Program List Mode

In this mode, press the A Button to start a program, and the X Button to delete a program.
While holding the L and R Buttons, press the X Button to delete all programs that are currently displayed.

The upper screen displays the following items for the program description.

For CTR applications, Application is displayed for retail versions, Demo for downloaded application demos, and AddOnContents for add-on content. For TWL applications, TWL is displayed.

For CTR applications, the unique ID and product code of the title code specified in the RSF file appear in the lower screen.

For TWL applications, the initial code and title name specified in the RSF file appear. 
 34534e41    MY APP NAME
Some of the system applications and system data titles that can be used from the CTR application are displayed.
System applications and system data titles are described separately below.

System Applications Shown in the Lower Screen

These applications can be deleted to verify operations when system applications are not in the CTR application on the system.
Select one of the programs in the list as shown below, and press the X Button to delete it.
You cannot remove multiple programs at the same time. (You cannot use the batch-delete operation of pressing the X Button while holding down both the L Button + R Button.)

After you delete an item, reset the debugger or power cycle the system.
If you keep using the system without doing this, the HOME Menu will freeze.

    System Data Titles Shown in the Lower Screen

    The following system data title is used to create the state of a system update for the CTR system.
    By deleting this system data title, you create the state of a system update for the CTR system.
    Deleting it has no effect on system operations.

    To revert to the original state, perform a system update from System Settings.

    Dummy Data Title

    MarketDisplayed Name
    All markets00108 Dummy data title
    Applications that use the eShop REST API must check whether the system needs to be updated. This feature is used for that check.
    If there is an error to display, error code 009-1000 is shown.
    Even for e-commerce applications that do not use the eShop REST API, this feature can be used to emulate the need for a system update.

    SDMC and HIO Mode

    In this mode, you can import CIA files on an SD card or on a PC by using HostIO into importable media.
    Press the A Button, and then press the A Button again to confirm that you want to import the program.
    The media that the CIA file is imported to is automatically determined from the CIA file information.
    While holding the L and R Buttons, press the A Button to import all CIA files in the current directory.
    If an error occurs while importing all files, any files not yet imported at the time of the error are not imported.

    The upper screen displays the following items for the CIA file description.

    ItemDescription
    File TypeThe file type.
    Displays CIA (for a valid CIA file), Invalid cia, Directory, or Unsupported (for any other file type).
    CategoryApplication type.
    For CTR applications, Application is displayed for retail versions, Demo for downloaded application demos, and AddOnContents for add-on content. For TWL applications, TWL is displayed.
    Unique IDUnique ID (only shown for CTR applications).
    Import MediaThe media to import to.
    For CTR applications, this shows SDMC. For TWL applications, this shows CTR NAND.
    VersionRemaster Version
    File SizeThe size of the CIA file.
    Required SizeThe free space needed for import. The exact amount of space required to download a program from the eShop cannot be determined locally.

    If using HostIO, you must set the environment variable DEVMENU_ROOT on the PC and specify the directory to use for import.
    Then, you must start HIO-Daemon.
    If you are using development hardware, you can import from a PC using PARTNER-CTR UIC-MIDI.
    However, when using UIC-MIDI, the HostIO speed is slower. The time it takes to import on CTR development hardware is approximately 1.5 times what it takes on PARTNER-CTR Debugger, and the time on SNAKE development hardware is approximately 2.0 times what it takes on IS-CTR-DEBUGGER.

    You can reopen a currently open folder by using START.

    You can delete CIA files by using the X Button.
    Press the X Button while holding down the L and R Buttons to remove all CIA files in the current directory.

    Press the A Button while holding down the X Button to remove CIA files after importing them.
    Press the A Button while holding down the L/R/X Buttons to remove all CIA files after importing all CIA files in the current directory.

    Save data is retained if you reimport a program that has already been imported.
    However, if the application being imported has a differently sized save data region than the application being overwritten, the entire contents of the save data region are cleared. If you need to clear the save data, delete the imported program in program list mode before importing it again.

    If you attempt to import a file that is identical to a program that was previously imported, the following message appears, and the import is canceled. 

     Program already exists.
 Import canceled.

    If the SD card is write-protected by the LOCK switch, the following message appears, and file operations are disabled. Make the SD card writeable and try again. 

     SD card is write protected.

    After attempting to import a corrupted CIA file, import is not performed and instead a message similar to the following is displayed.
    Check that the CIA file was created correctly. 

    This cia is invalid.

    Expanded Save Data Mode

    In this mode, you can display and delete expanded save data.
    You can only delete expanded save data on a per-ID basis.
    To delete, select the ID of the directory with data you want to delete, and press the X Button.

    The HOME Menu uses the expanded save data "00000000 00000082."
    Because the HOME Menu mounts this expanded save data, you cannot delete it if DevMenu is running from the HOME Menu.
    If you need to delete this data, run DevMenu from the development menu.

    Free Space Display

    The Free Space field in the upper screen shows the free space for the SD card and the NAND memory of the system. The top row shows the size in bytes, and the bottom row shows the block size, the number of free 128-KB blocks. The block size is used by Nintendo eShop and the System Settings.

    ItemDescription
    SDThe free space on the SD card to which the CTR application is imported.
    TWLThe free space in the system NAND memory to which to TWL application is imported.
