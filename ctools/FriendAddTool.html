FriendAddTool

Overview

The FriendAddTool reduces the workload involved in building friendships when developing software that uses the friend feature.

Specifically:

  • It allows you to register friends by batch reading files that include the friend codes to be registered from an SD card when you want to test use the friend feature among several CTR handheld systems.
  • It allows you to simulate situations where the friend list has become full by registering dummy friends when preparing a large number of devices might prove difficult.

Registering Friends From an SD card

If you want to register several friend codes, you can register them all at one time by loading a file that includes those friend codes from an SD card.

Registration Procedure

To register friends from an SD card, first collect the friend codes of devices between which you want to establish friendships.
Next, save the collected friend codes to an SD card as a text file (friend_code.txt).
Insert the SD card into a CTR handheld system, start the tool, and batch register friend codes listed in friend_code.txt by executing Register listed Friends in SD.

Registering friends from an SD card has the same result as if friend codes were entered by selecting the system feature Register Friend followed by Internet on the Friend List. This procedure handles only one side of friend registration. The other system must also complete registration to establish a friend relationship.

Important

  • Communication with the friend server is required to register friends.
    Be sure to make connection settings with the access point ahead of time under System Settings.
  • Even if friend registration is being restricted by Parental Control, this tool ignores that restriction and registers friends anyway.

friend_code.txt Format

Place the text file to load (friend_code.txt) in the root directory of the SD card.
Start the tool and press the load button. The file is loaded from the SD card and saved as a list inside the tool.

List the friend codes to be registered one per line using single-byte numeric characters.
Friend codes are loaded ignoring any single-byte hyphens or single-byte spaces that may be included.

Important

  • Any lines that contain characters other than single-byte numeric characters, hyphens, or spaces are skipped.
  • If the friend code of the system registering friend codes is encountered, that line is ignored and the next line is loaded.
  • Up to 100 friend codes can be loaded from an SD card. If the file contains more friend codes than that, friend codes past the 100th are not loaded.
  • The maximum size of this text file is 5120 bytes. If this file size is exceeded, no data is loaded at all.

Example of friend_code.txt

012345678901
1234-5678-9012
2345 6789 0123

Registers Dummy Friends

If you want to verify conditions when many friends have been registered, but cannot ready enough separate devices to fully populate the friend list, you can simulate a full friend list by registering dummy friends.

Registration Procedure

Start the tool and execute Register Dummy Friends. Dummy friends are registered until the number of friends reaches 100.

Dummy Friends

The result of registering a dummy friend is the same as registering a friend by selecting the system feature Register Friend followed by Local on the Friend List.
As with a regular friend, you can get the friend key, Mii and Mii name, system profile, and friend relationships for a dummy friend.

Cautions About Dummy Friends

Although a friend relationship is established, registration has only been made on the system running the tool. The status is the same as if you were removed from the friend list on the other system. There is no change in the status of the friend being online or a Mii being updated.
Because the account does not really exist on the server, it cannot be used in server-based processing such as obtaining ranking information using the NEX library.

Removing Friends

All friends, friends listed in friend_code.txt, and dummy friends
Friends can be removed all at one time by using these range specifications.

Screen Description

The number of friends, one's own friend code, and a description of operations are displayed on the upper screen. A log of operations and results are displayed on the lower screen.

How to Use

The method of operations are as follows.

Registering Friends

Button Operation
A while holding down L Registers dummy friends.
A while holding down R Registers friends listed on the SD card.

Removing Friends

Button Operation
X while holding down L Removes dummy friends.
X while holding down R Registers friends listed on the SD card.
X while holding down L and R Removes all friends.

Other Settings

Button Operation
Y Reloads data from the SD card.
