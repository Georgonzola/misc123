Introduction

ctr_VertexCacheSimulator simulates the vertex cache of the GPU built into the CTR.

How to Use

Commands

% ctr_VertexCacheSimulator32 <input filename> [options...]

One filename for simulation target must be provided as the input file. Then, option specifications can be made. Option specifications are in no particular order.

Input Files

Specify the files to be input to <input filename>.
Specifies the binary file of the unsigned short-type index array.

Options

The following options can be specified in place of [options...].

Options Description
-s<number of indices> Checks the number of indices.
Specify a decimal value for the argument.
And error is output if the specified number of indices and the number of indices in the input data differ.
There is no error check if this option is not specified.
-m<maximum index value> Checks the maximum index.
Specify a decimal value for the argument.
An error is output if an index exceeding the value of the specified maximum index is discovered in the indices within the input data.
There is no error check if this option is not specified.
-short_info_result Outputs only the value of the cache hit-rate results.
-? or -help Displays help.

Case

C:\>ctr_VertexCacheSimulator32.exe -?
C:\>ctr_VertexCacheSimulator32.exe index.bin
C:\>ctr_VertexCacheSimulator32.exe index.bin -s180
C:\>ctr_VertexCacheSimulator32.exe index.bin -m300
C:\>ctr_VertexCacheSimulator32.exe index.bin -m300 -s180
C:\>ctr_VertexCacheSimulator32.exe index.bin -m300 -s180 -short_info_result

Result Output

The output results are as indicated below.

Input index count: xxx (Index count of the input index array)
Cache hit count: xxx (Number of vertices that had a cache hit)
