ctr_makeMetaDataContent

Overview

ctr_makeMetaDataContent is a tool for creating required content (metadata content) from XML data.
For more information about the required metadata content, see the overview.
Use CTR Master Editor version 1.8 or later to check the attributes and icons of created required content.

Runtime Options

Option Description Examples:
-o, --output-file Specifies the name of the CFA file to create. --output-file metadata.cfa
-m, --metadata Specifies the XML file containing the metadata. --metadata metadata.xml
-d, --icons-dir Specifies the directory containing the image data. --icons-dir ./icons
-i, --icon Specifies the icon file (ICN file) for the data title.
Note that the icons for the data title specified here are different from the icons used for the content.
For the data title, specify icons generated using BSF files.		 --icon metadata.icn
-Dname=value Specifies options to pass directly to ctr_makerom. For more information about specifying options, see RSF Files Used to Create Add-on Content in ECDK: Making Data Titles. -DUNIQUE_ID=0xFFFFF

ctr_makeMetaDataContent32.exe -o meta.cfa -m meta.xml -d ./icons -i meta.icn -DUNIQUE_ID=0xFFFFF -DDATA_TITLE_INDEX=0 -DPRODUCT_CODE=CTR-M-META-01 -DTITLE=title

Metadata XML Format

Metadata XML uses the following format.
Also refer to the sample included in the ECDK package.

metaData

Root element for metadata XML.

formatVersion

XML format version. The current version is 0.2. 
<metaData formatVersion="0.2">
.....
</metaData>

metaData/regions

Specifies content attributes.
It must describe the following information.

code

The region code. See the table for available codes.

defaultLang

The UI is in this default language if the user configures a language that is not supported in the region. See the table for the available codes.

language

Describes the languages supported in the region. See the table for the available codes.
<regions>
    <region code="JPN" defaultLang="ja">
        <language>ja</language>
    </region>
    <region code="USA" defaultLang="en">
        <language>en</language>
        <language>fr</language>
    </region>
</regions>

metaData/contents

Specifies content attributes.
It must describe the following information.

title

Content title, up to 64 bytes long (including the string terminator) in UTF-8 encoding.

DESC

Content description, up to 128 bytes long (including the string terminator) in UTF-8 encoding.

iconIndex

Content icon index number.
See below for the index number definition.

lang

Indicates the language of the content. Specify languages as REGIONCODE_languagecode.
For example, specify Japanese as JPN_ja.
Market Useable Market Language Codes
Japanese market JPN_ja
North American market USA_en, USA_fr, USA_es, USA_pt
European market EUR_en, EUR_fr, EUR_de, EUR_it, EUR_es, EUR_nl, EUR_pt, EUR_ru
Chinese market CHN_zh
Korean market KOR_ko
Taiwan/Hong Kong market TWN_zh
  <contents>
    <content contentIndex="1">
      <contentInfo lang="JPN_ja">
        <title>First content item name</title>
        <desc>First content item description</desc>
        <iconIndex>0</iconIndex>
      </contentInfo>
    </content>
    <content contentIndex="2">
      <contentInfo lang="JPN_ja">
        <title>Second content item name</title>
        <desc>Second content item description</desc>
        <iconIndex>1</iconIndex>
      </contentInfo>
    </content>

metaData/icons

Describes the image files to include in the data title.
Use the iconIndex values to specify the content and item attributes.

  <icons>
    <icon iconIndex="0" fileName="icon1.ctpk"/>
    <icon iconIndex="1" fileName="icon2.ctpk"/>
    <icon iconIndex="2" fileName="icon3.ctpk"/>
    <icon iconIndex="3" fileName="icon4.ctpk"/>
  </icons>

Icon File Format

The following lists the icon images that can be included in metadata content.

Image Formats

- Image size: 48px high x 48px wide
- Image format: RGB565 (NW4C texture format with no header data.)

How to Create Images

Either use the ctr_TexturePackager tool, or use the plug-in included with NintendoWare.
