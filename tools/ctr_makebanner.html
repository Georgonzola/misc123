Introduction

ctr_makebanner is a tool for creating BNR (ctr BaNneR model data) and ICN (ctr ICoN data) files.
BNR and ICN files are generated based on the content in banner spec files (BSF).

BNR files contain banner model data and sound data. ICN files contain banner headers and 2D icon data.

How to Use

Requirements

You need the following to create BNR and ICN files.

  • A CBMD file containing banner model data created using ctr_BannerModelConverter
  • A CTPK file for large and small icons generated by ctr_TexturePackager, or an NW4C TGA file for large and small icons created by the NW4C plug-in.
  • A BCWAV file containing banner sound data created using ctr_WaveConverter
  • BSF

BSF files contain the information that is required in order to create BNR and ICN files. They must be created individually for each application.

Commands

% ctr_makebanner32 [-d] BANNERSPECFILE [BANNERFILE] [ICONFILE]
% ctr_makebanner32 [-d] -noicn BANNERSPECFILE [BANNERFILE]
% ctr_makebanner32 [-d] -nobnr BANNERSPECFILE [ICONFILE]

You must specify BANNERSPECFILE (the BSF file). All other options are optional.

Options

Options Description
-noicn Does not output ICN files.
-nobnr Does not output BNR files.
-d Provides detailed output for debugging.
BANNERFILE Specifies a BNR file. If omitted, the output file has the same basename as the BSF file used as input, but has a file extension of .bnr instead of .bsf.
ICONFILE Specifies an ICN file. If omitted, the output file has the same basename as the BSF file used as input, but has a file extension of .icn instead of .bsf.

BSF

This file contains the settings for the BNR and ICN files to generate.
It must be coded and saved in Unicode (UTF-16LE with BOM).
Notepad in Microsoft Windows 2000, Windows XP, Windows Vista, and Windows 7 supports Unicode.
Select Unicode from the Encoding drop-down list when saving the file.

Parameters

The delimiter between parameter specifications and values is a colon (:). If the same parameter occurs more than once, the instance written last in the file is used.

Format

Parameter specifications must come at the beginning of each line.
All characters after a "#" are treated as a comment.
Space and tab characters at the beginning and ending of the value are ignored. All other spaces are included in the string. If you want to include leading spaces or if you want to include quotation marks ("), "#" symbols, or colons (:) in the string, surround the string with quotation marks ("").

Data Restrictions

The following restrictions apply to banner data.

Restriction on Data Size

The compressed volume of data after conversion must be no larger than 1 MB.
For more information, see ctr_BannerModelConverter.
Note: Data in violation of these restrictions causes an error during conversion by BannerModelConverter.

Sound File Restrictions

The following restrictions apply to sound files.
Note: Data in violation of these restrictions causes an error during conversion by BannerModelConverter. (Not implemented.)

  • Data length must be no longer than 3 seconds.
  • There are two channels (stereo).
  • The file is not configured to loop.
  • The compression method is either DSP ADPCM, PCM16, or PCM8.

Icon Cache

Icon cache refers to icon data stored temporarily on the SD card so icons can load faster on the HOME Menu.
Once icon data is stored in icon cache this way, it is used from there until the icon cache is explicitly deleted.
To delete icon cache, start the HOME Menu while holding down the X Button and Y Button.
